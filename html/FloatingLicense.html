Floating License

In this section, users will learn the ways to install and use the Agnisys® License Manager (ALM) server. ALM is a web-based license server that enables the user to use floating licenses with Agnisys® tools. ALM is capable of overcoming following challenges that are usually faced by license managers:

  • Existing license servers were not capable of providing the special functionality needed by the MS Office plugin.
  • Existing license servers did not use a widely available modern web-based server.
  • Existing license servers were expensive, and Agnisys® did not want to pass the additional costs to our valued customers. 


Note:The ALM server (version 1.2 and later) supports floating licenses only. Node locked and site licenses are still served by older file based licenses.


Requirements


ALM has a very small footprint and can run on any Windows or Linux machine. The following are the minimum requirements to install ALM server: 

  • A minimum of 250 MB of disk space. 
  • Active and stable LAN connectivity. 
  • MachineIP: 2464 is open, and not occupied by any other application such that ALM server can easily access this port. 
  • Administrator privileges.



Server Download and Setup


  • Download the ALM zip file from http://www.agnisys.com/release/alm/ALM_2.7.1.0.zip 
  • Unblock the file:  
    • Right-click the downloaded zip file, select properties, and click Unblock (This step is mandatory for ALM service run under Windows). 
    • Unzip the file. For more details on installing the server, please refer to the Server setup section.



Server Setup


The server is identified by a Universally Unique Identifier (UUID) that is generated by getUUID.sh/getUUID.bat. To setup the server: 

Run the UUID generator. A uuidfile.txt file is generated. 

 

  • Send an email to support@agnisys.com to receive a license file (license.dat). 
  • Save the license.dat file at the root. Ensure that the license file is in the same directory as the readme.txt file at the ALM_x.x.x.x directory. 
  • Run the server.  
    • Run server without verbose using startup.sh / startup.bat 
    • Run server with verbose using startup.sh –v. 
    • For Windows, set the environment variable AGNI_LICENSE_DEBUG=1,  DOS> startup.bat 
      • run ALM server as a Window service, login with administrator privileges, and run InstallWindowsService.bat.
      • For more details, please refer to the readme.txt. 
  • Checkout License in the client (IDSWord). See more details in the Client setup section.


Note: If you are using IDS-WordEditor or IDS-ExcelEditor, then don't set AGNI_LICENSE_FILE variable.


Client Setup


  • To setup the client, set environment variable AGNI_LICENSE_SERVER or AGNI_LICENSE_FILE to the path where the server is running, for example 192.168.2.100:2464.
  • To enable the Agnisys® Ribbon, click on the Checkout button as shown below:

                                       

                                                                   


  • After the license is checked out successfully, the IDesignSpec™ ribbon is enabled and a message “Successfully checked out license” is displayed. If there is any issue regarding license check out, respective error messages will be displayed using which user can also proceed troubleshooting the issue.


Note: The check-in happens automatically when the application (Word/Excel) is closed, or the machine crashes, or is shut down. Also, IDS Ribbon can be disabled manually by clicking on the 'CheckIn' button in toolbar.


Server Status


To check the server status, user can either:


  • Visit https://192.168.2.100:2464, and login using the default username and password: admin/admin.
  • Or, click Status in the IDesignSpec™ ribbon. A Security Alert window pops up. Click Yes to proceed. 


                                                   



  • A License Dashboard window appears. 



                                                     



Note: Users can check the details of all the users using the license from the license server, and the product information of all the products encapsulated in the licenses.
