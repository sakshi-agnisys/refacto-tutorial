IDS with Vivado

Introduction

Xilinx is a supplier of high performance, high density Field Programmable Gate Arrays (FPGAs). IDesignSpec™ is a specification driven system that targets FPGAs and Application Specific Integrated Circuits (ASICs) for Intellectual Property (IP) and System on Chip (SoC).  This document details the integration between IDesignSpec™ and Xilinx tools. 

This integration enables users to have an uninterrupted flow of information from the specification to the bitstream, helping create a more efficient development methodology.


The Problem of Addressable Registers

Hardware designs with configuration and status registers that are addressable by a register bus such as AXI4LITE are very common. Manually creating registers in RTL, then verifying them using the popular Universal Verification Methodology (UVM), creating the other artifacts and code fragment for firmware is tedious and error prone.

The RTL can be automatically generated with a register generation tool however, the problem of integration with rest of the application logic still remains. Manual integration is again time consuming and prone to errors when the specification changes. 

Vivado’s interface offers a good solution in this integration process. However, files need to be generated such that the integration is possible without any hiccups along the way.

Supported flows with IDesignSpec™(IDS)

IDesignSpec™ (IDS) is a tool for capturing specification of a hardware. It is a specification driven tool, which enables Architect/System Designer to capture hardware/software specifications in any format such as Word, Excel, SystemRDL, IPXACT, RALF or CSV  and generate all possible outputs like Verilog, VHDL, SV, SystemC, UVM, C headers, SystemRDL, IPXACT, HTML, Datasheet etc. without any data re-entry or duplication


Figure 7: IDesignSpec™ Flows


Note that IDesignSpec™ can be used to create many outputs and can import many inputs as well.


The Vivado-IDesignSpec™ flow

The joint IDS, Vivado flow enables users to have a very efficient process that is devoid of any duplicate data entry. The information flows seamlessly from one system to another. The basic linkage is through Verilog files generated by IDS and imported into Vivado. IP-XACT created by IDS has register, pin & parameters information, while that generated by Vivado has the constructs that enable it to work inside the Vivado system. 


Figure 8 Joint Vivado - IDesignSpec™ flow


Vivado Requirements

Vivado v2015.4.2  and above

IDesignSpec™ Requirements

IDS ver 5.20.40 and above

Detailed instructions for creating the joint flow

  • Create a register specification in any IDS supported formats, like MSWord,  MS Excel, SystemRDL, IPXACT, RALF, or CSV
  • Generate Verilog RTL and a Vivado's TCL API supported script using IDesignSpec™.
  • Create a project in Xilinx Vivado
  • Source the generated script (IDS_vivado.tcl) to automatically add sources,  compile the design, create and package IP with AXI interface

Benefits of the integrated flow


  • Capture specification in any IDS supported format
  • Automatically generate RTL and many other outputs using IDS
  • Import the RTL to Xilinx Vivado
  • Package it to a complete IP with register maps
  • Seamless integration with Xilinx FPGA

Conclusion

The joint IDesignSpec™ with Xilinx Vivado flow makes the whole “Specification to Bitstream” process fully automated.
