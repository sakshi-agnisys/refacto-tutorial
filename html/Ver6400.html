24th Feb 2017

  • RTL Enhancements
    • Paged Block - User can select blocks on the same address using paging technique. Paged Block
    • Concatenation operator - Repeating "default" value for register can be specified using SV concatenation operator. Concatenation operator
    • Avalon Interface - IDS support bursting and invalid address errors in case of Avalon bus. Avalon Interface
    • RTL property (rtl.byte_enable) - To exclude byte enable signal in case of Proprietary. 
    • Offsets and Defaults in VHDL - Offsets and field defaults constants are now declared in package instead of architecture. Records of field defaults are created in package. More details.
    • New VHDLalt2 architecture - VHDLAlt2 also uses the VHDLAlt1 architecture. As before, Records are used in alt1 while alt2 uses flat ports.
    • registered=false - Fields can be written by SW. rtl.precedence is also supported. Unregistered Fields
    • sw_bit_enable - New property that points of a reg/field which masks the write to another register/fields bits. More details
    • Parameters > 32 bits - Supported parameters greater than 32 bits in VHDL by using std_logic_vector. This avoIDS integer overflow situation.


Backward Incompatibility with a work-around

    • High priority of hardware interrupts - The Hardware driven interrupt event is now higher priority then acknowledging or software interrupt event, but earlier behavior can be obtained by using {rtl.precedence=sw} property. More details
    • registered=false - Additional ports are now generated rd_valid_out and wr_valid_out. These can be used for any trigger in the application logic. For backward compatibility use software access "ro" and hardware access "rw/wo/na". More details  
  • UVM Enhancements
    • uvm.coverpoint.b=dut - Defining user-defined coverpoint value for register bit coverage in UVM. More details
    • Indirect register - Are now supported inside the reg group also.
    • IP-XACT access write as read (wr) - Is supported in uvm as read-write (rw).
  • Cheader Enhancements
    • cheader.name_format - Can be used to change the naming scheme for the Cheader outputs.
