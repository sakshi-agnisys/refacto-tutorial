RW registers by accessing structs in ISS Firmware

Introduction


ISequenceSpec supports write/read to CSR by using write/read register APIs. In this the data and the address is given to the APIs and accordingly the register write/read happens.

 

Another way to access the registers is using the ‘Structs’ directly. In this case a  pointer to the structure will be made and accordingly the register write/read will be supported. For accessing the registers by using structures, no write/read APIs for the registers will be required.


Python input


class sequences: 

    def seq_name1(self,ip='chip1.rdl',desc='     '):

        var1 = iss.variable(0,'var1')

        var2 = iss.variable(0,'var2')

        iss.write(Block1.reg_name.f1,0x2)

        iss.write(Block1.reg_name.f2,0x3)

        iss.write(Block1.reg_name2,0x5)

        iss.read(var1,Block1.reg_name2)

  iss.write(Block1.reg_name2.f1,0x7)

  iss.read(var2,Block1.reg_name)


This functionality can be switched on by making the “structure” option  to “1” in the Firmware configure section in the python file.

With Base Address property


Base address can be used to place the entire sequence to a particular offset. In that case, “baseAddress” will be added to the address of the top component. For example:

volatile chip_name_s *ptr = (chip_name_s *) (chip_name_s_ADDRESS + baseAddress);
