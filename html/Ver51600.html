Ver 5.16.0.0

v5.16.0.0

Apr 28th 2015


IDesignSpec™ (IDS-Word, IDS-Excel, IDS-Batch CLI) Enhancements

  • RTL Enhancements
    • Protection and response support (AXI and APB bus, in Verilog): IDesignSpec™  AXI bus has been enhanced to support protection(AWPROT, ARPROT) and response (RRESP, BRESP) signals in Verilog. For details click here.
    • WISHBONE Bus: The WISHBONE BUS standard protocol used to interconnect architecture for Portable IP Cores has been supported in IDesignSpec™. For more details click here.
    • Signals in VHDL(alt1): "Signals" can now create named external ports, in the block, and connect certain internal design signals to the external world, in VHDL(alt1) RTL also. This feature was already available in Verilog. For more details click here.
  • UVM Enhancements
    • UVM Package in separate file (Minor Backward Incompatibility): IDS will now generate the UVM output in two files, one is "<file_name>_pkg.regmem.sv" which is a package, that will include the generated reg model and all required UVM packages. Second one is the old register model file "<file_name>.regmem.sv" which is generated by IDS but without package. This way, user has more flexibility to either use the package or use the regmodel without the package. User will need to include "<file_name>_pkg.regmem.sv" instead of "<file_name>.regmem.sv" in the environment.
    • Name Format Properties: The generated names class names in UVM can now conform to a format. For more details click here.
    • Multiple File Generation: IDS can now generate multiple files for UVM output. Separate files will be generated for each block/chip which helps reuse. For more details click here.
  • ARV™ Enhancements
    • Support for different SW access:ARV™  enables verification of registers with almost all kind of UVM software accesses.
    • Custom RTL integration: Enables user/custom RTL to be integrated inside ARV™ environment. This is done by binding user RTL in ARV™ top level module.
    • External registers inside section: ARV™ supports verification of external registers and external registers within the section. A separate Verilog module is created with all external register signals declared. User may have to use those external signals and insert it's external logic in this module, in order to test the external registers behavior.
    • Verification of registers with property "registered= false" supported in ARV™: Dealt the same way as external registers
    • Shadow Register: Shadow register implementation in UVM also supported, with new sequences to test their behavior. Shadowing of registers within section supported as well.
    • Improvement in code coverage: with the support in verification of different kinds of registers like external-true, shadow registers, registered=False registers etc and for register with mixed access.
    • Functional Coverage up-to 100%: with the enhancements in the ARV™ sequences and with the availability of knobs for changing the parameters of the simulation
    • Bit Enable : Now bit enabling is supported in case of the IDS Proprietary bus.
  • Documentation output Enhancements
    • Special Registers in HTML: Special registers with additional information will be viewable in IDS HTML(alt1 and alt2) outputs. For more details on HTML outputs click here.
    • PDF alt2 output in IDS-Excel: A customizable form of output in IDS-Excel that enables selection of columns (IDS Header), which will go in the "Tabular PDF" form of documentation for design specification. For more details click here.
    • HTML(alt2) responsive view: The web documentation view in form of HTML(alt2) has been made responsive according to the window size.
  • TCL API Enhancements 
    • IDS_outdir: Variable to store the path of the directory where IDS outputs are generated.
    • get_params: Command to return the list of all parameters. 
    • get_IDS_version: Command to return the version of IDesignSpec™.


For more details on enhancements in TCL API for custom output generation, click here.

  • New Outputs
    • C++ output: C++ header file to contain classes for register groups, blocks, and chips alongwith standard macros for read/write masks etc. For more details click here.
    • VHDL Header: VHDL package consisting of constants and records related to register spec. For more details click here.
    • Verilog Header: Verilog header consisting of parameters related to register spec. For more details click here.
  • General Enhancements 
    • Define/Parameter Dependent Properties: Value to IDS property can now depend on defines/parameters. For more details click here .


    • Variant Dependent Properties: Variants in IDS can now be used to create variation on various properties in a single specification. For more details click here.


    • Environment Variables in Ref Path: Environment/System Variables are now supported in IDS, while specifying the path of the referred document. For more details click here.


    • Auto calculation of Size: The block/chip/board size is now configured to be null by default, in "Configuration" settings. IDS will automatically calculate the size on basis of registers in the specification document.


    • New Properties
      • "output_file_name" property: To change the generated output file name with the specified property value for Verilog, VHDL and UVM outputs. For more details click here.
      • "uvm.field_class" property: To change the field class when it applies on the top component(chip/block/reg) in UVM. For more details click here.
      • "hdl_path_internal" property: This property enables users to add a special prefix to the "hdl_path" of internal registers in UVM. This is useful for specifying external and internal registers in a block. For more details click here.


    • RALF import
      • IDS now supports "user_code" in ralf import
      • It is now possible to import a RALF code with multiple bus domains and generate required outputs. For more details click here.

IDS-Excel Enhancements

    • Defines/Parameters: IDS-Excel, user is now able to create a define table and specify various parameters/defines, typically at the top of the sheet and use them as macros at different places in the spec. For more details click here


IDS-Word Enhancements

    • Performance of Table of Content update process: "Table of Content" generation/update process has been accelerated to save time while performing sanity checks and generating outputs.
    • Track Changes: MSWord functionality enabling users to track changes in the document, has been enhanced in IDS-Word, to track any changes while writing the register specification and automatically updating register's address and default value. For more details click here.
    • Address Sorting: A new feature in IDS that enables registers to be specified in any order, irrespective of their address/offset. For more details, click here.


IDS-Batch CLI Enhancements

    • Excel cell expressions: IDS-Batch CLI with excel as input file, has been enhanced to support all Excel Formulae and Functions in the workbook, while writing register specification.
    • Ref : Referred file of type "DOCX" is now possible with all possible inputs in IDS-Batch. For more details click here.
    • XQuery Support: Generation of Custom XML is now supported in IDS-Batch. With the help of an XQuery file user can generate their own Custom XML file from any input file types supported by IDS-Batch. For details click here.
    • Apple MAC OS X is now supported. Download a separate executable from the download site.

Bug Fixes

  • Navigation Links in HTML(alt1 and alt2), at block level were not navigating to correct location

Now it has been fixed


  • "uvm.reg_class" was not working on the top level component

Now it has been fixed to follow top to down hierarchy


  • Multiple Bus Domain Bug: Wrong size calculation when size or offset was defined by the user. The size was not adding to register offset

Now it has been fixed


  • AXI Widget Bug: "rvalid" used to de-assert without waiting for "rready" acknowledgment. 

Now it has been fixed to wait for "rready" from initiator


  • Illegal bins didn't generate appropriately for registers with side-effects.

Illegal bins have been redefined for SW accesses with side-effects. Fixed Now.
