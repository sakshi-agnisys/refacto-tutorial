We Property

'we' property is used to change the write enable of a specific field. It can be used in two ways: 

  1. we = signal_name/ reg_name.field_name/ reg_name@property_name
  2. we = signal_name:reg_name.field_name / signal_name:reg_name


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL

Limitations:

  1. When property used in the way “signal_name:reg_name.field_name”, then the width of the fields should be same for both the registers i.e.  the register on which property is applied and the one with which register’s field value is updated.
  2. When property used in the way “signal_name:reg_name”, then the width of the fields and number of fields of the registers should be same for both the registers i.e.  the register on which property is applied and the one with which register’s field value is updated.


 Behavior change in RTL:


In case of register’s field accesses “sw=r ; hw=w ;” along with reset value, hardware write enable configuration and software access properties mentioned in the Table - 14 mentioned in the SystemRDL 2.0 LRM. 

  • This case is for RDL fields having
    • sw=r and hw=w
    • no reset value
    • without hw write enable property (we/wel) or hw write enable (we/wel=false)


No flop (Storage Element) is created. Hw write value is assigned to the bus without being flopped. 

Generated RTL

wire *_q ;

assign *_q = *_in ;

assign *_rd_data = *_rd_valid ? { *_q } : 32’b0 ;

Incompatibilities : A warning is thrown by the compiler indicating change in RTL  behavior. This warning can be suppressed using the command line switch “-no_warn”.


Warning : There is a change in behavior for register with sw access = r and hw access = w to comply with SystemRDL 2.0 Standard. Earlier IDS created a flop now there will be a wire/bus - hardware assignment.


This case is for RDL fields having

  • sw=r and hw=w
  • With reset value
  • Without hw write enable (we/wel) or hw write enable (we/wel=false)

Flop (Storage Element) will be generated without “*_in_enb” signal.

Generated RTL

reg *_q ;

always @(posedge clk)

                if (!reset_l)

                          *_q <= reset ;

                else

                                *_q <= *_in ;

Incompatibilities : A warning is thrown by the compiler indicating change in RTL  behavior. This warning can be suppressed using the command line switch “-no_warn”.

Warning : There is a change in behavior for register with sw access = r and hw access = w to comply with SystemRDL 2.0 Standard. Earlier IDS generated *_in_enb” hardware signal when no hw write enable exists now it is removed intentionally.


This case is for RDL fields having

  • sw=r and hw=w
  • With/Without reset value (in case of no reset value, “x” value is default as “*_q <= ‘hx”)
  • With hw write enable (we/wel)

Flop (Storage Element) will be generated with “*_in_enb” signal.

Generated RTL

input *_in_enb ;

reg *_q ;

always @(posedge clk)

                if (!reset_l)

                                 *_q <= reset ;

                else

                                             if(*_in_enb)

                                                *_q <= *_in ;

This case is for RDL fields having

  • sw=r and hw=w, with onread=rset/rclr (sw access modifier mentioned in Table - 14 in SystemRDL 2.0 LRM)
  • with/without reset value and
  • with/without hw write enable (we/wel)

In this case, the generated RTL will be in effect with the sw access modifiers.
