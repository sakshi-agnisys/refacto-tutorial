Multiple Interfaces of Memory

Introduction


In IDS, there are two methods to generate multiple interfaces for external memory:

    1. By using property “repeat” on memory template
    2. By using “memory=rtl/uvm/all” with “repeat” on external reggroup


If user has an external section with property “memory=uvm” , "memory=rtl" or ''memory=all'' on it, then the external section will become memory. It will create multiple instances based on the value of repeat. The number of registers in section will become the depth of memory and size of register will become the size of the memory.


Moreover, following features are provided for memories in IDS currently:

    1. Parameterized memory
    2. Repeat on memory

Features of memories:


1. Parameterized memory


Memories can also be parameterised in IDS from v7.4.0.0 onwards by passing parameters either in "mementries" (if SystemRDL is used as input) or "depth" of memory template (if IDS-Word or IDS-Excel is used as input).


Case 1:


SystemRDL 


addrmap blocktop #(longint unsigned $depth=8)  {

    name = "Block1";


       mem Mem1 { 

                  mementries = $depth;

                  memwidth = 32;

               };


              Mem1 Mem1 @0x4;


    };

sw_access


This property specifies the software side port access type.


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View



IDS-NG Spreadsheet View



SystemRDL


property sw_access {type = string; component =mem|reg; };  


addrmap block1 {


 mem memory1{

  mementries = 4;

  memwidth = 32;

  sw_access =   "ro";

};

  memory1 memory1 @0x00;


};
