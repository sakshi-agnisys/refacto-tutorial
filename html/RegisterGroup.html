RegisterGroup

Training Video: Getting Started


A RegGroup is a set of registers. A Register or even a RegGroup can reside inside a RegGroup, but a RegGroup cannot be left empty. The RegGroup can be used to construct any arbitrary multidimensional register set.

Specification Example




The IDS RegGroup template consists of many property fields as shown above. The explanation of these field properties is given below:



Property

Definition

name

A unique string that identifies the name of the register group.


  Example : Design_group

offset

The address of the register group in the address map is with respect to the container's address. If the user defines the offset then value in the offset is the displacement added to the memory location of the container (for reggroup container is block). The value can be decimal value or a hexadecimal value.


Example : 10 or 0x10

external

The possible values of this property field can "true" or "false". The default value of this field can be "false". If this property is set to "true" then an RTL implementation is not created by the IDS, instead appropriate ports are created so that the user can provide their own implementation to that register.


Example : true or false 

repeat

  This represents the number of occurrences of the register in the address map.


  Example : 2 or 0xa

size

  This size property represents the size of the reg group per iteration in address units.

   



Note : Some output transformation may not support reg group inside reg group, so reg group must have at least one register.
