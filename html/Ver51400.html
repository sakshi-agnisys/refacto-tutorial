Ver 5.14.0.0

v5.14.0.0

Jan 12th 2015 

IDS Enhancements:


  • Verilog RTL enhancements 
    • Signals (Advanced): A new feature called "Signals" can now create named external ports, in the block or chip level, and connect certain internal design signals to the external world. For more details click here.
    • Clock Domain Crossing (Professional): IDesignSpec™ automatically inserts clock domain crossing circuitry at interface points. Click here for more details.


  • VHDL(alt1) RTL enhancements (Professional)
    • Multiple Bus Domain in VHDL : Multiple bus domains are now supported in VHDL(alt1) RTL output. Click here for more details.
    • Halt Enable and Halt Mask registers in VHDL : Now supported in VHDL same as Verilog. Click here for more details.
    • Counters in VHDL(alt1) : Counters now supported in VHDL(alt1) RTL same as Verilog. For more details click here. 


  • New RTL Properties (Professional)
    • undefined_rd_val : To set the undefined(reserved) bits of the register. Click here for details.
    • external_intf_stages : To remove the flops from the external output signal in VHDL(alt1). Click here for details.


  • New Outputs (Advanced)
    • System C RTL output (Beta Release) : A new "System C" RTL synthesizable output is supported in IDesignSpec™. Click here for more details.
    • CMSIS-SVD Output : The CMSIS System View Description format(CMSIS-SVD) formalizes the description of the system contained in ARM Cortex-M processor-based microcontrollers, in particular, the memory mapped registers of peripherals. This new Industrial standard "CMSIS-SVD" output is supported in IDesignSpec™.  For more details click here.
    • Perl Data Dump : All the design information can now be dumped in the form of PERL hashes. For more details click here.
    • Python Data Dump : A new design specification dump is now available in form of Python dictionaries. For more details click here.


  • Multiple Files Generation for RTL (Professional)

IDS can now generate individual files for each module/package/entity/architecture for RTL output. Click here for more details.


  • New Imports (Professional)
    • Import RALF Language :IDS  now supports import of the Synopsys-RALF file and generate the IDS register spec. For more details click here. 


  • Excel and CSV files can have Refs for IDS-Batch (Professional)

Referring various components in CSV/Excel file is now possible in IDS-Batch. For more details click here.


  • IDS Turbo Generation (Professional)

A faster way to generate outputs from Word has now been added. Click here for details.


  • New Editor License Mechanism

IDesignSpec™ now comes with a new license mechanism through which user can generate IDS-WordEditor and IDS-ExcelEditor License. For more details click here

Automatic Register Verification (ARV™):

(Version 1.0) (Available as a separate module)


Automatic Register Verification (ARV™) is a module that enables users to create all the files required for complete UVM based verification of registers in a hardware device. 


With IDesignSpec™ users create executable specification and generate a variety of outputs. These files could include UVM and RTL models. Without ARV™:

  • These models would still have to be stitched together in a UVM verification environment.
  • Bus agents, interfaces and proper sequences need to be created and everything needs to be manually connected. 
  • Next, the simulations need to be run and a test plan created and the results annotated onto the plan. 


All these tasks take up valuable resources and can be automated. This task is done by ARV™. For more details click here.
