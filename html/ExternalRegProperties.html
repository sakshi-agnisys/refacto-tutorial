External Reg Properties

external_ack 


When "external_ack" property is used, the "wr_req_in"  port behaves as "wr_ack_in". All the "wr_req_in" (which are now "wr_ack_in") will be low until external has acknowledged the write transaction. Module external_ack gets instantiated with every external register.This module extends the external register's wr_valid signals till the "wr_ack_in" is received which is further fed into the "request" signal. Now the "request" will initially be high which will allow the "address" and "wr_stb" signals to pass into the register block. And then request will wait for "wr_ack_in" to complete the transaction and also prevent the next transaction till current transaction has been completed.


This property is applicable on top(block/chip).


NOTE: User needs 'external_ack.v' file since it is instantiated in the top RTL. It can be found in supporting files(IDS docs).

Enhancement(Supported from v6.38.0.0)

In the former implementation(i.e. external_ack=true) explained above, for every external section/register group/reg, external_ack is instantiated which increases the area. Therefore, to decrease the area, external_ack module is instantiated only once for a module.

Figure: external_ack instantiated once

Property ext_decode_lines(Supported from v6.30.0.0)


Property "ext_decode_lines" gets the decode address at which read/write operation is performed. Possible value of property = 'true". This property is applied with "repeat" property on external elements like external reg, memory, section (except block) with external=” true”.


When this property is applied on external elements, it generates an output type port as: <external_elem_name>_out_decode_addr.


Note: It depends on decode, when decode is high then it will also be high. In AXI bus, two output ports are generated given as: 

<external_elem_name>_out_decode_addr.

<external_elem_name>_out_decode_raddr.


Examples: IDS-NG         IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View:


               


IDS-NG Spreadsheet View:




SystemRDL


property ext_decode_lines {type =boolean; component = addrmap|regfile|reg|mem;};

addrmap Block1 {

       name  = "Block1 Address Map";

       reg Reg1 { 

               ext_decode_lines = true ;

               regwidth = 32;

               field {

                       hw = rw;

                       sw = rw;

                       onread=r;

                       onwrite=w;

               } Fld[31:0] = 32'h01;

       };

         external Reg1 Reg1[4] @0x10;

};

Property external_intf_stages


Property "external_intf_stages" works only for  output VHDL-ALT1 to remove the flops from the external output signals. This property works with external chip or block components (“external=true” is applied). 


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL

IDS-NG Spreadsheet View:
