Read-Write pair Register

RWpair registers


A register containing write-only-fields (WO) sharing the same address with another register containing read-only-fields (RO, RC, RS). The fields in each registers are unrelated.


Logical view

Templates and Property


ROWO pair registers are specified in IDS by the property "rwpair" in 2 ways. 


Following rules apply for the modeling of RO and WO registers sharing the same address:-


  • Supported SW access of ROWO pair at same address is {WO, RO, RC, RS}.
  • The software access of the registers in the ROWO pair should be mutually exclusive. This implies if the register on which the ‚Äúrwpair‚Äù property is applied is readable (RO, RC, RS), then the second value of the property should be any writable SW access (WO).
  • If this property is applied on a register, then all of its fields should have the SW accesses of same type, i.e either write-only(wo) or read-only(ro,rc,rs).

1. Using single IDS template


The template for one register in the ROWO pair is created and the specification of the other register in the pair is captured by the property ‚Äúrwpair‚Äù. This property has two values, first is the name of one of the registers in the ROWO pair and the second value is the register's SW access.

2. Using separate IDS Templates for RO and WO register pair

(Compatible with IDS version 5.16.8 and above)


User can specify separate IDS template for both registers and pair these using "rwpair" property. The field definition for both registers may be different but SW access should be mutually exclusive.  

Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL

RWpair fields


RW-pair fields are the read only fields which share the same offset with the write only from the software side. So at one offset there are two fields in a register, one is WO and one is RO from SW side.



Usage of Property


  1. Property can be applied on the field with ‚ÄúRO‚Äù access with the name of the field.

 

reg regA { 

reg width = 32;

field {

         rwpair = "FldB" ;

         hw = w;

         sw = r;

} FldA[7:0] = 1'h0;

};


Field ‚ÄúFldB‚Äù will automatically be created with ‚ÄúWrite-only‚Äù access in RTL.


  1. Property can be applied on fields with ‚ÄúRO‚Äù access and provided with the name of the other field present in the same register with ‚ÄúWO‚Äù access.


reg regA { 

reg width = 32;

field {

         rwpair = "FldB" ;

         hw = w;

         sw = r;  

} FldA[7:0] = 1'h0; 

field {

         hw = r;

         sw = w;

} FldB[7:0] = 8'h0;

};

‚Äúrwpair‚Äù for repeated register


IDS allows users to have two registers in the same address inside a register address map with mutually exclusive access for read and write separately depending on sw access defining of the two registers.  So different IDS templates can be used for both registers, and they can be paired using the "rwpair" property.

Now support of ‚Äòrwpair‚Äô flow for internal registers with repeat is introduced where both the registers of the read-write pair have equal repeat value on either them or their corresponding ancestors.
