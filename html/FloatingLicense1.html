Floating License

Agnisys® License Manager (ALM)


This guide describes how to install and use the Agnisys® License Manager (ALM) server. 


ALM is a web based license server that enables you to use floating licenses with Agnisys® tools. The need to create a new license manager arose due to the following reasons: 

  • Existing license servers were not capable of providing the special functionality needed by the IDS-NG. 
  • Existing license servers did not use the widely available modern web based server. 
  • Existing license servers were expensive, and we did not want to pass the additional costs to our valued customers. 


The ALM server (version 1.2 and later) support Floating licenses only. The node locked licenses and site licenses are still served by the old file based licenses. 

Requirements


ALM has a very small footprint and can run on any Windows or Linux machine. The minimum requirements to install ALM server are as follows: 

1. A minimum of 250 Megabytes of disk space. 

2. Active and stable LAN connectivity. 

3. MachineIP: 2464 is open, and not occupied by any other application - so that ALM server can easily access this port. 

4. Administrator privileges.

Server Download and Setup


1) Download the ALM zip file from http://www.agnisys.com/release/alm/ALM_2.7.1.0.zip 

2) Unblock the file:  

a. Right-click the downloaded zip file, select properties, and click Unblock (This step is mandatory for ALM service run under Windows). 

b. Unzip the file. For more details on installing the server, please refer to the Server setup section.

Server Setup


The server is identified by a Universally Unique Identifier (UUID) that is generated by getUUID.sh/getUUID.bat. To setup the server: 

    1. Run the UUID generator. A uuidfile.txt file is generated.  
    2. Send an email to support@agnisys.com to receive a license file (license.dat). 
    3. Save the license.dat file at the root. Ensure that the license file is in the same directory as the readme.txt file at the ALM_x.x.x.x directory. 
    4. Run the server.  
      • Run server without verbose using startup.sh / startup.bat 
      • Run server with verbose using startup.sh –v. For Windows, set the environment variable AGNI_LICENSE_DEBUG=1,  DOS> startup.bat (To run ALM server as a Window service, login with administrator privileges, and run InstallWindowsService.bat.) For more details, please refer to the readme.txt.


To setup the client:

1. Set environment variable AGNI_LICENSE_SERVER or AGNI_LICENSE_FILE to the path where the server is running, for example 192.168.2.100:2464.

2. Run IDS-NG.

Server Status 


To check the server status, you can either: 

1. Visit HTTPS://192.168.2.100:2464, and login using the default user name and password: admin/admin. 

2. Or, click Status in the IDesignSpec™ ribbon. A Security Alert window pops up. Click Yes to proceed. A License Dashboard window appears.  


You can check the details of all the users using the license from the license server, and the product information of all the products encapsulated in the licenses.
