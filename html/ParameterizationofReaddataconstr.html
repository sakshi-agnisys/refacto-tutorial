Parameterization of Read data construct

In IDesignSpec™, If the value of parameter is changed to some other value different from the default value, then user may get compile warnings like the following:

Warning-[SIOB] Select index out of bounds


Due to:- 

assign Reg_rd_data_wire = Reg_rd_data[0][0]  |  Reg_rd_data[0][1]  | Reg_rd_data[0][2]...........


assign rd_data = Reg_rd_data_wire ;


To counter this a new IDS property “dynamic_empty_address=true” has been introduced using which the handling of parameters has been done. This  property enables the user to override the value of repeat during run time without any backward Incompatibility. This property is applicable both internal and external CSR.


When a user overrides the value of repeat with a value greater than the initial value then using a static check, a runtime error will get displayed.

When overriding the repeat value with value less than initial value of parameterized repeat then there will handle empty address.
