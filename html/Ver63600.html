Ver 6.36.0.0

v6.36.0.0 

(April 30th,2020)


IDesignSpec™ (IDS)


RTL Enhancements:

  • 3-way aliasing is supported to access a register from 2 different locations i.e. one from within the block and other from another block (More Details) 
  • ‘RO’ access is supported along with ‘RW’ access for alias register in another block (More Details)  
  • Repeat is supported when alias and aliased properties are applied (More Details) 
  • “field_prot=priv” property is introduced for field protection to support protection for the field instead of protection of whole register (More Details) 
  • i2c_widget has been enhanced with target address using parameter ‘sda_in’ and ‘sda_out’ as input and output respectively.
  • counter.decr.thld and counter.incr.thld properties have been enhanced to accept field and signal values as well, apart from an integer value (More Details)
  • With the help of property ‘rtl_hw_vector’ vectored hardware port can be created for any register (More Details) 
  • Support for Tile-Link 1.8 bus by default and Tile-Link 1.7 bus with switch '-tile_version=1.7' for backward compatibility in verilog along with '-fast' switch (More Details)


SV Enhancements:

  • Structures have been enhanced to support repeated internal registers and repeated/non-repeated external registers (More Details)
  • “sv_guard_band” property is introduced for generating the guard band in block interface file and the wrapper file (More Details) 


UVM Enhancement:

  • Enhancement done for support of signals on Lock Registers (More Details)
  • Enhancement done for multiple alias support within different blocks (More Details) 
  • "coverage.option_name=0"  property is introduced for disabling the default feature of option.name in case of repeated block, section and registers.
  • ARV™ is enhanced for supporting VHDL now (More Details) 
  • “type = nomem” property is introduced for making external regfile containing single register to be extended with uvm_reg_file and uvm_reg respectively, instead of the external regfile containing a single register, to be treated as a memory and the class to be extended from uvm_mem (More Details)
  • Enhancement done for support of repeat on Alias Registers (More Details)
  • "uvm.name_format=%l" property is enhanced in case of ref to change the naming mechanism of classes  (More Details)
  • Enhancement done for support of repeat on lock registers (More Details)


SystemRDL Enhancements:

  • "rdl_define": This property is used to pass the value of preprocessor macros, when SystemRDL file is referred. (More Details)
  • -rdl_perl_debug command line switch is introduced to dump RDL into .IDS directory (More Details)


General Enhancements:

  • Support for description in TOC in HTML-Alt2 outputs when multiout option is selected. (More Details)
  • Support for NodeLock FlexLM license for IDS-Batch CLI, IDS-Word, IDS-Excel and IDS-NG (More Details)
  • -include_css command line option is supported for ignoring the generated external css and supporting images for HTML outputs. (More Details) 
  • Support for property’s functional description in PDF output (More Details)
  • Support for enhancement of TOC to show repeated components in PDF outputs (More Details)
  • Support for Japanese characters in HTMLalt2 output (More Details)
  • asp-perl: user can use environment variables in rdl embedded perl.
  • -dir_out_specific command line switch is enhanced for directing output in specific output directory in case of SV output as well, e.g., -dir_out_specific “sv:abc” (More Details)
  • Support for AI based sequence detection (More Details) 
  • ARV™ made independent in IDS-Batch
  • "csharp_name.format={%c%b%s%r%f}" property supported in csharp output in IDS-Word.


Bug fixes:

  • Fixes done for <range> tag in IPXACT for repeated section/registerFile. Earlier, it's value was the entire size of the repeated registerFile but now it will indicate the range of 1 iteration of the registerFile as per the IPXACT standard.
  • Fixes done for Line Return in PDF output.
  • Fixes done for address calculation in case of stride for HTML output
  • Fixes done for variant support in SystemRDL1.0
  • Fixes done for adding extra column in existing register in IDS-Word for all outputs
  • Fixes done in RDL 2.0 in case of signal and only one addrmap
  • Fixes done for block file hyper link issues for HTML-Alt2 outputs in case of multiout
  • Fixes done for memwidth value calculation in case the value is not in the power of 2 in SystemRDL2.0 
  • Fixes done for encoding issue in HTML outputs
  • Fixes done for signals on top (and in case of chip as well) for RDL input 
  • Fixes done for formatting issue in HTML and PDF outputs
  • Fixes done for asp-perl hanging issue in case of big SystemRDL test case
  • Fixes done for ‘counter.signal’ property
  • Fixes done in sv when there is external section inside internal section (with
  • repeat)
  • Fixes done in ‘assign’ property when repeat is present on a register.
  • Fixes done in hdl_path in case of ref from other files
  • Fixes done in case of ‘external_ack=true’, with repeat on an external element
  • Fixes done for ‘module_name’ with ‘external_ack’
  • Fixes done for property ‘sharedextbus’ in case of same block referred multiple times in top


IDesignSpec™ (ISS)

 

Enhancements:

  • Command 'switch' is now changed to 'switch_intf'. (More Details)
  • Structures can now be randomized while creating sequences in Firmware, UVM and System Verilog output. (More Details)
  • Users can now use vectored variable in the sequence specification and also part-select while writing steps of sequences in UVM, Firmware and System Verilog output. (More Details)
  • Randomization, constraints and assert command is now supported in System Verilog output.
    • For more details on randomization and constraint, click here
    • For more details on assert command, click here


Bug Fixes:

  • Fixes done for address calculation in case of repeated section and blocks in Firmware and System Verilog output.
  • Repeated registers can now be used for Platform output.
  • Fixes done for passing arrays as argument in System Verilog output.
  • Fixes done for usage of parameters, defines and enums in Firmware output.
  • Fixes done for description not working for variables, constants, 'switch_intf' command and subsequence call in UVM and Firmware output.
  • Fixes done for assert, return and switch command for Python as input for sequence specification creation.

 


Specta-AV™ 


Enhancements:

  • ‘cross’ property is introduced for specifying cross coverage between two or more coverpoints or variables. (More Details)
  • Enhancement for creating cover points for LUT’s values which helps to ascertain that all possible values are covered in the verification run. (More Details)


IDS-NG


Enhancements:

  • Reviving of spreadsheet view done in IDSNG. (More Details) 
  • Support for both editor and generator base licensing. (More Details) 
  • -include_css command line switch is introduced for HTML and HTMLalt2 output for including the CSS files generated by IDS wherever required. (More Details) 
  • Support for hexadecimal format as well as decimal format in dynamic names. (More Details)


Bug Fixes:

  • Fixes done for the issue with “undo” in the param view without impacting the check boxes in param diff section.
  • Fixes done for the issue of not able to select multiple fields within a row for copy, cut or paste.
  • Fixes done for the issue of missing field descriptions in TCL generated outputs.
  • Fixes done for the issue of missing font colors of fields and descriptions in html output.
  • Fixes done for the issue of not able to copy or paste single/multiple row(s) in param view.
  • Fixes done for the issue of getting an error, if any formatings like color/font size/font style are added in the register view.
  • Fixes done for  the issue of generated html files being dependant on other css file , which limits its portability.
