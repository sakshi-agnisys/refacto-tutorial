Default Nettype


The directive `default_nettype controls the net type created for implicit net declarations. It can be used only outside of module definitions. Multiple `default_nettype directives are allowed. The latest occurrence of this directive in the source, controls the type of nets that will be implicitly declared. It contains the syntax of the directive. When no `default_nettype directive is present or if the `resetall directive is specified, implicit nets are of type wire. When the `default_nettype is set to none, all nets shall be explicitly declared. If a net is not explicitly declared, an error is generated. This is used by the RTL designer so as to avoid implicit nets and catch errors upfront in the design cycle.

SystemRDL:


property default_nettype {type=boolean; component = addrmap;};

 

addrmap nettype {

  default_nettype = none;


  reg reg1 { 

     regwidth = 32;

      field {

        hw = rw;

        sw = rw;

        onread=r;

        onwrite=w;

    } f1[31:0] = 32'h0;

  };

  reg1 reg1 @0x0;

};
