Ver 6.18.x.x

v6.18.0.0

(April 8th,2019)


IDesignSpec™

RTL Enhancements:


  • counter.both.enb: This property is supported for generating individual enable for both increment and decrement counter.(More Details)
  • Command line switches "-rtl_no_error" and "-no_external_mask" have been added, for turning on and off the ports IP_ERROR and IP_BE_MASK respectively and the dependency on "-no_lint_warn" switch has been removed for the third party block-level RTL.(More Details)



UVM Enhancements:


  • Indirect register is now supported in case register exists inside section.(More Details)
  • ignore_prop: This property is now supported in case of virtual register for UVM only.(More Details)
  • hdl_path_internal: This property has been enhanced to work hierarchically. 



Header Enhancements:


  • vertical_reuse: This property is now supported in Cheader output to avoid regeneration of the code.(More Details)
  • cheader_filler: This property is used to remove fillers from the Cheader output generated due to the presence of empty spaces.These fillers were causing too much memory allocation.(More Details)
  • cheader.enum.name_format: This property is used to modify enum name format in Cheader output.(More Details)
  • CHeader output has been enhanced with a macro indicating the repeat count value of the component and size of the first instance of that component which contains the repeat property.(More Details)
  • svh_filler: This property is used to remove fillers from the SVheader output generated due to the presence of empty spaces. (More Details)
  • IDS SV header output is now enhanced to support big endian and little endian structs with IDS Batch command line switch ‘-little_endian_svheader’ and ‘-big_endian_svheader’. For generating SV header with big endian and little endian structs, user need to add -little_endian_svheader -big_endian_svheader switches to our existing IDS-Batch command. 



General Enhancements:

  • Enhancement in IDS-Batch:
    • IDS_CACHE_DIR = <user specified directory>: This environment variable is set to change cache directory while referring in IDS-Excel(.xlsx , .xls, .csv or .ods) input format. All intermediate XRSL files will be created in this directory.
    • -in_third_party m: This switch is used for supporting third party CSV input format.(More Details)
    • -vertical_reuse: This command line option is supported to create separate interface for all blocks and subblocks in UVM.(More Details)
  • Enhancement in IDS-Word and IDS-Excel:
    • A new feature has been added in the configuration window of the IDS-Word and IDS-Excel for passing command line options for IDS-Batch CLI.(More Details)
    • TCL interpreter has been enhanced with get_param_objects option which can be used to change parameter name at the time of output generation. (More Details)
  • Htlmlalt2 output has been enhanced with a search bar box, user can now search various IDS elements and the description using it.
  • Enhancement has been done in the IDS htmlalt2 output to support commands and status structures in external section with IDS property {memory=html}.(More Details)
  • -fast_html: This command line option has been supported to generate htmlatl2 and html outputs faster.(More Details)




Bug fixes:


  • Cheader parameter issue related to ‘h has been fixed.
  • Empty address issue has been fixed in RTL in case of deep hierarchy. 
  • Bug fix in SVHeader output in case of repeat on section.
  • Bug fix in case of regalign property.
  • Bug fix for hdlpath property when register is used with repeat property in case of third party output and deep hierarchy.
  • UVM output fix in case of parameters usage for deep hierarchy. 
  • auto_volatile=true property has been supported without using -fast option.
  • uvm.map property has been fixed in case of chip and block.
  • Replication of enum issue at board level when chip is referred more than once has been fixed in Cheader output.
  • Removed fillers in case of subblock inside subblock for Cheader output.
  • Signals have been fixed in case of no assignment in Verilog output.
  • lock.clear, lock.set, and lock.toggle properties have been fixed to depend upon the correct signal/field now instead of write valid.
  • Command line option -rtl_no_error has been fixed to remove read, write, and empty address errors in Verilog output.
  • Fixed hard_resert with asynchronous resetsignal in Verilog output.
  • Counter support with reset signal has been fixed.
  • Fixed the issue related to read hangs in case of back to back read from an external target for APB bus for RTL output. 
  • Defines for repeated registers in both Block and Board level for Cheader output.
  • "Hex issue (Verilog syntax instead of c syntax)" in both Block and Board level C headers output.
  • Fixed the issue related to IDS-Batch command line switch '-(big|little)_endian (true|false)' property in case of UVM Output.



Automatic Register Verification (ARV™)


ARV™-SIM Enhancements:


  • SystemVerilog is now supported.
  • Alias register between different blocks are now supported.
  • Wide registers are supported for all buses.
  • Configurable Randomized Sequences can be generated with Velocity Template Feature now.



ARV™-Formal Enhancements:


  • Wide registers are supported for APB bus.(More Details)



IDesignSpec™


Enhancements:

  • Enums, defines and params are now supported for UVM output. More Details 
  • Randomization supported on variables for UVM output by using ‘rand()’ keyword. More Details
  • $time functionality is now supported in UVM output by using ‘time()’ keyword. More Details
  • All constants would be declared as ‘static’ in Firmware output.
  • Sequences can now return values for UVM output. More Details
  • {uvm_door=peekpoke}: By using this property, peek and poke functionality can be used in sequences. More Details
  • Two new outputss have been added in the ISquenceSpec:
          • InFact , More Details
          • TCL , More Details 
  • Arrays, while loop and nesting of loops are supported in the CSV output.
  • Directory structure of Firmware output is altered, four files will now be generated as: <seq_name>.c , <seq_name>.h , <register_model>.h  and <seq_pkg>.h files.
  • ‘uvm.frontdoor’ and ‘uvm.backdoor’ will now be used as ‘uvm_door=front’ and ‘uvm_door=back’ respectively. 
  • Support for hierarchical sequences in Firmware output.


Bug Fixes:

  • External function called in ISS will be declared as extern rather than virtual.
  • Bug fixed for multiple declaration of ‘status’ in sequences’ class.
  • Bug fixed for the path while setting and writing into the register when ‘uvm.regmodel’ property is used.
  • Bug fixed for passing complex expression as value in variables and while writing registers or fields.
  • Bug fixed for descriptions of variable and registers in Firmware output.
  • Bug fixed for missing ‘lvar’ variable in UVM code when expression of variables is passed to the variable.
  • Syntax of display command fixed in UVM.
  • Bug fixes done for usage of for loop in Matlab output.
  • Bug fixes done for ‘base_address=true” property in Firmware output.
  • Annotation check on commands that are not part of ISS. 
  • While doing consolidation read in Firmware, bug fixed for redundant brackets.
  • Bug fixes for variable with same name as field name, in case of UVM.
  • Bug fixed for assignment of an instance of register (with repeat) to the variable.
  • Bug fixed for writing an array of register with variable declared as an array.
  • Bug fixes for registers with same name inside different sections for UVM output.
