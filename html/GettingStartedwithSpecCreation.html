Getting Started with Spec Creation

AgnisysÂ® promotes specification driven development. This approach brings ease to a document editor, system architect and designers to create an executable specification. It maintains the synchronization among different teams because changes made in the specification are automatically permeated to all the downstream views. This executable specification contains everything needed to fully describe the design including documentation and register descriptions.


While creating a specification, one of the following elements can be used:


  • Chip : It is a collection of multiple blocks. 
  • Block : It is a collection of multiple registers, register groups, memory etc. 
  • Registers : They are the basic building block of the register space. Each bit or set of bits within the register controls some behavior of the circuit. 
  • External registers : It is a register implemented by the user outside the generated RTL created by IDS. 
  • Wide registers : In specification, a register with width greater than the bus width is a wide register
  • Unregistered fields : Often Designers require hardware status bits to be accessed by Software without being registered inside the IDS generated block. This can also be achieved in IDS. 
  • Register groups : A RegGroup is a set of registers. A Register or even a RegGroup can reside inside a RegGroup, but a RegGroup cannot be left empty. The RegGroup can be used to construct any arbitrary multidimensional register set. 
  • External RegGroups : Just like registers, register groups can also be declared as external to the register map. 
  • Register arrays : Multidimensional arrays can be created in IDS using RegGroups inside another RegGroups. 
  • Memory : A memory is a storage with contiguous address locations.
  • Enums : The Enumerated type is a data type consisting of a set of named values called elements, members or enumerators of the type.
  • Defines : In IDS a define is used to name arb
  • itrary integer values. Any number of identifiers are used to name arbitrary integer values.
  • Ref template : To reuse the already specified specification in IDS.
  • Hierarchical designs using variants.
