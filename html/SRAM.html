SRAM


Register bits on a chip can be categorized into following two classes:

1)      Static algorithm specific hyperparameters

2)      Dynamic host centric control, status and debug registers

Dynamic registers need HW/SW accessibility all the time and best suited for FF based register implementation. They amount to a small section of registers (<5%) in relation to entire register bits available on the chip.

Static algorithm specific registers are always programmed by SW at boot or in controlled environments, before HW is allowed to consume the data, avoiding simultaneous SW writes and HW reads. They amount to large portions of the total register bits available on the chip.

A typical static register block will have multiple sections, with register bits in each section being an exact replica of other sections as shown in figure below (1.1):

Fig. 1.1 shows a static register block on AHB bus having 512 repeated sections having 32 registers each. 

Fig 1.2 shows the HW interface of the 2 port SRAM with the hw read logic. On read the field data for the required SRAM address will be available on the corresponding field ports.

Register bits (registers) in a section represent a static configuration of the chip at any instance in time. These static register bits are not time sensitive and can be available after a defined (predictable) number of clock cycles for HW reads. HW needs visibility to only one section of register bits at any particular instance in time represented by section address as shown in figure above (1.2)

Register bits implementation in IDS:

-   Internal registers: These are implemented as FF for each register bit. Dynamic registers, which are few, can be implemented as internal registers.

-   External registers: Structure for these is defined, but not implemented as FFs, and access ports are provided for use as per application needs. Static registers, which are large in number, can be marked external and implemented as 2 port SRAM with new IDS property “mapto_dpsram=true”. So a 512 * 32 SRAM can be conceptualized as a repeated external section having 32 registers (each 32-bit wide)  and a repeat count of 512. This way by mapping Static registers to external 2-port SRAM 40% of the chip-area can be saved. 

The SW interface of this SRAM which is both readable and writable is connected to IDS external IDS interface,while the HW interface which is only readable will have its application logic (required to read the value from the SRAM) outside of IDS scope while the buffer registers on this end will be defined inside the IDS block. Both Internal and External registers occupy the same address space.

SystemRDL


property memory { type = string; component = regfile; };

property mapto_dpsram { type = boolean; component = regfile; };

     

addrmap block1 {

  name  = "block1 Address Map";


  reg dynamic_reg1 { 

     regwidth = 32;

    field {

      hw = r;

      sw = rw;

      onread=r;

      onwrite=w;

    } f1[31:0] = 32'h01F;


  };

 

  regfile sram {

    name  = "sram Address Map";

    memory = "rtl" ;

    mapto_dpsram=true;

   


    reg mreg1 { 

       regwidth = 32;

      field {

        hw = rw;

        sw = rw;

        onread=r;

      onwrite=w;

      } mreg1f1[31:16] = 16'h0;

      field {

        hw = rw;

        sw = rw;

        onread=r;

      onwrite=w;

      } mreg1f2[15:0] = 16'h0;


    };

 

    mreg1 mreg1 @0x00;


  };

  dynamic_reg1 dynamic_reg1 @0x00;


  external sram sram[512] @0x04;


};

module blockname_IDS(

    

    

    // SECTION(EXTERNAL) : SRAM1 PORT SIGNAL

    sram1_rd_ack_in,

    sram1_rd_data_in,

    sram1_rd_valid_out,

    sram1_wr_req_in,

    sram1_wr_valid_out,

   

    address_out,

    wr_data_out,

    

   

    //SRAM HW RO SIDE

    //One set of fields

    mreg1_f1_r,

    mreg1_f2_r,


   // hw side signals

    hw_clock,

    hw_rstn,

    hw_rd_data_in,

    address_hw,

        

       .   .    .    .    .

       .   .    .    .    .

       .   .    .    .    .


    output  [ addr_width-1:0] address_out;

   output  [ bus_width-1:0] wr_data_out;

    

    

    //SRAM HW RO SIDE

    //One set of fields

    output [15:0] mreg1_f1_r;

    output [31:16] mreg1_f2_r;

    

    input  hw_clock;

    input  hw_rstn;

    input [bus_width-1:0]  hw_rd_data_in;

    input [addr_width-1:0]  address_hw;

     

        .    .    .    .   .

        .    .    .    .   .

        .    .    .    .   .

        .    .    .    .   .

        .    .    .    .   .

     

    assign rd_data = int_rd_data | ext_rd_data;

    

    assign error = 1'b0;


    // SRAM HW SIDE BUFFER LOGIC

    

    localparam NUM_REGS_PER_MEM_INST = 512;

    localparam NUM_REGS_PER_MEM_INST_LOG2 = 9;

    

    wire [NUM_REGS_PER_MEM_INST_LOG2-1:0] sel_REG_hw_Port;

    reg [bus_width-1:0] address_hw_delay;

    assign sel_REG_hw_Port = address_hw_delay % NUM_REGS_PER_MEM_INST;

    

    //All fields and registers in a external section as defined in   specification (memory structure)

    reg  [bus_width-1 : 0]  Mreg1_q;

    assign mreg1_f1_r = Mreg1_q[15:0];

    assign mreg1_f2_r = Mreg1_q[31:16];

    always @(posedge hw_clock or negedge hw_rstn)

        begin

        if(!hw_rstn)

            address_hw_delay <= 'h0;

        else

            address_hw_delay <= address_hw;

        end

        

        always @(posedge hw_clock)

            begin

            if (!hw_rstn)

                Mreg1_q <= 'h0;

            else if (sel_REG_hw_Port == 'h0)

                Mreg1 <= hw_rd_data_in;

            else

                Mreg1_q <= Mreg1_q;

            end

  endmodule   


Limitation:

  1. There can be only one SRAM structure per block.
  2. The property “mapto_dpsram” can only be applied on an external section having a repeat of 1 or more than 1.
