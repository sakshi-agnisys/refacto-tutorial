Paged Register

Paged Register

(Compatible with IDS Version 5.18.0 and above for SystemC RTL)


Paged register is set of registers sharing same physical address and another selector register is used to decide which out of them is actually accessed.



Following usage of page register is covered in the following section:

  • Paged register in IDesignSpec™ (RTL point of view), refer here
  • Paged register in UVM-RAL, refer here
  • Paged register in ARV™, refer here
  • Separate read and write select for page register, refer here
  • Passing parameters in page_count, refer here
  • Separate page select for both hardware and software, refer here
  • Paged register across blocks, refer here


Paged Register in IDesignSpec™

SystemRDL


property page_select { type = string; component = reg;};

property page_count { type = number; component = reg;};

addrmap SpecialReg{

     reg Page{

       page_count = 16;

       page_select = "select.fld[3:0]";

       field{

         hw=rw;

          sw=rw;

       } fld[31:0]= 32'h0;

     };

     reg select{

       field{

         hw=rw;

          sw=rw;

       } fld[31:0]= 32'h0;

     };

 Page Page ;

 select select ;

};

Paged Register in UVM RAL


In UVM-RAL model we have to add the page register and its different context based on select register on different address map. In the above example we have two 32 bit registers, if page register is specified at 0x0 location, than  Page[0] is mapped at 0x0 and all other context registers i.e Page[1], Page[2], Page[3] ..., Page[15] are outside the address space range specify in RTL.  

For verification point of view we have added the task called pagedWrite for verifying the page register based on the value in select register.


class SpecialReg_block extends uvm_reg_block;

    `uvm_object_utils(SpecialReg_block)

   

    rand SpecialReg_Page Page[16];

    rand SpecialReg_select select;

    local uvm_reg_addr_t m_offset;

    

     covergroup cg_addr;

        `ifdef INCA

        option.per_instance=1;

        option.name="SpecialReg_block_cg_addr";

        `endif

        

        Page : coverpoint m_offset

        {

            bins hit =  { 'h0};

        }

        select : coverpoint m_offset

        {

            bins hit =  { 'h4};

        }

        

    endgroup

    

    // Function : new

    

    function new(string name = "SpecialReg_block");

        super.new(name, build_coverage(UVM_CVR_ADDR_MAP));

        

        if (has_coverage(UVM_CVR_ADDR_MAP)) begin

            cg_addr = new();

        end

        

    endfunction

    

    

    // Function : build

    

    virtual function void build();

        

        //PAGE

        foreach (Page[Page_i])

        begin

            Page[Page_i] = SpecialReg_Page::type_id::create($sformatf("Page[%0x]", Page_i));

            Page[Page_i].configure(this, null, $sformatf("Page[%0x]", Page_i));

            

            Page[Page_i].build();

        end

        

        //SELECT

        select   =   SpecialReg_select::type_id::create("select");

        select.configure(this, null, "select");

        select.build();

        

        

        

        //define default map and add reg/regfiles

        

        

        default_map= create_map("default_map", 'h0, 4, UVM_BIG_ENDIAN, 1);

        

        foreach (Page[Page_i])

        begin

            default_map.add_reg(Page[Page_i], 'h0|Page_i << 3, "RW");

        end

        

        default_map.add_reg( select, 'h4, "RW");

        

        

        foreach (Page[Page_i])

        begin

            Page[Page_i].clear_hdl_path();

            Page[Page_i].add_hdl_path_slice($sformatf("Page_Fld_q", Page_i), 0, 32);

        end

        select.clear_hdl_path();

        select.add_hdl_path_slice("select_Fld_q", 0, 32);

        

        

        lock_model();

        

    endfunction

    

    

    protected virtual function void sample(uvm_reg_addr_t offset, bit is_read, uvm_reg_map map);

        if (get_coverage(UVM_CVR_ADDR_MAP)) begin

            m_offset  = offset;

            cg_addr.sample();

            if (this.get_parent() != null) begin

                this.get_parent().sample(m_offset+'h0,is_read,map);

            end

        end

        

    endfunction

    

endclass : SpecialReg_block

Page Register for ARV™ 


If user select the ARV™ output then this sequence task is generated in RAL model .


task Page_pagedWrite(uvm_reg_data_t count,uvm_reg_sequence parent_seq); 

        uvm_status_e status;

        select.Fld.write(status, count, .path(UVM_FRONTDOOR), .map(default_map), .parent(parent_seq));

        Page[count].write(status, 32'ha, .path(UVM_FRONTDOOR), .map(default_map), .parent(parent_seq));

        Page[count].mirror(status, UVM_CHECK, .path(UVM_FRONTDOOR), .map(default_map), .parent(parent_seq));

    endtask

We have shown how paged registers are described in IDesignSpec™ i.e IDS-Word  or IDS-Excel, and  their generated RTL and UVM Reg Model. UVM model is flexible enough to describe any quirky registers. We can change the model and the Register abstract layer according to verification needs.


Note:

  1. Field definition of all the paged registers will be same.
  2. Only one register among the paged registers is accessible at a  time, both from hardware or software side, based on the selector value

Separate read and write select for page register

User can also control the read and write of a paged register using two separate registers. This is done by using “page_rd_select = <regname>.<fieldname>” and "page_wr_select = <regname>.<fieldname>" property. Also, the page_count can be a non 2^n number. 

As shown in the diagram given below, there are 3 registers and two multiplexers, one for read and one for wite, also there is a select line as  “Sel_line[1:0]” which selects the register to be read or written.

Seperate page select for Hardware & Software

Separate page select register  for the hardware and software side have been supported. It can be used with the property 'page_select', 'page_wr_select' or 'page_rd_select'. The properties have been modified in the way:-

  • {page_select=<Select_Reg1_Name.field_Name>:hw,<Select_Reg2_Name.field_Name>:sw}
  • {page_wr_select=<Select_Reg1_Name.field_Name>:hw,<Select_Reg2_Name.field_Name>:sw}
  • {page_rd_select=<Select_Reg1_Name.field_Name>:hw,<Select_Reg2_Name.field_Name>:sw}

Paged register across blocks


A set of registers sharing the same physical address and another selector register is used to decide which out of them is accessed for inter-block in which the select register as well as the paged register lie in different blocks.



Block Diagram showing the Select register and Page register in two different blocks


As illustrated in the above block diagram, the select and page register lies in two separate blocks Block1 and Block2, respectively. The select register ‘SelReg’ controls which Page register from PageReg0, PageReg1, ……. PageRegn will get selected on which read/write operation will be performed either from the SW/HW side.
