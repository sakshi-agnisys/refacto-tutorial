Clearing field after sw write


“rtl.sw_write=clear” clears the register field location one cycle after the sw or hw write occurs on it. So when“1” is written on the field then it gets set for 1 cycle and then gets cleared in the next cycle.  The sw read value from this register field can be then assumed to be zero always (it can be non-zero in case of pipelined transactions, where wr_stb and rd_stb come in consecutive cycles). 


Note: It is supported in Verilog (v.6.12.20.0 onwards) and UVM (v7.8.0.0 onwards)

      rtl_sw_write = "clear";

    };

Generated UVM


When this property is used, “agni_wr_clear_cb”, is attached with all writable fields. The callback updates the field’s mirror value to 0 in case of the write transaction during post_predict stage. This allows both 0 and 1 write transaction values to get sampled but at the end of the transaction the mirror value is 0. In UVM RAL, the mirror value is the value that the model predicts DUT to be holding. The  callback, thus, allows the UVM_CHECK mirror sequence to pass, as on read both DUT and UVM model mirror values are 0.


The “agni_wr_clear_cb” callback also disables the UVM_CHECK feature of the mirror transaction on UVM_BACKDOOR reads, to avoid mismatch error in the uncertainty window (when DUT is still 1 while model mirror value updates to 0). In a sequence “sw write = 1 and backdoor read”, the DUT clears after one cycle so the backdoor read value is 1, but the model value would be 0 (mirror gets updated at the end of write = 1 transaction), causing a mismatch. Thus one cycle after a write transaction is the uncertainty window that DUT takes to clear out.


class agni_wr_clear_cb extends uvm_reg_cbs;

. . .

virtual function void post_predict(input uvm_reg_field  fld,

                                     input uvm_reg_data_t previous,

                                     inout uvm_reg_data_t value,

                                     input uvm_predict_e  kind,

                                     input uvm_path_e     path,

                                     input uvm_reg_map    map);


if (kind == UVM_PREDICT_WRITE) begin 

value = 0;

end 

if ((kind == UVM_PREDICT_READ) && (path == UVM_BACKDOOR)) begin 

// Disable UVM CHECK to avoid uncertainty window (as DUT clear out after one clock cycle)

fld.set_compare(UVM_NO_CHECK);

end 

endfunction

endclass


Note: While creating this UVM model, an assumption is made that the SW read back value from the RTL of the register is always zero. This is mostly true (except for pipelined transactions on IDS target which support one cycle read) and matches the design intent. 

Coverage

For a field having sw access “rw” and “rtl.sw_write=clear”:


Bus Transaction 

Coverpoint Values


Write 

0 and 1

Both write value=1 and write value=0 must be sampled

Read

0

Only read value=0 must be sample 

Note: read value=1 is illegal



Property rtl.write=swclear


Example:        IDS-NG        IDS-Word        IDS-Excel        SystemRDL


The power and HW write clear problem can be avoided by flopping the SW write signal ‘Reg1_wr_valid’ and with the help of this flopped signal ‘Reg1_wr_valid_delay’, register’s value can be cleared. So, this way IDS can reduce the power and will clear only SW write.

For doing this we have introduced a property ‘rtl.write’. The value of this  property would be ‘swclear’ i.e., {rtl.write=swclear}.
