IDS Generated JSON Output

JSON (JavaScript Object Notation) is a lightweight data-interchange format. It is easy for machines to parse and generate. It is based on a subset of the JavaScript Programming Language Standard ECMA-262 3rd Edition - December 1999. It is a text format that is completely language independent but uses conventions that are familiar to programmers of the C-family of languages, including C, C++, C#, Java, JavaScript, Perl, Python, and many others. These properties make JSON an ideal data-interchange language. It uses a drag & drop user interface for mapping JSON structure into metadata (which describes the structure of the data and holds the information about each data element), broken down to individual objects. It is most frequently used to read complex JSON documents or large files. It simplifies the hierarchy of generated output according to input specification.

Basic Hierarchy flow consists of following attributes for each component  (Chip, block, section, memory, register & field).

Following components like address, config, address, address unit, etc. are the common components which are generated in case of chip,block,section,memory,register and field.

JSON data is written as name/value pairs. A name/value pair consists of a field name (in double quotes), followed by a colon, followed by a value.

Case - I (Chip) 


Following input scenario consists of chip level. Generated JSON output consists of the components chip which consists of blocks followed by more attributes like address, config, bus, address unit,offset, size, etc .

JSON objects are written inside curly braces.


SystemRDL Input:


reg reg_1 {

  field {

    sw=rw;

    hw=rw;

  } f1[31:0] = 0;

};


addrmap block1 {

  reg reg_1;

};


addrmap chip1 {

  chip = true;

  block1 block1;

};


Generated JSON Output:


{"chip": {

    "address": 0,

    "chip": [

        true,

        {

            "address": 0,

            "block": {

                "address": 0,

                "config": {"busdomains": {"busdomain": {

                    "address": 0,

                    "addressUnit": 8,

                    "bus": "APB",

                    "endaddress": 3,

                    "name": "default_map",

                    "offset": 0,

                    "size": 4

                }}}

"endaddress": 3,

                "global": true,

                "name": "block1",

                "offset": 0

Conclusion: 


JSON is the most commonly used format for transmitting data (data interchange) from a server to a client and vice-versa. JSON data is very easy to parse and use. It is fast to access and manipulate JSON data as they only contain texts.

JSON is language independent. It can be used to create and use JSON in other programming languages too.
