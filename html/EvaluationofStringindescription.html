Evaluation of String in description

The support/scope of IDS keyword ‘eval ()’ has been enhanced for its usage in description area of in a register specification.

The eval ()  keyword can now extract and add names/size/offset/address/repeat index value of any field/register/section components in appropriate hierarchy to evaluate a string in description of another field/register/section component where it is used. This basically allows users to customize a description of such a specification component at runtime.

The section, register and field components can be accessed for their attribute values as below –

  • %s.<attribute_name>     to access section(regroup) attributes like section name/size/address/repeat index (in case of flattened repeat)
  • %r .<attribute_name>     to access register attributes like register name/size/address/repeat index (in case of flattened repeat)
  • %f.<attribute_name>     to access field attributes like field name and field size

 

The attributes of such components that can be accessed in eval ()  expression at runtime are –

  • name (e.g., %s.name / %r.name / %f.name )
  • offset (e.g., %s.offset / %r.offset)
  • address (e.g., %s.address / %r.address)
  • size (e.g., %s.size / %r.size / %f.size)
  • repeat index value in case of flattened repeat (e.g., %s.iter / %r.iter)

Note:-

  • The keyword for evaluation can be used as eval ( <evaluating string>  ) , with compulsory space character between ‘eval’ and ‘(‘ and also the evaluating string is to be separated with ‘(‘ and ‘)’ using at least a single space character. This has been done to separate this IDS keyword with any normal description content with ‘eval’ as characters in it.
  • The eval () keyword allows concatenation of string (between single quotes )  as well as extracted attributes (accessed as %s/%r/%f.<attribute_name> ) using the character ‘+’ .

e.g., eval ( ‘This is name of ’ + %f.name + ‘ field ‘ )
