counter.signal property


(compatible with IDS version 5.14.0.0 and above)


counter.signal property can be used to control the increment and decrement events of a counter. It is an Active-High event.

These do not control the increment or decrement values, as “counter.incr.val” and “counter.decr.val” , but controls the event of actual increment/decrement of the counter.


counter.signal can contain any external signal name or reference of some dynamic property

Possible Values

  • counter.signal=<some_input_signal>;


some external input signal can be used to control the incrementing/decrementing event of the counter.


  • counter.signal=<reg_name>@overflow;


The overflow of another up-counter can be used to control the incrementing/decrementing event of the counter.


  • counter.signal=<reg_name>@underflow;


The underflow of another down-counter can be used to control the incrementing/decrementing event of the counter.


  • counter.signal=<Another_reg_name>@intr;


Interrupt from another register can be used to control the incrementing/decrementing event of the counter.
