v6.14.16.0

10th August 2018


IDesignSpec™


RTL Enhancements


  • Added specific file header in third party svheader and cheader (6.14.6.4). More Details
  • Changed the third party cheader fileName from xxx_access.h to xxx_reg_access.h (6.14.6.4)
  • Remove UVM warning related to clear_hdl_path() in *_regdef.sv file using simulator switch +define+NOT_CLEAR_HDL_PATH More Details
  • Command line option "-no_lint_warn" is added to remove IP_ERROR & IP_BE_MASK in specific case and offsets incase of unaligned memory More Details
  • Command line option "-parity_widget" at top will remove PPROT & PSTRB ports More Details
  • Blank sw access to fields in third party CSV has been made "rw" to sw and "ro" to hw in IDS format (6.14.6.2)
  • Handled X as default in input third party CSV for verliog output (6.14.6.2)


Bug Fixes:


  • 'd0 instead of 'd in UVM in case of X as default in the spec
  • During hw write, in specific cases 0 could not be written
  • Lint warning removed for width mismatch in comparison. LHS was less than RHS
  • Lint warning removed for port “pstrb” width mismatch while tying it to high in “mrb” file
  • Lint warning removed for *_valid set but not read.
  • Fixed issue third party cheader and vheader when enums have same name in different fields (6.14.6.2)
  • Fixed longInt error in third party cheader when a memory is present with one depth (6.14.6.6)
  • Fixed parselong error in generating UVM coverages (6.14.6.8)
  • Fixed multiple bus domain handling at top level (6.14.6.6)
  • Fixed packed struct bits sort issue (MSB to LSB) in third party cheader file (6.14.6.6)

v6.14.0.0

21st June 2018 (DAC edition)



IDesignSpec™


RTL Enhancements

  • type=content: This property is used to copy all the content within a block or register group when referred in some other document More Details
  • parity: This property has been enhanced to remove redundant logic from Parity, to make it more efficient More Details
  • ecc_sniffer: This property has been enhanced to check the parity of all the regmap addresses when no bus transaction is taking place More Details
  • intr.detect_nometa: This property is used to remove the Flip Flop (FF0) (FF0 helps in  avoiding meta-stability) while detecting the edge in interrupt source. The property is useful for users who would like edge detection without any extra delay More Details


UVM Enhancements

  • Soft constraints: Soft constraints are supported which are used to define default values and default distribution for generated fields More Details


Industry Standards


  • Verification of APB Bus:  A flip flop version of APB Bus has been supported in Verilog (apb_widget_ff.v) More Details
  • Soft reset: Soft resets are now supported in IP-XACT input
  • SystemRDL 2.0 Beta Version: Now the systemRDL 2.0 output can be generated from IDS More Details
  • YAML: IDS now support YAML input More Details


General Enhancements:

  • -fast: When this option is used code can be generated at a very high speed. This option is now supported for Verilog output (Beta Version) as well More Details
  • Default value: Default value of any custom property can now be specified on IDS_template sheet also



Automatic Register Verification ( ARV™ ) Enhancements


  • w1 and wo1: The w1 and wo1 accesses are now supported in ARV™ formal


IDS-NG



  • The hierarchy view of register specification has been made simpler after the annotation check 
  • IDS-NG now supports Custom Word output similar to IDS-Word More Details 
  • IDS-NG now supports Spreadsheet as user specification, similar to IDS-Excel but it is platform independent More Details
  • Param view feature has been added for spreadsheet too More Details
  • Spreadsheet awareness, sequences can automatically generate hints according to the ip More Details 
  • MATLAB output has been supported in ISS in IDS-NG More Details
  • All IDS files can be imported in IDS-NG, making it compatible with IDS More Details
  • TCL file in output generation has been supported More Details
  • GIT has been integrated with IDS-NG More Details
  • TOC for indirect Registers has been supported in IDS-NG 


IDesignSpec™ ( ISS ) Enhancements:


  • Multibit signals: User can now use signals while writing sequences for reading and writing from a port More Details
  • Constraints: Constraints in variants are supported for UVM, C and MATLAB outputs More Details
  • Format: Format property has been supported in SV output More Details
  • Repeat: repeat property and multidimensional repeat has been supported in C, MATLAB and UVM outputs More Details




Bug Fixes:


  • Trigger Buffer: Bug fixed for read channel in trigger buffer
  • Memory: Memory offset representation is now consistent in HTML output
  • HDL Path: redundant comma removed from the code in case of repeat on register group
  • addr_sort: Bug fix for addr_sort option which caused empty space issue in case of memories, structures and typedefs
  • wait and read commands: annotation checks for read command and wait command bug fixes in csv output
  • In ISS, when specification is given in some other format other than excel, it is not neccessry to add IDS template, sequences can be added directly.
  • registered=false: This property has been fixed in ARV™ Formal
  • Lock: Lock register bug fixed in ARV™ Formal
  • Shadow: Shadow register bug fixed in ARV™ Formal
  • Alias: Alias register bug fixed in ARV™ Formal
  • rtl.hw_clear: This property has been fixed in ARV™ Formal
  • rtl.hw_enb=false: This property has been fixed in ARV™ Formal
  • wrs: This access has been fixed in ARV™ Formal
  • hard_reset: This property has been fixed in ARV™ Formal
  • Chip size/block size/regwidth calculated automatically now for RALF 
  • No need to give reg_width separately now for RALF



Known Issues:

            


  • Parity/Sniffer:
    • parity: This property is not compatible with alias register and repeat       property.
    • ecc_sniffer: This property will work with atleast two registers having the parity property. The value of sniffer should always be greater than 4.


  • ISS:
    • Repeat:   Multidimensional repeat and repeat for upper hierarchy has not been supported for firmware output  


  • ARV™:
    • ARV™ Formal supports AXI4FULL Bus partially


  • -fast option for verilog generation:
    • trigger buffer register is not supported
    • alias register and aliased register is not supported
    • different regwidth registers are not yet supported
    • counters with repeat are not supported
    • bus domain at chip level is not supported
    • some properties with repeat are not supported
    • block with repeat is not supported
