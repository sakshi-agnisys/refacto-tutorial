SystemRDL UDPs

SystemRDL UDPs


User-defined properties enable the creation of custom properties that extend the structural component types in a SystemRDL design. IDesignSpec™ support multiple properties for registers, which are currently not supported by SystemRDL. To Support these properties multiple UDP are given by Agnisys®, which can be used to add this functionality.


Below is the example to show how clock edge can be changed, which helps to sample the values of register on specific clock edge.


User Defined Property declaration Syntax


property property_name {attribute; [attribute;]*};


where,


  • property_name specifies the new property
  • attributes are specified as attribute=value pairs, e.g. type=number


Please ref to SystemRDL spec for more semantics on SystemRDL UDPs.

Sample SystemRDL code


property clock_edge {type = string; component = addrmap|reg;  }; // User Defined Property declaration


addrmap block_name {

  name = "block_name Address Map";

  clock_edge = "negedge" ;               // Use of UDP


  reg reg_name {

     regwidth = 32;


    field {

      hw = rw;

      sw = rw;

    } Fld[31:0] = 32'h0; 

  };

 

  reg_name reg_name @0x0000;


};


Note: All the IDS properties containing a dot (.) in its name will be converted to underscore (_) while specifying in SystemRDL. (For example: intr.pending will be converted to intr_pending)

Remove UDP’s forward declaration while converting to rdl from any input format


When input test cases have some UDPs on any of its components “chip/block/section/reg/field” and generating SystemRDL output then, that particular UDP is declared in the generated RDL output file as shown below:

property <property_name> {type = string ; component = addrmap|regfile|field|reg ; };


Some users do not need a UDP declaration as they already have a companion package file “IDSProperty2a.rdl” having all the UDPs already declared. Therefore, we implemented the guard band feature in SystemRDL output as well as in the companion package file “IDSProperty2a.rdl”.
