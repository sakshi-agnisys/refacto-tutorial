Struct template in Headers

Struct template for Svheader and Cheader 


For Svheader velocity template is used and for Cheader normal command line is used. Struct is supported only for chip level and block level testcase.

For svheader there are two cases which are defined below:


Case 1

/*////////////////// Struct:  Struct1 ///////////////////*/

typedef struct {

    union {

        #ifdef IDS_BIG_ENDIAN

  struct { 

hwint F5 : 2; 

hwint T2 : 48;

               hwint Q5 : 48;

hwint padding : 30;

         } word0_bf;

        #else

  struct {

hwint padding : 30;        

hwint Q5 : 48;

hwint T2 : 48;

               hwint F5 : 2;

         } word0_bf;

        #endif

        hwint32 word1;

    };

} Struct1;


//////// SV header

typedef struct packed {

               logic [1:0] F5;

               logic [47:0] T2;

               logic [47:0} Q5;

} Struct1;

       } Struct1_padded;

In case of padding,


typedef struct packed {

       Struct1 Struct1_data;

       } Struct1_padded;


If testcase is a chip level testcase then, use “chip:” before velocity template name.

e.g., if velocity file name is “xyz” then command line is


IDSbatch <input-file> -velocity chip:xyz  


For Block level testcase command line is


IDSbatch <input-file> -velocity “xyz”



Structs generated for the structure template when present inside chip as well as block


Example - 

Suppose we have a chip level testcase with struct inside chip as well as block as follows -




Code


SVHeader


typedef struct packed {
logic [9:0] S1; 
logic [19:0] S2; 
} Struct1;

typedef struct packed {
Struct1 Struct1_data; 
logic [69:0] padding; 
} Struct1_padded;

typedef struct packed {
logic [29:0] S1; 
logic [59:0] S2; 
} Struct2;

typedef struct packed {
Struct2 Struct2_data; 
logic [9:0] padding; 
} Struct2_padded;


CHEADER -
