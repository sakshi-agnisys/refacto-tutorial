Interface External Memory/Section

(compatible with IDS version 5.10.0.0 and above)


The earlier versions of IDS did not support more than one external memories/section with different latencies. The scheme adopted in earlier version only supported single external memory or multiple memories with single cycle read/write transfers. The new scheme adopted in IDesignSpec™ version 5.10.0.0 and above supports multiple memories with different read/write latencies.

 

External Register Video

Using IDesignSpec™ you can connect an external 

1. Register 

2. Register Group - Register Array/Register File. 

3. Memory    

4. Block 


to an IDS generated rtl module. In order to do so, the Register/Register Group/Block template in the IDS specification must have the property "external" set to true/1. Please note that for Memory template external is true by default.


Further in this document we shall refer these external Register/Register Group/Memories as “external targets” to the rtl module.  

IDS provides appropriate signals to interface with these external targets. Following is the list of these signals:


 address_out

 raddress_out

 wr_data_out

<targetName>_wr_req_in 

<targetName>_rd_ack_in 

<targetName>_rd_data_in 

<targetName>_wr_valid_out 

<targetName>_rd_valid_out 


Where definition of each is 

  • address_out  -- Write address bus to target with AXI bus and same port is used for Read & write for all other buses due to single channel.
  • raddress_out -- Read address bus to target with AXI bus.
  • wr_data_out  -- Write data bus to target.
  • wr_valid_out -- Write Valid Out. It is an input to the external target. This signal informs the corresponding external target that the bus initiator wants to write on it.


  • rd_valid_out -- Read Valid Out. It is an input to the external target. This signal informs the corresponding external target that the bus initiator wants to read from the target.


  • wr_req_in -- Write Request In. It is an output from the external target. This signal is used to indicate wait states for the write transaction. By default it should be high, indicating it is ready to be accessed by bus initiator. Otherwise, if it is low it indicates external target is busy and no further transaction is allowed (in case of AXI, write transaction is not allowed).


Note: If the external has a latency >= 1, then "_wr_req_in" must be pulled down immediately, when "_wr_valid_out" is asserted high. This is done because, if the external logic has zero latency then it will work with a single cycle write.


  • rd_ack_in -- Read Acknowledge In. It is an output from external target. This signal is used by the external target as framing signal, to indicate whether valid data is present or not on signal “rd_data_in".  The signal shall be asserted high by the external target when the valid read data is available on “rd_data_in”.


Note: If the external has a latency >= 1, then "_rd_ack_in" must be pulled down immediately when "_rd_valid_out" is asserted high. This is done because, if the external logic has zero latency then it will work with a single cycle read.


  • rd_data_in -- Read Data In. This signal is a bus and an output from external target. The target shall connect its read data bus to this bus. Its width is equal to the bus width.



Each of the signal below from all of the external targets in a block, 

1. _wr_req_in    - are ANDed together to form “request”.

2. _rd_ack_in    - are ORed together to form “rd_data_valid”.

3. _rd_data_in   - are ORed together to form “rd_data”.


The following signals are used for handshaking between initiator and external registers.

1. request -- This signal is an output from the block to the initiator cpu or the chip containing the block. This signal shall be asserted high when a register or external target is ready for the write transaction else if busy the signals shall be asserted low. All “_wr_req_in” signals from all of the external targets are ANDed together to form the request signal.


2. rd_wait -- This signal is also an output from the block to the initiator cpu or the chip containing the block. This signal shall be asserted high when a register or external target is ready for the read transaction, if busy the signals shall be asserted low.

 

3. rd_data -- Read Data bus. These are the read bus signals. The data read from the registers are made available on this bus.
