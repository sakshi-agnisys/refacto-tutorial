IDS-Batch CLI

IDS-Batch CLI is a command line utility for transforming register specifications. It accepts various input formats and can generate a variety of outputs. This tool is typically used by Design, Verification, Firmware, and documentation groups for managing register map and memory map information. 


To install, refer section Installation Steps-> IDS-Batch


Following command line can be used to generate requisite output from IDS-Batch CLI: 


idsbatch [optional arguments] –out “format(s)” [-tcl "<list of TCL files>”] input_file 


input_file: It is the input file to be transformed. Supported input files are:

Generating MS Word files 

One of the real advantages of IDS-Batch is its ability to create Word files from register specifications. This allows you to automatically document the register specification. The format of the output files can be specified via user defined templates. IDS-Batch understands special keywords that are substituted for real data when generating the files.

Sample Runs 

1. IDS-Batch command to generate OVM from IPXACT

% IDS-Batch -out ovm chip1.xml 


2a. IDS-Batch command to use MS-Word as an input on Windows OS. 

% IDS-Batch –out “rdl vmm uvm” soc_chip.docx 

 

2b.  IDS-Batch command to use MS-Word as an input on Linux/Windows OS and not using IDS cache files to create the outputs.

% IDS-Batch -nocache –out “rdl vmm uvm” soc_chip.docx 


3. IDS-Batch command to use MS-Excel as an input on Windows OS. 

% IDS-Batch –out “uvm verilog” mem_bdl.xlsx 

 

4. IDS-Batch command to use CSV as an input on Linux/Windows OS. 

% IDS-Batch –out “uvm verilog” hre_krsn.csv 

 

5. IDS-Batch command to generate Word 2007/10 files in output directory ‘doc’ using default templates from System RDL 

% IDS-Batch –out word2007 –dir doc chip1.rdl 


6. IDS-Batch command to generate RALF and Word 2007 files using Custom Word templates from System RDL

% IDS-Batch –out “vmm word2007” –template my_template_dir chip1.rdl 


7.  IDS-Batch command to generate custom outputs using tcl file from OpenOffice

% IDS-Batch –tcl myout.hxx.tcl chip1.odt 


8. IDS-Batch command to generate Verilog and XRSL output using CSV input file

% IDS-Batch -out "verilog xrsl" -bus amba DesignSpec.csv


9. IDS-Batch command to use customized "IDS_template" sheet

% IDS-Batch -out xrsl -template_file D:\IDS_examples\IDS_template.csv DesignSpec.csv


10. IDS-Batch command to import IP-XACT design with bus and other IP components 

% IDS-Batch -design "<top design file>"  "<component reference file(s)>" -bus_comp "<bus component file(s) separated by comma>" -chip_size "<any integer value>" -addr_sort  -out  "xrsl verilog uvm" -dir "<desired output directory>"


11. IDS-Batch command to use customized word template directory for word output 

% IDS-Batch "<input file path>" -template_dir "<path to word template directory>" -out "word"


12. IDS-Batch command to exclude post-processors (define/enum/variant) for word output.

% IDS-Batch -out word  -docexclude "<any of the following, enum|define|variant|udp>" "<any input filename>"

 

13. IDS-Batch command to discard formatting from excel|word input for html/pdf

% IDS-Batch -out html -no_formatting "docx or xlsx file"


14. IDS-Batch command to create word datasheet (SoCe datasheet).

% IDS-Batch -out worddatasheet  "<any input filename>" 


15. IDS-Batch command to create output in form of Duolog format using IDS input file 

% IDS-Batch "<any IDS input filename>"  -out "<any output type>" -out_third_party d 


16. IDS-Batch command to create output in form of Duolog format using Doulog input file 

% IDS-Batch "<any Duolog format input filename>" -in_third_party d  -out "<any output type>" -out_third_party d 


17. IDS-Batch command to create output in form of IDS format using Doulog input file 

% IDS-Batch "<any Duolog format input filename>"  -in_third_party d  -out "<any output type>" 


18. IDS-Batch command to override multiple bus domain table

% IDS-Batch "<any IDS input filename>" -out "<any output type>"  -mbdc  <"busDomianName:bus:AddressWidth"> 

             Note: arguments of -mbdc must be semi semi-colon separated


19. IDS-Batch command to specify property at top level

% IDS-Batch "<any IDS input filename>" -out "<any output type>"  -top_property <"property_name=property_value">

Note: arguments of -top_property must be semi-colon separated 

     20. IDS-Batch command to ignore enums

     % IDS-Batch "<input xml filename>" -ignore_enums -out "<any output type>" 


     21. IDS-Batch command to remove redundant hierarchy in IP-XACT import

      % IDS-Batch "<input IP-XACT filename>"  -out "<any output type>" -top_property "ipxact_compact=true"


     22. IDS-Batch command to switch from big-endian to little-endian

      %IDSbatch <filename> -out <output format> -big_endian false -little endian true


     23. IDS-Batch command to switch from little-endian to big-endian

      %IDSbatch <filename> -out <output format> -big_endian true -little endian false


     24. IDS-Batch command to define board hierarchy in SystemRDL

       %IDSbatch <filename> -out rdl -top_element board:<top element name>


     25. IDS-Batch command to override with default path and will search included rdl files in the requisite directory

        %IDSbatch  <top.rdl>  -search_path <path of the included files> -out <output format> -bus <supported bus protocol>


     26. IDS-Batch command switch used in case multiple files are included/referred from different directories in another systemRDL file. The relative path of multiple files can be given semi colon separated

       %IDSbatch <top.rdl> -incdir <relative path of the included files; relative path of other included file> -out <output format>

  

     27.IDSBatch command:

         %IDSbatch <top.rdl> -incdir <relative path of the included files; relative path of other included file> -out <output format>


     28. IDS-Batch command to run SystemRDL 1.0

       %IDSbatch <filename> -out <output format> -rdl_version 1


     29. IDS-Batch command to direct output to a specific folder

       %IDSbatch <top.rdl> -out "<output1> <output2> <output3>" -dir IDS_output -dir_out_specific "<output1>:folder_name1, <output2>:folder_name2, <output3>:folder_name3" -bus custom

       Syntax: -dir_out_specific "<name of the output>:<name of the folder>" multiple outputs should be comma separated


     30. IDS-Batch command to generate standard APIs for IP

       %IDSbatch <IDSng file of IP> -out <verilog> -slipg_sdk "uvm firmware" 

       NOTE: APIs are generated in UVM and Firmware format only.


     31. IDS-Batch command to support hierarchical generation of files w.r.t chip/block/section

% IDS-Batch "<input_file>" -out htmlalt2  -if_html  -if_html_section 


32. IDS-Batch command for low power coding style 

     % IDS-Batch "<input_file>" -out verilog -bus <bus_name> -lowpower


      33. IDS-Batch command to use "capi_add_name_prefix" property 

     % IDS-Batch "<input_file>" -out header -c_api 


      34. IDS-Batch command to use verbosity switch 

     % IDS-Batch "<input_file>" -out "uvm verilog" -bus <bus name> -verbosity high


     35. IDS-Batch command to use -capi_return_type switch

     % IDS-Batch  "<input file>" -out header  -c_api  -capi_return_type <int/hwint>


     36. IDS-Batch command to use -third_party_rtl switch

     % IDS-Batch  "<input file>"  -out uvm  -bus <bus name>  -third_party_rtl


      37. IDS-Batch command to use top_property switch 

     % IDS-Batch "<input_file>" -out <output> -bus <bus name> +top_property


NOTE: 

A: For the first iteration, user gives -top_property and in second and third iterations, user gives +top_property. In such case, all the values of all three top_property will be appended.

B: For the first iteration, user gives -top_property, in second iteration, user gives -top_property and the third time user gives +top_property. In such case, the value of the second and third top_property will be appended because the second -top_property will override the first.

C: For the first iteration, the user gives -top_property and for the second iteration, user gives +top_property and for the third iteration, user gives -top_property. In such case, the value of only third top_property will be considered because the third -top_property will override the second.


Conclusion: Whenever there is "+" sign in top_property, then it appends but "-" sign in top_property switch overrides.

Troubleshooting 

Error message

 What to do 

Word 2007 not found 

On Windows machine, indicates that you do not have Microsoft Word installed. Word 2007 is needed if Word 2007 files are generated.

IDSbatch not found

Make sure the IDS-Batch installation directory is in your PATH. On Linux/Solaris machines, check the execute permission of IDS-Batch 

Word 2007 file not generated

Make sure MSWord 2007 in installed on the machine.
