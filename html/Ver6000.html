Ver 6.0.0.0

v6.0.0.0 - Major Release

05/31/2016


Latest Supporting Files are here.

IDesignSpec™ (IDS-Word, IDS-Excel, IDS-Batch CLI) Enhancements

  • General Enhancements
    • Structs (IDS-Batch CLI) - User can now define the descriptors and data packets used between the data memory and controller using “Structs” in IDesignSpec™.
    • Array Indexing -IDesignSpec™  now comes with new feature where the name of array elements can be indexed.
    • IDS with Vivado - IDesignSpec™ provides a unique specification driven flow that enables users to create design specification and with the help of Vivado™ create bitstream. 
    • Parameterized Register Field - IDesignSpec™ register fields can now be made generic using parameters. 
    • Local Memory Maps (IPXACT1.5) - In IPXACT (1685-2009) local memory maps can be used as input with IDesignSpec™ to generate all IDS supported outputs.
    • Import - Performance of import functionality in IDS-Word and IDS-Excel has been improved to use up less system memory and import hundreds/thousands of registers in minutes.
    • Hierarchy Based Address Sorting - User can now restrict addressing sorting to single level using "hier_sort=false" alog with addr_sort property
    • Direct Indirect Address Map (Verilog, UVM, ARV™) - It is now possible to specify indirect virtual address maps in IDesignSpec™ and connect these with direct address map using proprietary interface.


  • RTL Enhancements
    • FIFO (Verilog) -  IDS provides the FIFO (first in first out) functions through the parameterizable single clock IP cores.
    • AXI-4 LITE (Verilog & VHDL) - AXI-4LITE bus supported has been enhanced by removing 1 cycle latency for read and write process.
    • Signal Based Reset Value (Verilog & VHDL) -Reset Value for a register field can be now driven using an external signal.
    • Field Lock (Verilog) - The software write access of a register field can now be locked based on an expression consisting of different register fields or some logical expression.
    • Multiple Reset Values (Verilog) - IDesignSpec™ now supports multiple reset value for register array.
    • Interrupts per Channel (Verilog) - IDesignSpec™ now supports to have multiple interrupt outputs from the block. 
    • HW-SW precedence (Verilog) - The write action from software and hardware can be now prioritized using "rtl.precedence" property.
    • Counter Enable (Verilog) - Counters in IDesignSpec™ can be now enabled using any register bit.
    • Soft Reset (Verilog) - Soft reset for register fields can be customized to be another register bit.
    • Interrupt Signals (System Verilog) - Interfaces for interrupts signals has been enhanced to System Verilog Wrapper
    • Clock Domain Crossing Signals (System Verilog) - Interfaces for cdc signals has been enhanced to System Verilog Wrapper
    • Lock for Set/Clear/Toggle(Verilog) - The software set, clear and toggle access can now be locked based on an expression consisting of different register fields or some logical expression.
    • Reset Signal properties (Verilog) -  New properties for signals have been implemented to configure external signals nature
      • <signal_name>.type - To configure the reset signal to be of asynchronous or synchronous type
      • <signal_name>.level - To configure the reset signal to be active high or low


  • UVM Enhancements
    • Custom Code - Now it is possible to add custom UVM code in the function "new" for any UVM class in IDS


  • Header Enhancements
    • Multiple Files Generation(C header)- Used to generate separate header file for each block and chip.
    • IDS Parameters (Verilog header) - IDS parameters are now available in Verilog Header output


The following header enhancements have been made. For more details, please see the C/C++ Properties.

    • "header.field_mask" - Used to generate field mask values
  • IDS-Batch CLI Enhancements 
    • "Worddatsheet" output - A new output in DOCx format can be now generated using IDS-Batch CLI on the Linux OS.
    • CSharp output - CSharp output can be now generated using IDS-Batch CLI.
    • C API - C API can be now generated using IDS-Batch CLI
    • IO Ports (Signals): User can now create named external ports in Excel format, in the block or chip level, and connect certain internal design signals to the external world.
  • TCL Enhancements 
    • get_reg_width - Returns the register width for given register object
    • get_unit - Returns the address unit for the given object
    • get_domains-  Returns the list of all bus domains on which the given object exists.


Various enhancements have been made for TCL API for custom output generation. 

  • Industry Standard Output Enhancements
    • SW Access (IPXACT1.5) -IPXACT(1685-2009) generated from IDS has been enhanced to support all IDS supported SW access. 
    • Signals (SystemRDL) -  Signals component in SystemRDL standard has been now structured with connectivity in IDesignSpec™. 
  • General Changes
    • UpgradeIDS  - The Upgrade IDS functionality behavior has been changed to improve the plug-in load time.


Bug Fixes


    • Wrong naming generated in VHDheader and C header for subblocks
    • Wrong blocksize was populated to IDS generated Excel output
    • IRQ signals was declared incorrectly for reggroups with repeat
    • Memories inside subblock, with sw access as read-write was not handled correctly in IPXACT output
    • Write signals for external register inside reggroup were not assigned in VHDL alt1 output
    • Decimal values in IPXACT input were interpreted as Hexadecimal
    • Wrong offsets were generated in UVM output for repeat=1 on register.
    • End addresses for register in VHDL alt1 output was not generic as per the block offset
    • Multiple files for UVM output were not correctly generated for subblocks
    • Components Description was lost with automatic coverage and HDL path generation settings
    • CustomCSV output did not create on customizing CSV header
    • Register was declared external on specifying "external=0"
