Spec Formats

Here we have four different Spec formats(views) which are interchangeable to each other. User can move from any of formats to any other formats by clicking on the particular view user wants to move to.


1. Register View: This is the view for creating IDS-Word like specs. Click on the appropriate button from the tool bar to add an IDS-NG template. User can fill required columns in the template with an appropriate name according to the desired specifications. IDS-NG also has an additional feature of hints for both s/w and h/w access, Along with this, it has the capability to provide hints for the property which helps in identifying the name of property for correct spell check. Hints can be accessed by typing dot, that is '.' , in the respective section.



2. Sequence View: 

Sequences are a “set of steps” that involve writing/reading specific bit fields of the registers in the IP/SoC. These sequences can be simple, or complex involving conditional expressions, array of registers, loops, etc. 

IDS-NG enables users to describe the 'programming and test sequences' of a device, and also automatically generate sequences ready to use from an early design and verification stage to post silicon validation. To inculcate this functionality, user can switch to the ‘Sequence view’ by clicking on that as we have provided the functionality. 

In IDS-NG, User can move to the sequence view by clicking on “Sequence”. Add the Sequence template and write commands in command column and write the values on particular register field. 

IDS-NG supports following outputs for sequence:

1. UVM

2. Firmware

3. System Verilog

4. MATLAB

5. CSV


Features: Following are some additional features of sequence view in IDS-NG:

  • User can have multiple sequence template.
  • In Sequence template, there is hinting provide for the reggroup, registers and fields.
  • User can have multiple generated files for multiple sequence even if they are using ‘call’ command.
  • When user move to sequence view and add a sequence template, then he/she gets already written IP and sequence name.

3. Param View:

User can switch between reg view and param. Param View is based on Yaml file (.yml). YAML is a human readable data serialization language. It is commonly used for configuration files, but could be used in many applications where data is being stored (e.g. debugging output) or transmitted (e.g. document headers). It uses both Python-style indentation to indicate nesting.


IDS-NG, Param view has the information about the registers and their fields. It does not have the information about the block, chip or any other template used in IDesignSpec™. 

Also, while hovering over fields in param view, IDS-NG gives information like, name, size, software/hardware accesses and default values of the fields to the user.

User can move the fields of registers from one address location to another by dragging them and corresponding changes reflect in reg view. Changes made in the reg view or spreadsheet view are also reflected back in the param view. Now, the param view is editable. We can add, delete or paste registers in param view. Also, if we do any modification in param view, then it will get reflected in the register view and the other views. 


In param view, when user will lock the specific parameter then its param will not be updated.

After Import: 




Similarly, to update param, choose ‘Update Param’ option. Updating new file will reflect the changes in the param view. And all the differences between the originally imported file and the updated file can be seen in the ‘Param diff viewer’. In ‘Param diff viewer’, user can check/uncheck a particular field as per required register specification. Changes are also reflected back in the corresponding Register/Spreadsheet View.



Additional features in Param view:


  • User can edit delete and add  reg through button 


    

  

  • When the file is updated in Param view the previous changes will be shown in Param diff viewer, and if the diffs were checked the changes will occur in updated IP also.

            



  • Deleted reg in param view: deleted reg shown in updated param.







  • User can add and delete register at a time. 




  • Deleted and add register both shown in param view.


 

  • Param diff viewer will not show in 2D reg output

Imported Yaml file label and param viewer are removed from 2D reg output.

5. Sequence and Spreadsheet as-par with Excel


In IDS-NG the GUI of spreadsheets has been enhanced. Font size and font style has been changed. On the right click on any cell “Insert Row Above” and “Insert Column Left” options has been added.


6. Supported pop-up for opening a project directory if it is not a project directory in IDS-NG



Earlier when the user tries to open a directory which doesn’t have the project configuration directory, it only gives the pop-up that shows a message that it is not a project directory, which doesn’t allow a user to open old projects but now a pop-up has been added to ask the user if the directory selected is not a project directory and whether he/she wants to open the current directory as a project or not.
