Wide access memory with greater buswidth

IDesignSpc™ already has a property wide_external_access, in which it supports external memory having width greater than the bus width for direct access. So, for example if a 128-bit memory is attached to an external bus, only the 32 lsb will be accessed directly using a 32 bit register bus, the remaining 96 bits will be left unused. 


Fig 1. Wider External Access with Smaller Bus



To support direct access to external memories having width greater than the bus width, we will implemented a logic, similar to Trigger Buffer Register. This additional logic will aggregate individual smaller  writes/reads into one combined memory write/read. Here, combined memory writes have already been supported with the help of property wide_external_access and combined read has been implemented in this proposal with an additional argument “:r” in the same property.

Suppose we have a 128 Bit memory which is being accessed by a 32 Bit bus. For a combined read transaction, we will read the entire 128 Bit on the external interface when the first (LSB) read transaction comes on the bus interface. The lower 32 Bits will be sent to the bus interface while the rest of the 96 Bits will be stored in the buffer, which can be then read through required 32 bit read transactions on the bus interface. 

Therefore, in the default case of a combined read transaction for wide external memory, we read the entire bits on the external interface when the first (LSB) read transaction occurs on the bus interface. Earlier, in the case of a combined read transaction, we read the entire 128-bit on the external interface when the first (LSB) read transaction comes on the bus interface and the data was part selected and sent as 32-bit for each read transaction at bus interface.  But now, an extended property Wide_external_access=true:r  has been proposed where additional logic will aggregate the individual smaller reads by introducing the read back buffer stages similar to the buffer trigger register.


SystemRDL Example


property wide_external_access {type = string;component = mem;};

addrmap Chip1 {

  addrmap Block1 {

       mem {

         memwidth = 128;

         mementries  = 4;

         wide_external_access = “true:r”;

       }Mem1;

    reg  {

       regwidth = 32;

       field {

        hw = na;

       sw = rw;

      } F1[31:0] = 32'h0C;

    }Reg1;

  };

  Block1  Block1 @0x00;

};
