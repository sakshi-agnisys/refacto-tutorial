Ver 6.34.0.0

v6.34.0.0 

(Feb 28th,2020)


IDesignSpec™ (IDS)


RTL Enhancements:

  • Parity is supported on per-byte basis, where 1-bit in each byte will indicate the parity value. (More Details) 
  • {parity_bus_error=true} property is supported for assertion of parity error into the bus error. (More Details)   
  • CDC widget has been supported for AMBA-APB bus protocol to have CDC from software side. (More Details)
  • {field_genvar=true} property can be used for field parameterization to enable parameterized field bit offset in case of repeat.(More Details)
  • {aggregation_logic=axitocustom} property is supported for connecting AXI4FULL widget with individual custom widgets of different blocks. (More Details)
  • Trigger Buffer with repeat value greater than one is supported in reggroup.
  • {rtl.clear=swclear} property is enhanced for assigning 0 or clearing the register. (More Details) 
  • A new sw access ‘r/wo1’ is supported for reading sw, but writing sw only once.



SystemRDL Enhancements:

  • "-no_warn" switch is introduced for skipping all SystemRdl warnings generated by the compiler. (More Details)


UVM Enhancement:

  • {uvm.enum.name_format} property is supported for customizing the names of enum and its labels.(More Details)
  • Expressions in hdl_path are supported in case of repeated elements.
  • Addition of bins in cross coverage is supported.(More Details)
  • Auto-mirroring is supported for updating the hardware interface value on regmap whenever a hardware event occurs.(More Details)
  • Hard and soft constraints with integers, interdependent register values and enum are supported with the following features:
      • Static constraints and unique constraints.
      • All the supported constraints are allowed on all the templates, i.e., block, section, registers and field.
      • Single constraint and multiple constraints on a single template.


IPXACT Enhancements:

  • Displayname  in enumerated values is supported while importing IPXACT in documentation outputs.
  • {ipxact_exclude_param=true} is supported for blocking the generation of parameters while importing IPXACT. (More Details)
  • Multiple regwidth registers(not in power of 2) are supported while importing IP-XACT.


General Enhancements:

  • Performance improvement through Incremental Generation functionality for Verilog & HTML outputs.(31% improvement in performance on an average size testcase).(More Details)
  • -hide_htmlalt2_prop command line switch is supported for hiding property's description, only the property name with their values are visible in the description.(More Details)
  • Flexlm license is supported in IDS-Word, IDS-Excel.(More Details)
  • -verbose command line switch is supported to display the IDS-Batch generation process on the console. (More Details)
  • “description” property is supported to shrink/suppress descriptions of particular entities like chip,block,section,register or field in case of C header (alt2) outputs when given at the top element (More Details)


Bug fixes:

  • Fixes done for aggregation logic and chip logic with 'module_name' property. 
  • Fixes done for {vhdl.entity} property with {module_name} property in VHDL.
  • Fixes done for AMBA3AHBLITE widget for ‘hresp’ in case of  ‘error after delay’.
  • 'assign' property in signal table has been fixed to support all reduction logic.

Specta-AV™ 


Enhancements:

  • Specta-AV™ Checkers are supported in python. (More Details)
  • Scope of knobs are extended for controlling specific tests/sequences to run, these tests are to be run for specific register accesses and for  number of iterations.
  • Complete report is generated using python script showing which tests/sequence are run along with their pass/fail result.
  • A text file report is generated to show total code coverage.


IDS-NG


Enhancements:


  • Flexlm license is supported.(More Details)
  • Specta AV is supported.
  • Enhance Update Param with Param locking, filter and category base update functionality is supported.
  • Filter base update with different catagories is supported.


Bug Fixes:


  • Fixed error in 2D regmap output.
  • Fixed the dependency of project for creating, opening and import files.
  • Fixed the issue, “copy a row in cross different files”.
  • Fixed the issue, "unable to create a new file without a project".
  • Fixed the issue of configuration settings window not accepting relative path.
  • Fixed the issue, “properties are missed in html output”.
  • Fixed the issue of character encoding.
  • Fixed the issue, “formatting causes error in template”.
