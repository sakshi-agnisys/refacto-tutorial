Flexlm License(Standard)


Introduction to Flexlm


  • Flex license is a licensing tool that is used to manage the software license and protect software against overuse.
  • It is a prominent license management solution that enables software vendors to impose restrictions on the number of software seats available to their customers. Flexlm supports different licensing policies such as Floating and Node licenses.
  • It is similar to the Agni License Server (ALM)


Prerequisite requirements


For using flex license, you should set the environment variable "LM_LICENSE_FILE" with the ip address where the license.dat is present and must unset the environment variables "AGNI_LICENSE_FILE" and "AGNI_LICENSE_SERVER".


Categories of flex-based license in IDesignSpec™


There are following categories of flex-based license in IDS:

  • SOC: Have all the IDesignSpec™ features 
  • ADV: Have the advanced features 
  • PRO: Have the basic Features


License priority in Environment variable for IDS


There are three types of licensing supported in IDesignSpec™. There list, in the order of their priorities, is as below:

  • AGNI_LICENSE_SERVER (server-based Agni license)
  • AGNI_LICENSE_FILE (file-based Agni license)
  • LM_LICENSE_FILE (Flex licensing)


Additional commands for flex license in IDS-Batch


-lic: It returns the license information like expiry date total number of count and other additional information.

Usage: IDS-Batch -lic   


-queue_lic:  It is used to queue the process when user reached the count limit.

Usage:  IDS-Batch <input_file> -out <output_formats> -queue_lic <time_in_minutes>   

            In above command line, <time_in_minut> is the waiting time of batch running.


-retry_time: <license retry time in second>: Specify license checkout retry time in seconds. 

Usage:  IDS-Batch <input_file> -out <output_formats> -queue_lic <time_in_minutes>  -retry_time <time_in_seconds>

           In above command line, time_in_minutes is timeout-period and it is in minutes; and time_in_seconds is retry after time in seconds.


e.g., IDS-batch input file -out output file -dir directory -queue_lic time in min -retry_time time in sec.

 

Note: Now, both floating and node lock license are supported.


And also modifications in "Prerequisite requirements" can be done:


For using Flex License, user have to set the environment variable "LM_LICENSE_FILE" and its value in port@server where port will be 8090 and server is address of the server.

In case of node lock license, put the license file path as the path value.


Note: In case of Flex license, user must unset the environment variables "AGNI_LICENSE_FILE" and "AGNI_LICENSE_SERVER".


Flex License Queuing


Flex queue used for assigning a License for a user in random order. Ex. License having only 2 count and 5 users send request for checkout license then first 2 user check-out a license and other are in queue.

In IDS-Batch CLI, user can use queue functionality using “-queue <wait_time_in_minutes>” command line switch.

Ex:

idsbatch <spec_file> -out <outputs_name> -queue <wait_time_in_minutes>
