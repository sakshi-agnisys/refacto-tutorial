Signals for RTL


“Signals” create, named external ports, in the block or chip level ,and connect certain internal design signals to the external world.





How to use Signals in IDS-Word


The signals can be specified in the IDesignSpec™ Word using the Signals table. Signals table is shown in the figure given below.

 

 


Signal template

  

  • User Defined Name: It is a unique name given to a set of signals, that can also be used to ref those signals at a chip level. This name must be unique in a same level of hierarchy. 


  • Signal Name: Name of a signal in the port list, that can also be used to connect that signal in the design. Choosing a “Signal Name” is mandatory, and it must be unique within the same block/chip.        


  • Port Type: It describes the type of port. The port can be input, output, and inout in the release 5.14.0.0 and above. User can use input/in for input port or output/out for output port and inout for an input-output port. 


  • Description: User defined description for signal.

How to use Signals in Excel

(Compatible with IDS-Batch version 6.0.0 and above)


The signals can be specified in the Excel format using the IO Ports sheet. Add the IO Port sheet to the workbook using the button in ribbon

 

 


Signal template

  

  • port_name: Name of a signal in the port list, that can also be used to connect that signal in the design. Choosing a “Signal Name” is mandatory, and it must be unique within the same block/chip.        


  • direction: It describes the type of port. The port can be input, output, and inout in the release 6.0.0.0 and above. User can use input/in for input port or output/out for output port and inout for an input-output port. 


  • io description: User defined description for signal.


Note:

  • spec_type and ams_discipline columns are reserved for future use.
  • By default, port type is input. 



Other Properties for signals: 


  • next- The next property is used for an output signal to assign value on signal. 
  • field_reset- The field_reset property is used to make a signal default soft reset for the block. 
  • level- To change the level of signal to active high or active low, this property can be used. By default level is active low.
  • ref- To reference the signals present in other blocks, user can use the ref property in signals definition as shown in examples below. 



NOTE: The "field_reset" property can be applied only on 1 signal within 1 block.

Reset Signal (resetsignal)


The IDesignSpec™ by default creates a reset signal for generated RTL called “reset_l”. This reset signal is active low by default and is used to synchronously reset the flip-flops. The “reset_l” is called hard reset because it is same for all the block or chip unless some other properties applies.


A “resetsignal” property is used to reset the register/field with many other ways. “resetsignal” are used as soft reset to reset the register/field with more than 1 signals.


Property name

Feature

Applicable

Value

resetsignal

External input signals to reset the field 

Field/reg

resetsignal=<signal_name>

resetsignal=<register.field>


If "resetsignal" is applied on a register then all the fields have a same soft reset, unless, the field has a different "resetsignal".

Signals for IP-XACT


Now, by using Signals table of IDS, user can generate one or more bus interfaces and signals interfaces at IP level. It generates a "bus definition" and an "abstraction definition" files for the specified bus in the output directory with the name of bus.


        For example, if bus is PROPRIETARY then the "bus definition" and "abstraction definition" will be as PROPRIETARY_abstarction_defination.ipxact.xml and PROPRIETARY_bus_defination.ipxact.xml.


 For using this feature:

1. Buses are defined as “userDefinedBusNAme (Valid bus name)” in "Signals" template.

for example Bus1(AMBA-AXI), busIP (OCP) etc.

    

 


Valid Buses are: AMBA-AXI, OCP, AMBA-AHB, AVALON, AMBA-APB, AMBA3-AHB-lite, WISHBONE, PROPRIETARY. 


  1. All signals and buses defined in Signal table are by default created at IP level.
  2. If user does not want any signal defined in Signal table, to be created at IP level, using a property called “export=false/0” can make it happen. 


How to create multiple bus domain in IP-XACT using IDS:


  1. Buses defined in bus domain table, should be defined in "Signals" table with same user defined name


   


If "Bus" defined in "Signal" table is not present in Bus domain table, IDS will generate standard signal interfaces for IP level. The signal interface for "BUS3(WISHBONE)" bus in above example, is shown below-



All signals and bus signals are defined in <spirit:model>. In <spirit:modelParameters>, user can configure the address width and the bus width.
