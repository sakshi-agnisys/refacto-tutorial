Variant in IDS-Word


Variants are used to create multiple outputs from a single specification that contains small variations.


 The variations can be due to any reason, for example:


  • Target device (such as Xilinx, Altera) 
  • Different versions of the product under development 
  • Target audience (for example, some registers are for internal use only)  


The variation could be one of the following:


  • Addition/reduction in registers
  • Changes in access types 
  • Changes to the definition of the registers (fields or description) 
  • Register name change 
  • Offset changes 
  • Changes to any other register property

USE OF VARIANT IN IDS-Excel

Variants can be specified in IDS-Excel in a similar way as IDS-Word. .A single IDS-Excel spreadsheet can be used to generate multiple variations of a specification using the concept of “variants”.


To use “variants” specify all the variants in the IDS_template sheet as below:


   


All the variants are specified under the “Default values” column in the IDS_template sheet separated by a comma or a Alt-Enter (for new lines). The IDS identifier on the row set to “$variant”.

All the specified variants will be available as checkboxes in the IDS config for the user to select single or multiple variants for which the output is to be generated. Note that the order in which these variants appear in the config window is the order in which it is specified in the IDS_template as shown below:



USING VARIANTS 


Variants specified in the IDS_template sheet can be used by either adding a column for the variant or adding it as a property in the description part of the element.


   


  • Multiple variants can be specified for a single element. Multiple variants can be specified either comma separated or a complex Boolean expression evaluation is also possible. When comma-separated, an “OR” between the various variants is implied e.g. v1, v2 means v1 || v2.
  • As illustrated in the figure above for the register “reg_4”. This particular register will be generated in output if the evaluation of the expression “v6&&!v7||!v6&&v7” results to be true.
  • For this register to be present in the output, either variant v6 or variant v7 should be selected in the configuration window.
  • Note that variants can be applied to fields, registers, section, block etc.

Use of Variants in IDS-Batch CLI


With IDS-Batch to assign "true" value to any "variant" inside the input file, pass variants through command line by including "-variant" option followed by comma separated list of variants-


idsbatch <input_file> -out <list of outputs> -variant <list of variants>


Sample run-


idsbatch variantSample.xlsx -out "xrsl verilog uvm" -bus amba -variant "v1,v4"


Note: 

  • For Excel input file, the variants are listed in the IDS_template sheet inside the Design Excel workbook.
  • For CSV input file, to make use of variants, it is necessary to include "IDS_template.csv" or "IDS_template.xlsx" containing variants list, through command line using "-template_file" option.


Example: IDS-Batch variantSample.csv -out "xrsl verilog uvm" -bus amba -template_file "IDS_template.csv" -variant "v1,v4"
