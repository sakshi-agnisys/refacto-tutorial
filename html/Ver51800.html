Ver 5.18.0.0

v5.18.0.0

7/31/2015


IDesignSpec™ (IDS-Word, IDS-Excel, IDS-Batch CLI) Enhancements

  • General Enhancements
    • Parameterized Description - You can now use parameters to add description to IDS templates. For more details please see the Doc property.
    • Block Inside Block - You can now change the block hierarchy. A block can now be placed inside another block at document level. A section can also be configured to behave as block in UVM output.
    • New Properties
      • reg_sw -  This property allows you to specify SW access at register level even if it is already specified on each field. If unspecified, the register access type is used. For more details, please see the reg_sw property.
      • reg_hw -  This property allows you to specify HW access at register level even if it is already specified on each field. If unspecified, the register access type is used. For more details, please see the reg_hw property.
      • reg_default -  This property allows you to specify default value at register even if it is already specified on each field.  If unspecified, the register default value is used. For more details, please see the reg_default property.
    • Property_hint for ref template - The description row of the “ref” template now shows all IDS properties for all templates like chip, block, register, etc. You need to select the property for the referring node manually since IDS does not determine the referred node at the design time. You can add or edit the properties.
    • RoWo registers - You can now use separate templates for read-only or write-only registers in the ROWO pair. The templates can be paired using the rwpair property.



  • RTL Enhancements
    • RTL_Wire feature(Verilog) - RTL_Wire option for Verilog has been made default. Wired data types will generated instead of wor.
    • Paged Register (SystemC) - Paged register made up of a set of registers sharing same physical address are now supported in IDesignSpec™ in synthesizable SystemC RTL. A selector register decides the accessibility.
    • Verilog 2001 - Verilog RTL is now also generated following 2001 standard.
    • Chip level (VHDL alt1) -  Chip level design is now supported in VHDL alt1. 
    • AXI Bus Enhancement (Verilog) -  AMBA-AX14LITE standard is now completely supported in IDesignSpec™.
    • Omitted reduction operator for single bit field (Verilog) - The reduction operator for single bit field in case of hw clear/set pulse has been omitted to remove redundancy. 
    • Counter for SW events (Verilog) -  Counters in Verilog have been enhanced to count read or write events from SW interface alongwith HW events.


  • TCL Enhancements
    • set_property - Command to set/change value to property like name, offset, uvm.class etc. 
    • set_error        - Command that lets you add custom error messages. Various enhancements have been made for TCL API for custom output generation. 
  • UVM Enhancements
    • Memory width - The memory width in specification will now go as it is in the UVM memory configuration. Earlier the width was "ceiled" to the maximum register width in the design.


  • Documentation Output Enhancements
    • Memory width (HTML)- The memory width in specification will now go as it is in the HTML output. Earlier the width was "ceiled" to the maximum register width in the design.


  • Industry Standard Output Enhancements
    • IDS and UserDefined Properties (IPXACT) - All Properties supported by IDS or any user-defined property is mapped inside vendor extension as <spirit:IDS_properties>. For more details click here.
    • Signals and Bus Interfaces (IPXACT) - You can now use Signals table of IDS  to generate bus interfaces and signals interfaces at IP level. 
  • New Outputs
    • System Verilog - A new synthesizable System Verilog output is now generated by IDesignSpec™.
    • SystemC RTL - A new synthesizable SystemC RTL is now generated by IDesignSpec™. 
    • Custom CSV Output - You can now generate a customizable file in CSV format.
    • VHD header in 2008 standard - The VHDL header output is generated compliant to 2008 standard. 


  • New Inputs
    • Custom CSV Import - IDesignSpec™ now support import of a Custom CSV file.


  • IDS-Word Enhancements 
    • Import User Defined Properties -  You can now import user-defined properties in register custom template.
    • Exclude IDS templates - You can now exclude IDS template in the spec by using double forward slashes (//) before the name of the template.
    • Empty Spaces in TOC - You can configure the TOC to show empty spaces in the design address map. 


  • IDS-Excel Enhancements  
    • Multiple Bus Domain template -IDS-Excel now supports bus domain template for multiple buses.
    • Trigger Buffer Register - Trigger buffer register can be implemented using  IDS-Excel.
    • IDS Template Button -  You can now automatically add IDS_template sheet to new Excel workbook by clicking on "IDS Template" button provided in IDesignSpec™ ribbon.

Bug Fixes


The following bugs have been fixed:

    • "uvm.name_format" was not working with reggroup


    • Linter error of "reg_offset = block_offset + 'h4". Linter would complain about the constant specified


    • Wire name for "hready" AHB3AHBLITE signal at chip level was generated wrong.


    • Floating error signal for memory was generated at the block level.


    • 'Ref' within the document was not working with TURBO


    • Null Exception of "", for null offset value was produced in turbo


    • "registered=false" with hdl_path was not working


    • For external module some ports were missing in ARV™


    • IDesignSpec™ used to stop working due to memory leak issue


    • String value parameter override through the ref template was not working for UVM


    • Generation of word document using IDS-Batch window stopped working in some previous version
