TCL based configuration of IDesignSpec™


Over the last decade IDS has grown in its capability. This has largely been because of the customer feedback. Since there are a large number of customers with varied requirements, it has been a challenge to please everyone. A side effect of this approach has been that the product’s command line options, its properties and possible outputs have grown in a rather haphazard manner. We have recently added options for generating widgets from the command line. Constantly users are asking for more ways to customize various aspects of the generation and this will lead to many additional options and properties.

Although this is adding powerful capabilities for the tool, it is also making it difficult to use and even more difficult to get started. Hence it is envisioned to have a Tcl Based Config API so that various aspects of the tool can be controlled via it. The config can be versioned and diffed as well. Standard configs can be created for various standard outputs. E.g. a config for 3rd party outputs rather than a 3rd party output switch.


Its main objective is to provide a convenient way to the customer to check and generate the outputs using the tcl Script.

Usage

The usage syntax of IDSBatch with TCL script is shown below:

 IDSbatch -s <TCL script>

TCL Based IDS configuration API Methods

List of implementing methods that are used in TCL Script:

Note: Symbol [<command>] is used for optional Argument.


    General

  • IDS_read  <input_File> [-s <sheet_name>] [-csv_template <file_name>][-variant <list_of_variants>]: Read the input Files that are used for generating the  output.

-sheet <sheet_name>:  (optional)Specify the sheet name that is required when the input file is excel.

-csv_template <file_name> :(optional) For CSV input - Override the default csv template.

-variant  <list_of_variants>: (optional) Specify or override the variants.

 

  • set_outputs  <list_of_output_formats>  [-blk <object_of_block>] [all_IDS_command] : set the outputs on specific blocks or globally  .

-blk:  (optional) Objects of block that are used to generate the output.

-all_IDS_command: (optional) Supports all IDS command line options

Note :  It supports  all IDS command line options.

             It overrides the generate command list configuration. 

 

  • generate   -output [<list_of_output_formats>][<all_IDS_command>] :  Used for generating the outputs. 

-all_IDS_command: (optional) Supports all IDS command line options

 

  • check: Used for only check the spec.
  • set_traceablity: To enable Traceability .
  • set_multi_out_file < value >: Generate a separate file for each block or chip [Verilog/VHDL/UVM/HTML].
  • reset_config: Reset all config settings that are changed. 
  • set_inp <input_format> : forces the input format to be the specified format. This option is useful when input files have nonstandard extensions.
  • set_vertical_reuse: Used  to a void regeneration of the code. It is mentioned for only C header output (effect can only be seen in uvm output). Vertical reuse will only work for out_third_party option, in case the users don’t wish to generate the UVM model again at board or chip level.

config settings

  • set_config <list_of_prop_and_value>: Used for setting the configuration values. Where <List_of_prop_and_value> List of property and value by using “=” operator. e.g., {repeat=2 external=true}.

          List of config Properties:

       company_name=<company_name>: Specify Company Name.

       copyright=<Copyright Header>: Specify Copyright Header.

       board_size=<value>: Set default board size

chip_size=<value>: Set default chip size

               block_size=<value>: Set default block size

top=<topName>: Name of the top element that needs to be transformed. This option is used for input files where there can be more than one top level element. If there is only one top level element, then that is transformed automatically. Specify chip:<top_elem_name> to make the top level address map as chip and all the underlying address maps as blocks. By default top level address map maps to block and all others maps to regroups

endian= <little|big> : For C header output, if <little> then Little Endian output is generated and if <big> then Big Endian output is generated. If nothing is mentioned then it overrides the setting in the XRSL or ODT file and does not generate the output.

addr_sort: To sort registers in ascending order.

bus=<bus_name>: For RTL (Verilog or VHDL or SV) code output the specified bus interface is generated.

unit=<address unit>: Set  Default address unit.

  • hide_htmlalt2_prop [<List_of_prop>]: Used to hide the property. This method works only on Html alt2 output. By using “-hide_htmlalt2_prop” none or “-hide_htmlalt2_prop “<space separated property list>”” or “-hide_htmlalt2_prop all” switch in the command line, user can hide property English description and shows the property name with their values in description.
  • set_prop  <List_of_prop_and_value> <hierarchy_of_block>: Used for setting the property.   

        <List_of_prop_and_value>: List of property and value by using “=” operator.

                  Ex:  {repeat=2 external=true}

        <hierarchy_of_block>: Used to set the property by using name.

Ex: set_prop repeat=2  block1/reg1/f2

Ex: set_prop widget=apb::ff::apb_widget chip1/block1

 

Command to enable the switches

  • set_switch <list_of_switch> : Used to enable the different types of switches which can affect the  output generation..

Where  <list_of_switch> is a list of different types of switch of specific output.

User passes the switch value(if required) separated with “=” operator eg { third_party_out=d   parity_widget}

E.g.  set_switch { low power third_party_out=d parity_widget no_lint_warn}

where <List_of_prop_and_value> List of property and value by using “=” operator. eg( {repeat=true external=true}).

 

Switch list  for RTL Design

  • low_power: Generate low power rtl design.
  • out_third_party_out=<type> : Support specific output format where type is user format like “d” for duolog
  • in_third_party_in=<type> : Support specific input format  where type is user format like “d” for duolog and “m” for Mentor CSV format .
  • parity_widget: It will remove PPROT & PSTRB ports at TOP (only supported in verilog).
  • no_lint_warn: This option removes IP_ERROR & IP_BE_MASK related warnings for third party output and offsets in case of unaligned memory (only supported in verilog).

Switch for Html and PDF

  • no_formatting: It  does not preserve the formatting of spec description in output for HTML, PDF only text in description is created.

 

Switch for Word

  • docexclude =<list_of_table> :To exclude post-processors (define/Enum/variant) for word output. Where 

                      <list_of_table> :Tables formats can be one of the following types:-

Define, Enum, Variant and UDP (User Defined Property)

Switch list for UVM 

  • hdlpath: Automatically generates hdlpath.
  • coverage: Automatically turn coverage on.
  • illegal_bins: Automatically generates illegal bins in coverage.
  • arv_constraint: Automatically generates constraint from constraint expression.
  • arv_coverage: Automatically generates coverage from constraint expression.
  • arv_assertion: Automatically generates assertion from constraint expression.

Switch list for Verilog

  • rtl_wire: Generates wire data-type instead of wor in case of repeat property on block or reg.
  • bus_port_case=<upper | lower>: Generates Bus ports in uppercase or lowercase format.
  • default_reset=<async | sync>: Generates asynchronous or synchronous reset by default. It overrides the properties in the spec.
  • rtl_no_error: This disables the generation of all the error signals in the IDS block. No error signals are generated by using this command-line option, irrespective of any access..
  • no_external_mask: For turning on and off the external signal IP_BE_MASK in case of third party rtl. The test case must have external registers or memory.

 

Switch for PDF 

  • limit_toc : limit TOC hierarchy in pdf output

 

Switch list for System RDL

  • searchpath=<path>: This IDSBatch switch has been added which will override the default path and will search included rdl files in the directory specified in the search path switch.
  • incdir=<list-of-paths>: This switch is used in case multiple files are included/referred from different directories in another systemRDL file. The relative path of multiple files can be given semi colon separated.

 

Switch list for IP-XACT 

  • ipxact_comp=<block | board>: Output is generated as per the specified Hierarchy for IPXACT taking the top level as board/chip if board is used in the option, otherwise top level is considered as block.
  • bus_comp=<List_of_ files>: specified components files for IP-XACT.

 

 

Switch for C header

 

  • c_type=<value>: For C header output, the specified data type is used in the output.
  • c_tests:  Generates C tests automatically. -out header option is mandatory for getting this.        
  • c_api: Generate C API.
