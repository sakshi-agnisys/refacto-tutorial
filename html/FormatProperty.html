Format Property


In computing, signed/unsigned and fixed point number representation is used widely. A fixed-point number representation is a real data type for a number that has a fixed number of digits after/ before the radix point.

Normally only integers are represented in fixed point numbers. This implicitly means that the decimal point is at the right most corner on number. Also integers can be signed (2’s complement) or unsigned. In Math intensive applications, fixed point representation can also be used to represent floating numbers. In this case the decimal point moves towards left of the number.


A new property is being introduced called “format” to define format definitions of fields, having possible values uint, sint, ufixed, sfixed, used to represent unsigned, signed, unsigned fixed point and signed fixed point fields respectively.

Eg:

  • {format = uint} : unsigned type field
  • {format = sint} : signed type field
  • {format = fixdt(0,,2)} : unsigned fixed point type field with word length (length of field on which format property is defined) and fractional length = 2.
  • {format = fixdt(1,,4)} : signed fixed point type field with word length (length of field on which format property is defined) and fractional length = 4.

In VHDL:


Field type and its corresponding hardware signals (input and output) are defined as unsigned/signed/ufixed/sfixed depending upon the " format type" defined on it, while software side remains in std_logic/std_logic_vector.


Note: If format property is not defined on a field then field type remains to be in std_logic_vector type, as earlier.

Unsigned and signed type are defined in ieee.numeric_std package, while ufixed and sfixed type are defined in ieee.fixed_pkg package.
