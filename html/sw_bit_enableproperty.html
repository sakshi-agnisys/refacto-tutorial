sw_bit_enable property

Bitwise enabling of software data

(compatible with IDS version 6.2.0.0 and above)


The sw_bit_enable property refers to bitwise enable s/w-side data (i.e. ‘wr_data’) to a register (let ‘Data’ reg) or a field of a register (i.e. register bits, either all or some) depending upon another register's (let ‘Enable’ reg) data bits at corresponding locations as previous register. 


This can be done by two ways :-

 

OPTION 1 : Enable1 is the bit-enable register

              Data1 is the data register. Whenever Data1 is written, the new data is written into Data1 for the positions in which Enable1 has a bit set to 1


The equation would be: Data1_Fld_q = (Data1_Fld_q & (~Enable1_Fld_q)) | (wr_data & Enable1_Fld_q)


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View

                           


IDS-NG Spreadsheet View



SystemRDL


property sw_bit_enable  {type = string; component = reg|field ; };


addrmap block1 {


  reg data1 { 

     sw_bit_enable = "Enable1" ;

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } Fld[31:0] = 32'h0;


  };

 

  reg Enable1 { 

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

     

    } Fld[31:0] = 32'h0;


  };

 

  data1 data1 @0x0;


  Enable1 Enable1 @0x4;


};


OPTION 2 : Same as above, but the mask bits are in the same register. The value of the property in this case is only the field name. Half the register is enable bits and half the register is data bits.


Example: Data_Enable_Data_fld_q = (wr_data[15:8] & wr_data[7:0]) | (Data_Enable_Data_Fld_q & (~wr_data[7:0])


                         


Points to be noted :-


  • The Data & Enable bits can be both either registers or fields of equal widths.
  • If Data & Enable bits are from distinct registers then enabling of Data bits will depend upon previous bit values entered in Enable reg.
  • If Data & Enable bits are from same register then enabling of Data bits will depend upon current bit values entered in Enable reg.
  • Within same register field the enable field’s sw access should be ‘ro’ and hw access should be ‘na’ and the value of the property in this case is only the field name.
