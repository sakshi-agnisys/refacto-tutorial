Ver 6.8.0.0

v6.8.0.0 - Major Release

2nd November 2017


Several enhancements have been made based on industry requirements and trends. There has been a greater push to support Vertical reuse of generated code, specially in the SoC context. Two new serial buses for register access have been supported. The tool has been enhanced to work with other EDA tools like Ralgen and Qsys.

  • RTL Enhancements
  • out_third_party - 3rd party compatible output is generated when this command line option is provided. Flattened register code is generated for array of registers and regGroups. (More Details)
  • rtl.reg_enb - This property creates an output signal '<reg>_enb' in VHDL and Verilog in case the register field is RO from HW side. (More Details)
  • lock_r - This property locks the “reads” on the register field in Verilog and VHDL (More Details) 
  • bus_port_case -With this command line option user can change the case of the bus ports in Verilog. (More Details)
  • rtl_no_error - No error signals are generated by using this command line option. (More Details)
  • default_reset - The default reset can be sync or async using this command line option. (More Details)
  • Bit slicing - Range can be given for parameters. (More Details)
  • Passing array via parameter - User can pass array using parameter (More Details)
  • single_address_data_out - By using this property global signals are created for all the blocks in a chip in Verilog (More Details)
  • Alias Register – Now supported in case when buswidth is smaller than the regwidth
  • counter.sw.wr.enb – This property has been enhanced to include a second argument that is used to enable the counter based on an event (rd, wr, enb,intr). (More Details)


  • UVM Enhancements
  • uvm.handle_name_format - This property is used to customize the name and handle name of the class, %p option has been added for taking present node. (More Details)
  • hdl_path_gate – For gate level simulation this property is used. User can specify a pattern based on which gate level hdl_paths will be automatically generated. (More Details)
  • UVM access(ro, wo, rw) – Registers are configured according to the access of the field instead of just ‘rw’ earlier


  • ARV™ Enhancements
  • Multiple soft reset - The property ‘resetsignal’ is now supported in ARV™ (More Details)
  • Xcelium MakeFile - Cadence Xcelium is now supported in ARV™ (More Details)


  • SystemC Enhancements
  • SystemC duolog – Duolog format is now supported. (More Details)
  • Synthesizable SystemC – Synthesizable SystemC is now supported in IDS at chip level (More Details)
  • SV Header Enhancements

       sv_param – This property creates parameter inside the package instead of `define (More Details)

  • General Enhancements
  • AXI4FULL - AXI Bus has been extended to AXI4FULL which supports bursting (More Details)
  • User defined I/O ports and bus interfaces are imported from excel to IP-XACT(1.5)
  • unaligned_error- If this property is used then error response is created if addresses are unaligned(e.g. 2LSBs of 32bits address are non-zero) (More Details)


  • Industry Standards 
  • SPI Bus - SPI Bus is supported in IDS (More Details)
  • I2C Bus - I2C Bus is supported in IDS with its 7-bit addressing mode (More Details)
  • Ralgen: IP-XACT is now compatible with Ralgen for generating UVM model using property snps_ralgen. (More Details)
  • Third Party Interfaces 
  • Qsys Integration - Qsys can be integrated with IDS (More Details)
  • Bug Fixes:
  • Optimize for low power- Low power RTL code is generated for registers having larger regwidth.
  • Multi-Out- Using this option, the filename, module name and instance name are replaced by the name of top element of the referred block instead of ref name.
  • Indirect Increment Register-address correctly incrementing now
  • PDF table of contents generation issue related to Multiple Bus domains
  • Multiple Bus Domain- It works in case of any Bus name provided(eg:amba,amba1,amba2)
  • ARV™ for AXI Bus minor bug fixes
  • IP-XACT fix for large addresses which use to turn into an exponential value, also  placeholder field generated where no field is specified. 
  • Cadence makefile in ARV™ bug fix for supporting coverage
  • In ref cases, wrong addresses were created in case the ref file contains discontinues addresses with spaces when first time the user runs check in IDS-Word.
  • Parameterizing in memory- UVM output now include all the parameters declared for a memory.
  • Supported rtl.hw_set and rtl.hw_clear with interrupts(intr.status).
