Python As An Output

To generate Python HAL, user can use command line switch 'python_api' in IDS-Batch.


For register's read and write function, user can apply "py_write_template" and "py_read_template" in the "-top_Property" command line option.

Usage: -top_property "py_write_template=f:write_field(%im, %o,%d);py_read_template=f:read_field(%m, %o);"


       %m :-   For mask

       %im :-  For inverted mask

       %o :-   For offset

       %d :-   For data

Support for module name & dynamic assignments

A new property "top_display_name" has been introduced for the support of module name and dynamic assignments in python output along with the removal of duplicates of addresses, offset. By using the top_display_name we can flat all the instances in case of repeat.

Example: IDS-NG    IDS-Word   IDS-Excel   SystemRDL

IDS-NG Register View:



IDS-NG Spreadsheet View:



SystemRDL Input:


property chip { type=boolean; component=addrmap;};

reg regA{

  field{}F[31:0]=0;

};

regfile sectionA{

  regA regA[2];

};

addrmap blockA{

  sectionA sectionA[3];

};

addrmap child_chip{

  chip=true;

  blockA blockA[4];

blockA[3].sectionA[2].regA[1].F -> reset=32'hA;

};


Command Line : idsbatch <input_file> -output python -top_property "top_display_name=true"
