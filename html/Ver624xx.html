Ver 6.24.x.x

v6.24.0.0

(Aug 7th,2019)


IDesignSpec™(IDS)

RTL Enhancements:


  • Hierarchical Decode is now supported in Verilog output.The address bits are divided into two parts, the MSB part which is used to decode the block and the LSB part, which is used to decode the register being addressed, once the block address (the MSB part) is selected. More Details
  • Clock gating with user-defined clock gating module/logic is now supported in Verilog. More Details
  • User can now rename interrupt per channel output using property, “intr.out”.
  • “we” property is enhanced with two arguments, i.e., first is the signal or the field on which wait event will occur and second will be the value with which register’s field or any signal value should be updated. And, “we” can now be applied on registers as well. More Details
  • New property,“sv_interface=true” has been implemented for generating interfaces of regroups in SV output. More Details
  • Parity is now supported with repeat on the parity bit which is internally stored in the RTL. More Details
  • User can now inject error in parity as well. More Details
  • A new property, “ext_error=true” has been implemented to create a port for error from external. More Details
  • Tile-Link bus is now supported with '-fast' switch. More Details


SystemRDL Enhancements:


  • -rdl_version 2a: A new switch has been introduced for using a strong new SystemRDL compiler, which is compliant to SystemRDL 2.0 and error messaging has been improved in it. More Details
  • Stride operator with the symbol ‘+=’ is now supported in SystemRDL input. More Details
  • Stride property has been supported in all other input formats apart from SystemRDL (as it takes the symbol %=) and reflected in corresponding outputs: Verilog, UVM, Cheader and htmlalt2. This property is used to control the spacing between two instances of an array of components. More Details
  • Property ‘addressing=fullalign’ is now supported which is similar to regalign addressing mode except that it works on array components. More Details
  • Property ‘ispresent’ is now supported for Verilog, UVM and headeralt1.This property causes the given component instance to be removed or exist from the final specification based on a parameter value.  More Details


Following properties are supported with ‘-fast’ switch :


  • The property ‘anded’, ‘ored’ and ‘xored’ are now supported for logical ‘and’, ‘or’ and ‘xor’ of all the bits in the field respectively. More Details
  • ‘hwenable’ is now supported that determines which bits may be updated after any write enable. Bits that are set to 1 will be updated. More Details
  • ‘hwmask’ is now supported that determines which bits may be updated after any write enable. Bits that are set to 1 will not be updated. More Details



UVM Enhancement:


  • Index register is now supported at section level.


CHeader Enhancements:


  • Directory structure for header files in case of ‘c_api’ has been altered to generate two separate files as : <file_name>.c and <file_name>.h
  • All header files that declare functions will now be generated with C++ guard bands. 
  • A new property, “rsv_name_format=<string>%h%l%o”, has been implemented to change the names of reserved field names in C-header alt1 output. More Details
  • Following new #define have been added in ‘headeralt1’ output :
      • <field_name>_INV_MASK -> 1’s complement of the _MASK 
      • <field_name>_VALUE_MASK -> normalized mask, or shifted right mask 
      • <field_name>_INV_VALUE_MASK -> 1’s compliment of _VALUE_MASK 
      • <field_name>_SIZE -> Size of the field
      • <field_name>_DEFAULT -> Default value of the field



General Enhancements:


  • Supported “html_reglink=true” property to display register’s name instead of register’s index number for htmlalt2 output. On command line, it can be used as, “-top_property “html_reglink=true””. More Details
  • Property "expanded_toc=true” is implemented to expand arrayed blocks in top-level map for htmlalt2 output. The table of contents will be expanded using this property. More Details
  • A new property, “data_struct=package/dump”, has been implemented for generating a perl package for Perl output. More Details
  • User can add “html_header=<tag_as_string>” property in third party output to support extra html tags given by the user in the heading of htmlalt2 output. More Details
  • All the Bus Reports with Functional coverage and code coverage have been added and available for download now. More Details
  • Presentation has been added for better understanding of Velocity template. More Details


IDSCal(Version 2.6)-


  • In configuration window, a text box for ‘Return Type’ is added for UVM and C output of sequences to alter the return data type of function. More Details
  • Added ‘command line option’ text box in Configuration window to add additional IDS-Batch commands. More Details



Bug fixes:


RTL-

  • Bug fixes done for interrupt when register width is greater than bus width.
  • Fixed “rtl.hw_enb” property for register width greater than bus width.
  • Fixes done for “rtl.wp” with “soft_reset”.
  • Bug Fixed for counter with “rtl.hw_clear”, “soft_reset” and “hard_reset” property.
  • Bug fixes done for decode logic when there is repeat on external register.
  • Fixes done for aligned reg with repeat.
  • Fix in decode logic: Decode will be high in accordance to the address.


UVM-

  • hdl_path and hdl_path_internal property has been fixed in case of sectional hierarchy.
  • Property ‘is_rsv' is fixed in UVM output when its value is set to false.


Cheader-

  • Bug fixed for ‘read’ and ‘write’ function in C-API file. The lowest level register accesses are not implemented in api files since environment itself has its own implementation ways of accessing registers. 
  • Bug fixes done for hdl_path in case of deep hierarchy.
  • Fillers made for empty spaces in ‘header’ output would now be made according to their position of occurrence. 


IDSCal-

  • Fix import function issues for IP-XACT.



Automatic Register Verification (ARV™)


ARV™-Formal Enhancement:


  • Wide register and regalign are now supported. More Details
  • AXI bus is now supported.
  • RTL properties related to special control of registers, reset signal, shadow and lock are supported. More Details


ARV™-Sim Enhancement:


  • Special registers are now supported with special test and  coverage.
  • AXI4FULL is now supported.


Bug fixes:


ARV™-Formal-

  • Bug fix in reset signal property.



IDS-NG

(v1.8.3.4)


Enhancements:


  • User can edit, delete and add registers through button. More Details
  • When the file is updated in Param view, the previous changes will be shown in Param diff viewer, and if  the diffs were checked then the changes will occur in updated IP also. More Details
  • Registers can now be added or deleted from the param view. More Details
  • Copy and paste feature is supported for fields in param view.
  • Configuration for each document could be created separately.
  • “STOP”  button has been added for check and generate. Now, user can stop process for check and generate. More Details



Bug Fixes:


  • Param import fill all read/write access  for the all fields that is customizable. 
  • Bug fixes done for the ‘param update’.
  • Bug fixes done for skipping new line character in param description.
  • Fields could be copied from one location to the other.
  • In param import, if parameter value is not given then it is imported with default value.
  • After import param on a file, all “Bus” fields are created as registers with their respective names.
  • Memory dump output will generate correctly even if there is any empty space in the address range.
  • Bug fixes done for undo feature “Ctrl+Z”. Now, it works with increased depth as well.
  • Refresh button (from right click menu), does keep the same position within the document.
  • While switching from one view (Register view) to another view (Param View) and returning to the former view, the position of the pointer does not change.
  • It is currently  possible to do modifications in param view, if there is any empty locations available to move fields around.




IDesignSpec™ (ISS)

Enhancements:


  • A new output ‘Perspec’ has been added. More Details
  • Text based format (Python) is supported as input. More Details
  • To handle parallelism, following constructs are added in UVM:
        • fork_join More Details
        • fork_join_none More Details
        • fork_join_any More Details
  • Following two new commands have been added in the library of the tool:
        • return More Details
        • assert More Details 
  • User can combine variables and pass them as value to the read/write transactions by using keyword ‘concat(<variable_1>, <variable_2>)’ in UVM output. More Details
  • User can now access child sequence member in top sequence. More Details
  • User can now use signals/interfaces to read and write from ISS to DUT and vice-versa. For this, the  placeholder structure for signal has been made in IDS header output. More Details
  • User can alter the return type of function in C and UVM output. More Details
  • In C output, ‘wait’ command generates a while loop, which holds the execution till the required time is achieved. More Details 
  • Enums, defines and params are now supported for C output.
  • Randomization with and without constraints are supported in UVM output for both register/field and variables. More Details
  • Functionality for getting the current time of execution has been added in C output using ‘time ()’ keyword. More Details
  • Directory structure of Firmware output is altered for ‘multi out’ option as well, four files will now be generated as: <seq_name>.c, <seq_name>.h , <register_model>.h  and <seq_pkg>.h files.
  • Hierarchical sequences are supported in UVM output. More Details
  • Array of arguments are supported in C and UVM output. More Details
  • Structures and arrays can be passed as an argument to subsequence call. More Details



Bug Fixes:


  • Bug fixed for register writes, when structure is passed as value.
  • Bug fixed for multiple call of a same subsequence with different arguments.
  • Fixes done for max nesting when ‘Sequence Index’ is used in C and UVM output.
  • Fixed duplication of code when subsequence is called inside loop or conditional statement for UVM output.
  • Bug fixes done for C output when multiple ip level sequence are called inside top sequence.
  • Bug fixed for ‘format’ property on negative values in case of C output.
  • Bug fixed when constants are passed as an argument to subsequence call in UVM output.
  • Annotation check on variables with same name as registers/ sections/ blocks.
  • Bug fixed for use of Macros in C and UVM output.
  • Fixes done for external function call in C output.
