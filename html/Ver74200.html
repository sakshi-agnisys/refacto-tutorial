Ver 7.42.0.0

v7.42.0.0

(June 3rd, 2022)


IDesignSpec™ (IDS)


RTL Enhancements

  1. B#1776 - Support of packed array in RTL output using -rtl_array “pack” switch.More Details
  2. F#20011 - Support of re-usability of structs in SystemVerilog output when multiple instances are created of the same address map using property “rtl_single_package=true”.More Details
  3. F#16618 - Support of decoding of chip components inside the multicast/broadcast region in case of hierarchical decode.More Details


General Enhancements

  1. B#1777 - Support for generation of separate files for regfiles/section/reggroup with the help of switch -top_property “uvm_opt=8”.More Details
  2. B#1778 - Support for block arrays expansion in the HTML output while using display_name= “name_%d” property along with repeat property for the generation.More Details
  1. B#1779 - Support of ipxact_decrease property which decreases the offset value in IP-XACT file and is referred to in any input type. More Details
  2. F#20038 - Supported HTML anchor link in description in htmlalt2 and pdfalt4 output.More Details
  1. F#20213 - Support of the property “cheader_macros_opt” which generates two macros that helps in calculating size of the component.More details
  2. F#20212 - Support of mixed data types in headeralt1 output. More details
  3. F#20282 - Support for addition of stdint.h header in all tool generated .h and .c files.More Details

 

Bug Fixes


RTL


  1. B#1780 - Fix in the generation of RTL when string value (like any register’s field (regA.F1)) or any string is given to the incrthreshold property dynamically in RDL.
  1. B#1790 - Removal of extra hardware “in_enb” signal when“registered=false:nomux” property is applied on register.
  1. F#20351 - Fix in case of overriding ‘define values defined inside the register specifications via command line switch switch ‘-D ' <define_name>=<updated_value> ‘  at run time.
  2. B#1747 - Added an error message on usage of ‘rtl_hw_enb=false’ property and ‘intr.status’ property in the same register.
  3. B#1748 - Added an error message on illegal combination usage of buffer_trig UDP, the ‘rtl_hw_enb=false’ and  ‘rtl_precedence=sw’.
  4. F#16704 - Fix for extended lock expressions with custom strings as its key value.
  5. F#19754 - Fix in assignment of multiple default values in the RTL for a field inside a repeated register/section parent.
  6. F#20369 - Fixed issues on using SW side CDC along with interrupt properties together in a register.
  7. F#20411 - Fix in the axi_widget issues:
    1. change in rdata when rready signal is not high.
    2. Due to rd_stb, rvalid de-asserts even when rready signal is not high.
  8. F#20150 - Fixed issue in the parity property when “is_rsv” property is applied on the field.
  9. F#20249 - Fixed the missing block name in the external_ack widget instance name when “rtl.name_format=%m%r” is applied on the block.
  10. F#16456 - Fixed the issue with parity property when multiple fields present on the register have hw access “rw”.


UVM


  1. F#19754 - Fix in RAL in case of assignment of multiple default values in a field inside a repeated register/section parent.
  2. F#15957 - Fix in coverage sampling in case of cross between interrupt registers with special interrupt scenario where the pending register clears the status register on writing one to it.
  3. F#20520 - Fix in the hdl path issue in case of hdl_path property applied on field then automatic hdl path and property value were coming together.
  4. F#20507 - Fix the hierarchies issue for coverage in UVM RAL.
  5. F#20040 - Bug fixed in HDL_PATH when HDL_PATH applied dynamically on the repeated section.


General 


  1. F#20201 - Repeated component is not working in the IP-XACT output in case of chip-in-chip fixed.
  2. F#20371 - Fixed the overlap issue in case of an empty address block in IP-XACT input.
  3. F#20382 - Issue fixed for address calculation in IP-XACT output in case of chip in chip.
  4. F#20357 -  Removal Of “_s” from the naming of struct/define from top_address.h file when ‘cheader_name_format’ property is not used.
  5. F#20261 - Extra file is generated in case when inst_name_cppstyle property is applied in chip-in-chip scenario.
  6. F#16211 - Fixed the preserve line break used in the descriptions of source SystemRDL for ipxact2014 output.
  7. F#20094 - Fixed “not_generate” property when not_generate property is applied on chip in case of chip-in-chip.

  6. G#1667 - Fixed the issue of giving the same variables name or any undefined variables in a file by generating a GUI error pop-up window.
