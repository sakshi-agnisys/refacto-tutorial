Variant Dependent Properties

(Compatible in IDS version 5.14.18 and above)


Variants in IDS can now be used to create variation on various properties in a single specification.


Syntax to use the variants in various properties- $variant_name


Example:

      • clock_edge=$variant1?posedge:negedge
      • reset_type=$isAsync?sync:async



variant1, isAsync, v1 and v2 are variants


Note: 

  • Value to following properties cannot be made variant dependent-
      • repeat
      • field defaults
      • offset

In IDS-Batch


To pass variants through command line include "-variant" option followed by comma separated list of variants-


IDSbatch <input_file> -out <list of outputs> -variant <list of variants>


Sample run-


IDSbatch variantSample.xlsx -out "xrsl verilog uvm" -bus amba -variant "v1,v4"



Usage of Variants for Register Description


"-variant" command-line option has been enhanced for disabling the variant values. We can also use variants to enable or disable the register.


Command Line: 


IDS-Batch <input.rdl> -out htmlalt2 -dir <dir name> -variant <v1/!v1>


SystemRDL: 


// User defined properties: direction

property direction {type = string; component = signal ; };

property doc { type = string ; component = addrmap | reg | field ; };


// Address Map 

addrmap BlockA #(longint unsigned REGE_REP_NUM = 32) {

// Register definition BEGIN

   reg reg1 {

      doc = "$v1 ?\" this is description for reg1 \" : \"this is description for reg2 \"";

        field {

          doc = "$v1 ?\" this is description for field1 \" : \"this is description for field2\"";

          hw = rw;

          sw = na;

        }f1[31:0]=3;

   };

   reg1 reg1 @0x220;

};
