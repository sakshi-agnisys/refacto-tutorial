IP-XACT Chip in Chip Hierarchy flow

This Application note describes the proposal to support chip in chip hierarchy in IP-XACT output. Since, the IP-XACT does not support the chip in chip flow conventionally, we will create the hierarchy by using nested registerFile tags in IP-XACT. This IP-XACT output when taken as input in IDS-Batch, will create the same hierarchy according to the original input specification.


The following rules will be applicable when IP-XACT will be processed by IDS-Batch:

  1. The registerFile with chip=true in vendorExtensions will be treated as a chip component.
  2. Within a nested registerFile hierarchy, the first registerFile just after the chip=true registerFile will be treated as a block component.
  3. Consecutive registerFile after the block components will be treated as reg-groups. 


Please refer the following example:


SystemRDL Example:


addrmap top{

chip=true;

addrmap b1{

chip=true;

addrmap b2{

chip=true;

addrmap b3{

reg reg1{

field {}f1;

} reg1 ;

}b3;

}b2 ;

}b1 ;

};
