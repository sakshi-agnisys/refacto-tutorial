Define

In IDS a define is used to name arbitrary integer values. Any number of identifiers are used to name arbitrary integer values. The IDS template for the define is as shown below:


IDS-Word


 


IDS-Excel



IDS_TEMPLATE


IDS-Excel uses a default template for register data specification. The "User header" in the template file are used as column headings while creating the specification. However, this template can be changed by the user ( click here for details). Below is the part of this default template for using "defines" in IDS-Excel.




The properties that are supported by the IDS define template are explained below:



Property

Description

define name

A unique string that identifies the name of the define name.


  Example : NUM_PORTS

value

The value of the define is specified in this field. 


Example : 'h16 , 2000

description

This field is used to give the detail description if required for the particular define.


Example : Number of ports

private

This field is used to modify the access scope of identifiers. By default all identifiers are public in nature. Assigning value '1' make them private to that particular design module, else these can be overridden.


Possible values for this field are 0 or 1. 0 is for public and 1 for private access.


The IDS generates the appropriate code for these defines accordingly.
