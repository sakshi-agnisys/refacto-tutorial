Ver 7.12.0.0

v7.12.0.0

(May 6th, 2021)


IDesignSpec™ (IDS)

RTL Enhancements

  1. “hierarchical_decode” property has been enhanced to support decoding of block IPs in a multicast/broadcast manner within an address map. (More Details)
  2. “tmr_error” property has been enhanced to take a third argument 'activehigh' to make the tmr_error signal as active high. (More Details) 
  1. “registered” property has been enhanced with a new argument ‘retain’ (“registered=false:retain”), to retain the storage value for unregistered register/field. (More Details) 
  1. CDC feature is supported in the Tilelink bus interface. (More Details) 


UVM Enhancements


  1. “cross” property has been supported with variants for UVM RAL output. (More Details)
  1. “uvm.block_class” property has been introduced to extend the chip and block classes from the class specified as value of this property. (More Details)
  2. “uvm.regfile_class” property is used to extend the regfile classes with the class mentioned as the value of this property.  (More Details)

General Enhancements

  1. “rtl_wrapper=true” property is supported in Specta-AV™ and SV for chip level RTL. (More Details)
  2. “alignment” property has been supported for alignment of all the components inside the associated chip, block and section. (More Details) 


Note :

  1. If “-no_warn” switch is used, then input file name should be given before the switch in the IDS-Batch command line, preferably switch should be applied at the end of the command line. 
  2. Variants have been enhanced to be case sensitive.

Bug Fixes

RTL

  1. Fix for lint errors regarding operand bit-width mismatch in Verilog output.
  1. Fix for read operation in case of page register when different selector registers are used from hardware and software side. 
  1. Fix for missing CDC ports instantiation in case of AMBA-AXI4Full bus.
  2. Fix in “counter.incr.ral = <signal_value>” property to support the incr value of counter based upon the input signal pin.
  3. Checks are added to depict that “lowpower” switch and “rtl.byte_enable=false” are mutually exclusive.
  1. Fix for null pointer issue in case of nested external sections for Verilog output.
  2. Fix in Verilog output when the "-rtl_wire" switch is used with chip-in-chip flow.


SystemVerilog


  1. Fix for tmr error when there is repeat on register/section.
  1. Fix for issue in chip-in-chip flow when an SV wrapper is missing.


SystemRDL


  1. Fix for naming issue of iterator when dynamic reset array assignment is used in Verilog output.
  2. Fix for SystemRDL Compiler 2.0 freeze issue when RDLFormatcode closing tag is missing.
  3. Fix for SystemRDL Code Formatting issue for documentation output.
  1. Fix for issue in SystemRDL compiler regarding the nested preprocessor directives.


UVM


  1. Fix for callback registration order for pending register. 
  2. “Interrupt_monitor” will only generate in case of “intr.overflow”.
  1. Fix for issues in UVM output for "uvm_class_name_inst=true" property when referring to an external block inside the chip in case of chip-in-chip flow.


C Header


  1. Fix for naming issue in case of deep hierarchy of sections when “cheader.name_format” property is applied on the top.
  2. Fix for duplicate filler generation issue in C header output.
  1. Fix for issue in C Header output for "cheader.name_format" UDP in case of deep reggroup hierarchy.


General


  1. Fix for “dir_out_specific” switch for misrac, uvm, xrsl and autoseq output.
  2. Fix for all properties which takes variant as an argument to the value of property defined in the spec.
  3. Following fixes have been done while importing an IP-XACT file :
  1. Fix when register size is less than address unit
  2. Fix for number format
  3. Fix for field name
  4. Fix for RW register pair
  1. Fix for description issue for IDS-Excel input when description has extra curl ({) braces.
  2. Fix for markdown output to convert “\n”  into <br>.
  3. Fix for PDF-Alt4 layout issue in case of register table.
  4. Fix for IDS-Excel export configuration issue in case of multiout.
  5. Fix for address overlap issue in annotation check in case of chip-in-chip flow. 
  1. Improvement in generation of C-APIs using “-fast_capi” switch.

ISequenceSpec™ (ISS)


Enhancements


1. Following enhancements have been made in ISS uvm and firmware output :

    1. Loading Memory from a text file (More Details)
    2. Arguments and variables can be defined as two dimensional arrays, and can be further used as arguments/variables to function calls (More Details)

2. Two new outputs are supported in ISS :

 1. CMSIS : Users can specify ARM device drivers in ISS & can generate CMSIS driver packages which can be imported in Keil(R) software using “iss_cmsis” switch. (More Details)

2. MISRA C : Users can generate the MISRA C Compliant Firmware using “-iss_misrac” switch. (More Details)

3. Chip inside Chip is supported in the ISS. (More Details)

   


Bug Fixes

  1. Fix for undefined variables that are created when reading registers in case of chip with repeated block and register.
  2. Fix for one-dimensional arrays in case of function and argument.
  3. Fix for part select of the argument in ISS firmware and ISS uvm output.


ARV™

Enhancements


  1. “arv_report” switch has been introduced to generate the HTML reports for the coverage in ARV™-Sim.


Specta-AV™

Enhancements

  1. Chip Level scenario is supported in Specta-AV™. (More Details) 
  2. Two Dimensional variables are supported in checkers. (More Details)

Bug Fixes

  1. Fix for the register name in “tapper” interface file.


IDS-NG™


Enhancements

  1. Supported RDLEditor 2.0 in IDS-NG. (More Details)
