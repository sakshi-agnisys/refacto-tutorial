Altering name case formats



This part of Section contains IDS-Batch Switches for generating port signals in Upper or lower case formats. 


Register Port Case:


In IDesignSpec™ creation of ports/nets (in verilog) are in accordance with the input specification but users could convert the ports and signals name case to upper or lower using the switch “-reg_port_case” by providing argument. Possible Argument of Switch are - upper/lower


This switch can be added with IDS-Batch command line to generate various output. 


Example:

             1)To  generate signals in Lower case format.

Command Line: IDS-Batch <input_file> -bus <bus_name> -out <output_files> -dir <dir_name> -reg_port_case lower

             2)To generate signals in Upper case fromat.

Command line: IDS-Batch <input_file> -bus <bus_name> -out <output_files> -dir <dir_name> -reg_port_case upper

Support of the -reg_port_case switch in UVM


In IDesignSpec, the creation of ports/nets (in Verilog) is in accordance with the input specification, but users could convert the port and signal names to upper or lower case using the switch “-reg_port_case” by providing an argument. Possible arguments for switching are upper/lower. The switch "-reg_port_case" also changes the upper and lower of the <upper/lower> _q register with port. 

In order to enable the use of the -reg_port_case property in UVM, the IDS HDL path must be adjusted accordingly.

To align with the IDS implementation of upper and lower case register ports, including <upper/lower register name>_q in Verilog, it is necessary for IDS to modify the HDL path accordingly. This modification will enable IDS to correctly handle and support upper and lower case register ports, ensuring proper functionality and compatibility with the desired register naming convention. 


Command Line:


idsbatch <inp_file> -reg_port_case -dir ids

};


Generated UVM Output


virtual function void build();

        //define default map and add reg/regfiles

        default_map= create_map("default_map", 'h0, 4, UVM_BIG_ENDIAN, 1);


        //REG_N1

        Reg_n1 = block_name_Reg_n1::type_id::create("Reg_n1");

        Reg_n1.configure(this, null, "Reg_n1");

        Reg_n1.build();

        default_map.add_reg( Reg_n1, 'h0, "RW");


        //REG_N2

        Reg_n2 = block_name_Reg_n2::type_id::create("Reg_n2");

        Reg_n2.configure(this, null, "Reg_n2");

        Reg_n2.build();

        default_map.add_reg( Reg_n2, 'h4, "RW");


        if(Reg_n1.has_hdl_path()) begin

            Reg_n1.clear_hdl_path();

        end

        Reg_n1.add_hdl_path_slice("REG_N1_FLD_q", 0, 32);

        if(Reg_n2.has_hdl_path()) begin

            Reg_n2.clear_hdl_path();

        end

        Reg_n2.add_hdl_path_slice("REG_N2_FLD2_q", 16, 16);

        Reg_n2.add_hdl_path_slice("REG_N2_FLD3_q", 0, 16);


        lock_model();

    endfunction


NOTE:

  • This Switch does not work for IDSng input format. 
  • Component name should be unique while using this switch to avoid conflict, i.e. working of this switch is case insensitive, hence capital and small letter format for same name will be consider as one. For eg: "reg2" and "REG2" will be considered same.
  • For customizing the name of the ports and signals, please refer here (more details).
