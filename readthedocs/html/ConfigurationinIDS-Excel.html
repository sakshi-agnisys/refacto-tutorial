Configuration in IDS-Excel

Configuration Settings in IDS-Excel


The input specification in languages like behavioral VHDL, algorithmic SystemC, C++ etc and the RTL description in VHDL/Verilog is produced as the result of synthesis. 


The Configure button can be used to make settings according to requirement, and choose the kind of files that need to be generated. Make the appropriate settings in the Configure window. 







  • Click on the required check boxes from Options available. 
  • Click OK to apply the chosen options.


Configuration Options

  • Global


 General Window 




Options

Description

Company

Specify the company name that is to be included in the headers of various outputs.

Copyright

Specify the copyright information to be included in the headers of various outputs. 

Save Document

This option makes IDS automatically save the document after Check and/or Generate.

Force

Forcefully generate xrsl every time on Check/ Generate for ref.



  • Outputs


User can select the outputs which they desire to generate under the Outputs tab. 


 Outputs Selecting Window 




Options

Description

RTL

Designs are written in an HDL at a register Transfer Level RTL. The term RTL is used for an HDL description style. Verilog, VHDL, and SystemC are the most popular HDLs.

Verification

Verification basically Test Benches consists of drivers and monitors that drive the stimuli into DUT and analyze the output of DUT against expected output. Test Bench enables verification of the design intent. This tool helps in development of test bench in UVM, OVM and VMM environment.

Header/Classes

This tool can generate header files for C++ and SystemVerilog, which helps the firmware, software and diagnostic teams to create their deliverables.

Documentation

IDesignSpec™ generates HTML 4.0 and PDF documentation that can be shared on the web. It can also generate XML output which can be used to interface with other tools.It also generates the graphical view of the specification in the form of Scalable Vector Graphics (SVG).

Standards

Standards like IP-XACT, define and describe electronic components and their design that enables automated configuration and integration through tools. It ensures delivery of compatible intellectual property descriptions from multiple intellectual property vendors.

Sequence

Sequences enables users to document the sequences and create outputs suitable for various 

teams such as: Design, Verification, Firmware and Software. Having the same Spec drive outputs for the various teams,

 has the same impact as creating common register spec and sharing that with the teams 

does. In other words, all teams are automatically synchronized, errors are minimized which leads to 

higher quality and improved time to market.

Output Directory

Specify the user defined output directory.



  • User Defined Outputs


The user can select their custom outputs generated using the TCL-API under the ― "User Defined Outputs" tab.


 User-Defined Output Settings Window 




Options

Description

IDS_XFORM_PATH

Displays the transform path specified by user using the IDS_XFORM_PATH environment variable

Local Path

In addition to transform path specified by IDS_XFORM_PATH user can also select the local trans-formation directory. The advantage of Local transformation directory over the one specified using the IDS_XFORM_PATH is that user does not have to restart the instances of MS-Word for the effects to take place.

Local Path Outputs

All the user specific outputs generated using the IDS TCL-API available in the Local Path appears in this window.

IDS_XFORM_PATH Outputs

All the user specific outputs generated using the IDS TCL-API available in the directory pointed by IDS_XFORM_PATH appears in this window.



  • Settings






Options

Description

Address Units

Defines the address unit to be used for displaying all the addresses and offset in the specification. Must be a power of 2.

Reg/Bus Width

Defines the default register/bus width. User can specify its own reg width. Must be a multiple of 8.

Block Size 

Specify the Block size.

Chip Size

Specify the Chip size. Must be greater than or equal to Block Size.

Board Size

Specify the Board size. Must be greater than or equal to chip Size.

C Type

Default C type name for registers in header files. User can specify C type formats as uint_%d_t.

Little Endian

Generates C header outputs according to Little Endian configuration

Big Endian

Generates C header outputs according to Big Endian configuration


Note: When both Little Endian and Big Endian checkboxes are selected together then output is generated with macros corresponding to both endianess.

  • Variants Settings


Variants specified in the specification appear in this window. Enables the user to select single/multiple variants. 


Variants Setting Window


Note: This feature is now compatible with IDS-Excel and prior to 5.0.0.0 versions of IDS-Word. In new IDS-Word versions variants are enabled via variant's template itself. Refer to section Specification Creation -> Hierarchical Design ->  Variants for more details.


  • Formatting



Formatting Window



Options

Description

Index

Select the checkbox to use the IDS indexing

StartingIndex

Specify the starting index of the top level IDS template. By Default it is set to 1 and a few options are available as drop down. User can also specify the indexes other than the available in the drop down. For example 4.1.1.1, this will set the index of the top IDS element as 4.1.1.1 and its first child‘s index as 4.1.1.2 and so on.

Heading 

Select the checkbox to use the IDS Headings

Starting Heading

Specify the starting heading of the top level IDS template. By default it is set to "heading 1" and a few options are available as drop down. User can also specify the headings other that the available in the drop down.

Preserve Names

Select the checkbox if user wishes to keep the names specified in the documents to be unchanged at output level. Otherwise all the spaces and underscores are removed by IDS in outputs.

Address Sorting

Select the checkbox if user wishes to place register in any order irrespective of address/offset.

  • Advanced Verification


The IDS generated UVM reg model needs additional info like constraint on values, coverage blocks, HDL paths for back-door access etc. for effective verification. IDS in addition to getting this information using properties/syntax's, provides a way to automatically generate the said features using Advanced verification settings.



Advanced Verification Settings




The above options can be selected accordingly, on basis of what or what not to get.


  • Advanced Design




Options

Description

Memory Technology Mapping

Select the checkbox to enable Memory Technology Mapping feature

Optimize for Low Power

User can optimize the design for low power RTL by selecting this checkbox 

Multiple bus Domain

Select the checkbox to generate the design code for multiple bus domains

Interrupts

Select the checkbox to enable interrupts for design.

Special Reg

Select the checkbox to generate trigger buffer register.



User can reset the configuration settings to default by clicking reset button. The configuration can be exported or imported in the form of XML using Export and Import buttons.
