(Nov 30th, 2023)


IDesignSpec™ (IDS)


RTL Enhancement


  1. B#2528: Support for the property “bus_parity_e2e” for interaction between bus_parity and the reg parity. More Details


UVM Enhancement


  1. F#23206: Support for the removal of the parity callback class using the “ignore_prop” property. More Details


General Enhancement


  1. F#22344/F#23196: Support for inline enums for register and field for YAML input. More Details
  2. B#2659: Support for Python output implementation in java and making it equivalent to the XSLT version. More Details
  3. B#2540: Support for default properties json file having all properties used in testcase. More Details
  4. B#2703: Support for improvements in existing property not_generate by adding all for UVM and cheader. More Details


Bug Fixes


General


  1. B#2698: Fix for flowchart issue for IF/ELSE, IF/WAIT statement & WHILE loop.
  2. F#23046: Fix for the issue of value calculation of stride in case of decimal to hexadecimal conversion for excel\word\idsng\rdl inputs in IDS-Batch.
  3. F#23257/F#23236: Fix for linkable TOC support for IDS-Word output.
  4. F#23213: Fix for Japanese language in IDS-WORD description.
  5. F#23212: Fix for Japanese language in Htmlalt2 output description.
  6. B#2668: Fix for the issue while importing a file, the output is generated in the installation directory instead of the current working dir in IDS-NG.
  7. B#23148: Fix for the issue of using a variant with a description in RDL input for HTMLalt2, PDFalt4 and IP-XACT output. 
  8. B#2702: Fix for the issue related to default value when it is unknown in the word file.
  9. B#2704: Fix for the error related to clock_enable when there is space between its value.


UVM


  1. B#2674: Fix for the property uvm.reset_constraint in Java.


RTL


  1. F#23293: Fix for double-semicolons causing compile errors in simulation.
  2. F#23295: Fix for begin..end keywords positioning issue causing generation problems in Verilog2001 output.
  3. F#23228: Fix for the issue with "rtl.name_format=%f" property when combined with "interrupt" property.
  4. F#22988: Fix for the issue with property ‘registered=false’ when used with ‘rtl_reg_enb’ property.
  5. F#23311: Fix for addr width issue for the ‘agni_sync_hw_block’ property.
  6. B#2672: Fix for missing ‘*_rd_data’ buffer signal for property wide_external_access.
  7. B#2673: Fix for missing ‘wr_enb_in’ signal and bus_width signal definition.
  8. F#23081: Fix for ‘rtl_name_format’ when used with 'sv_interface' property at chip level with multiple sec-in-sec hierarchy in sv output.
  9. B#2656: Fix for chip-in-chip flow in generation for multiple instances in case of property sv_interface=true/struct in sv output.
