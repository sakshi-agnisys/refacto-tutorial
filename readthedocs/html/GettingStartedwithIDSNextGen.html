Getting Started with IDS-NG

IDS-NG enables users to capture the specification of IP and SoC. It is a platform independent tool with which user can create specifications on all version of RedHat, Ubuntu and others, Windows.


The steps given below explain how to get started with IDS-NG:

1. Creating a new Project:

  • Select File > New Project > create project to launch the new project wizard.
    • In the Project Name field, enter a name for the project. 
    • In the Project Location field, enter the directory name or browse to the directory location.



  • Click ‘Create Project’ to create the project.

2. Opening an existing project:

  • User can open an existing project by following step:

    Select File >         Open Project >


  • A window will pop up where the project to be open can be selected. User can create new IP, Register Spec or Text file in this existing project.


NOTE: User can also open an existing file, but that file won’t be included in the current project. To add any new file, User         must create a new file.


3. Create a new file:


  • Select File > New File > (‘Create IP’ or ‘Register Spec’ or ‘Text File’)
    • Creating IP: - A folder with a same name Register Spec in it.
    • Register Spec: - Specification file with a block and register in it.
    • Text File: - This is a simple Text Editor File. 


    • For referring documentation quickly, a link has been provided in menu help, i.e., Quick Start, as shown below:


                                 


Fig: Click on ‘Create’ to create a file.


NOTE: There are three types of file that a user can create in IDS-NG. These are: IDS-NG Document and IDS-NG Spreadsheet.


  • If a user creates IDS-NG Document , then user can create its specification by adding IDS-NG Templates as:


    • Click on the appropriate button from the tool bar to add an IDS-NG template.
    • Fill in the template with an appropriate name according to the desired specifications.
    • IDS-NG also has an additional feature of hints for both s/w and h/w access, which can be used by typing dot, that is '.', in the respective section.

  • If user is creating an IDS-NG Document then user is in the 'Register View'. From Register View, user can either switch the Param View or Sequence view.



  • If a user creates IDS-NG RDL Editor, then user can create its specification by following steps:


    • IDS-NG uses a default template for register data specification as shown in the picture below. Fill in the template with appropriate data according to the desired specifications
    • Create new project then user can open a rdl testcase by adding IDS-NG Templates as:
    • Create new project and select Register Spec.



    • After select Register Spec where three options will be given.
    • These are: IDS-NG Document and IDS-NG Spreadsheet and RDL Editor.
    • After that select RDL Editor  and open a rdl test case in editor.


 



    • System RDL Editor support keyword hinting also dynamic hinting 
    • This editor supports syntax checker, means it will show error if there is an syntax error in the spec.


  • If a user creates IDS-NG Spreadsheet, then user can create its specification by following steps:
    • IDS-NG uses a default template for register data specification as shown in the picture below. Fill in the template with         appropriate data according to the desired  specifications.

    • However, these templates can be changed by the user by altering the 'User Header' column. For example, if a user changes 'chip' to         'chip_name' in 'User Header' then while using chip in spreadsheet, the template name must be changed to 'chip_name' from 'chip'.



  • If user is creating an IDS-NG Spreadsheet then user is in the 'Spreadsheet View'. From Spreadsheet View, user can either switch to the Param View or Sequence view.


4. “Configuring” the Specification:

    • The Configure button can be used to make settings according to the requirement and choose the kind of files that need to be generated. User can make the appropriate settings in the Configure window.
    • User can select the desired outputs to generate under the Outputs tab. Click on the required check boxes from Options available.
    • Click OK to apply the chosen options.
