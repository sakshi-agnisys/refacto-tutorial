Setting reserved bits

Property undefined_rd_val


The property "undefined_rd_val" sets the undefined (reserved) bits of the register. The property is applicable on chip/block/reg for vhdlalt1, vhdlalt2 and verilog outputs.


For example:user has used "undefined_rd_val=true" property on a register in which sw_bit_enable property is applied using its own fields on one register and of the other fields on another register.


Here we have an example of using this property:


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View:



IDS-NG Spreadsheet View:



SystemRDL


property undefined_rd_val {type =boolean; component = addrmap | reg ; };   

addrmap blockname {  

  reg DATASWENABLE1 { 

     undefined_rd_val = true ;

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

     } F2[15:8] = 8'h0;

    field {

      hw = na;

      sw = r;

    } F3[7:0] = 8'h0;

  };

  reg DATASWENABLE2 { 

     undefined_rd_val = true ;

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

    } F3[23:16] = 8'h0;

    field {

      hw = rw;

      sw = rw;

    } F2[15:8] = 8'h0;

    field {

      hw = na;

      sw = r;

    } F1[7:0] = 8'h0;

  };

  DATASWENABLE1 DATASWENABLE1;

  DATASWENABLE2 DATASWENABLE2;

  };
