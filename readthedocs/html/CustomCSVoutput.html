Custom CSV output


(Compatible in IDS version 5.16.8 and above, in IDS-Batch only)


It is possible to generate a user customizable file in CSV format, usingIDS Word, IDS-Excel and all input file types supported by IDS-Batch.

IDSBatch


To generate Custom CSV output using IDS-Batch include "customcsv" in list of outputs required to be generated, followed by an option "-csvheader" which is optional.


Syntax:


<cmd> IDS-Batch <input_file> -out customcsv [-csvheader "<custom header mapping>"]


Note: '-csvheader' is required only when user wants to customize the header for CSV output file, else default IDS header is selected.

  • Below the Properties section in "CustomCSV" settings, user can add the properties, which will become header in the output. (see snapshot above)
  • Value in left textbox is IDS properties (like "repeat" in above snapshot) or user defined properties (like "u_expression" in above snapshot).
  • Value in right textbox is the new name for that property, which will appear in the CustomCSV output.


Note: 

  • CSV Header's should have no spaces in its name.
  • Right text box can be left empty if user does not want to change the name for the property (see for "hdl_path" in above snapshot).
  • If a value is written in right text box, it is mandatory to mention the actual property name in left textbox.

Enhancement for custom CSV output 


IDS supports a wide range of output types and one of them is custom CSV. To use this output in IDS user can give customCSV in -out args through which helps to generate CSV output from different types of IDS supported input types like docx, rdl, IDSng etc.example: - IDS-Batch <input file> -out customeCSV -dir <output_dir_name>Now IDS has improved the performance, flexibility and accuracy for CSV output.
This customCSV output has capability to customize the header by using “-csvheader” switch else it will use default header “chip, block , section name, register, repeat, width, offset, field, bits, sw access, hw access, field default, description”. In case of using  higher level components than block, it will contain their name header cell also.example: - IDS-Batch <input file> -out customeCSV -dir <output_dir_name>
-csvheader "chip:SOCmap,block:addrmap,reg:rtl_reg,reg_width:RegWidth,offset:offset,field:field,bits:field_bits,default:default_val,sw:sw_access,hw:hw_access,doc:Information"
