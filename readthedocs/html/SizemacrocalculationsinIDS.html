Size macro calculations in IDS

This document describes how the value of the Size macro is calculated in the C header.  In the Mid Jan 2022, we moved the *_PER_INSTANCE_SIZE macro from the top_address.h file to  that componentâ€™s header file and change the name of the  *_PER_INSTANCE_SIZE with *_SIZE. Also *_PER_INSTANCE_SIZE macro only generates for the arrayed components but *_SIZE macro will be generated for all the components (Arrayed and Non Arrayed). Now the value of the * _SIZE component will be the total space occupied by that instance. But there are some scenarios in which calculation(with STRIDE) of *_SIZE  looks improper. 

In the following paragraphs  we discuss  several possible scenarios with the explanation of how SIZE macro calculation should be done.  

Case I: When Stride is not applied

SystemRDL Input: 


addrmap block1 {

  reg reg2{

    field_sRW_hR_t #(.WIDTH(1)) f0[8:8] = 1'd0;

    f0->sw = w;

    field_sRW_hR_t #(.WIDTH(1)) f1[15:15] = 1'd0;

    f1->sw = w;

  };

  

  reg2 reg0_0;

  reg2 reg2_1 @0xC;

  reg2 reg2_2 @0x10;

};


addrmap ip1 {

  chip = true;


  block1 block1_0;

};


addrmap ss1 {

  chip = true;


  block1 block1_0;

  ip1 ip1_0;

};


addrmap top {

  chip = true;


  ss1 ss1_0;

};


Expected value of *_SIZE Macros 


Block1.h 

 BLOCK1_SIZE 0x14


ip.h 

  IP1_SIZE 0x14


ss1.h

  SS1_SIZE 0x28 


top.h 

  TOP_SIZE 0x28

Case II : When Count is applied but Stride is not applied.  


SystemRDL Input: 


addrmap block1 {

  reg reg2{

    field_sRW_hR_t #(.WIDTH(1)) f0[8:8] = 1'd0;

    f0->sw = w;

    field_sRW_hR_t #(.WIDTH(1)) f1[15:15] = 1'd0;

    f1->sw = w;

  };

  

  reg2 reg0_0;

  reg2 reg2_1 @0xC;

  reg2 reg2_2 @0x10;

};


addrmap ip1 {

  chip = true;


  block1 block1_0[4];

};


addrmap ss1 {

  chip = true;


  block1 block1_0[3];

  ip1 ip1_0[2];

};


addrmap top {

  chip = true;


  ss1 ss1_0[2];

};


Expected value of *_SIZE Macros 


Block1.h 

  BLOCK1_SIZE 0x14


ip.h 

  IP1_SIZE 0x50 (block1 size is 0x14 and repeat of 4 on block1) 


ss1.h

  SS1_SIZE 0xDC (Size of ip1 is 0x50 and repeat of 2 on it. Block1 size is 0x14 and

                repeat of 3 present on block)


top.h 

  TOP_SIZE 0x1B8 (Size of SS1 is 0xDC and repeat of 2 present on it. 


In the current implementation, IDS generates the size of components based on the instances for that particular component which is wrong. 

Case III : When Stride is applied.  


SystemRDL Input: 


addrmap block1 {

  reg reg2{

    field_sRW_hR_t #(.WIDTH(1)) f0[8:8] = 1'd0;

    f0->sw = w;

    field_sRW_hR_t #(.WIDTH(1)) f1[15:15] = 1'd0;

    f1->sw = w;

  };

  

  reg2 reg0_0;

  reg2 reg2_1 @0xC;

  reg2 reg2_2 @0x10;

};


addrmap ip1 {

  chip = true;


  block1 block1_0[3] @0x0110 += 0x0050;

};


addrmap ss1 {

  chip = true;


  block1 block1_0[3] @0x0100 += 0x0200;

  ip1 ip1_0[2] 0x0000_2000 += 0x01000;

};


addrmap top {

  chip = true;


  ss1 ss1_0[2];

};


Current Value of *_SIZE macros.

 

Block1.h 

 BLOCK1_SIZE 0x414


ip.h 

  IP1_SIZE 0x11C4  


ss1.h

  SS1_SIZE 0x6388 


top.h 

  TOP_SIZE 0x6388  


Expected value of *_SIZE Macros 


Block1.h 

 BLOCK1_SIZE 0x14


ip.h 

 IP1_SIZE 0x200 (As Stride Size is 0x50 and count on 3 applied on block and first  block is starting from 0x110.) 


ss1.h

  SS1_SIZE 0x4000 (Stride Size of ip1 is 0x1000 and it is starting from 0x2000.)


top.h 

  TOP_SIZE 0x8000  (repeat of ss1 is 2).


Reg/Regfile address macro


In current implementation, in case of Register and Regfile the value of *_SIZE macro is the total size occupied by that component i.e with the repeat value and while the original size of the reg/regfile is used by the *_PER_INSTANCE_SIZE macro. 


Now as per the new enhancement in case of count there is a new argument(idx) added in the *_address macro. The value of this argument is ((idx * reg_SIZE)+ reg_OFFSET). But in case of reg and regfile, logically this is not correct as  value *_SIZE macros is the (original size of reg/regfile * count value). So after the new enhancement, in case of reg/regfile *_SIZE macro is replaced by the *_PER_INSTANCE_SIZE. 


SystemRDL Input:


field field_sRW_hR_t #(longint unsigned WIDTH = 1) {

  name = "Config CSR field name";

  desc = "Config CSR field description";

  sw = rw;

  hw = r;

  fieldwidth = WIDTH;

};


addrmap ip1 {

  reg csr {

    field_sRW_hR_t #(.WIDTH(4)) VAL;

    VAL->reset = 0;

    VAL->onwrite = woset;

  };

  csr reg0;

  csr reg1[5];


  regfile rf {

    csr reg2;

    csr reg3[4];

  };

  rf rf0;

  rf rf1[3];

};


Old Implementation 


#define IP1_REG1_OFFSET 0x4U


#define IP1_REG1_SIZE 0x14U

#define IP1_REG1_PER_INSTANCE_SIZE 0x4

#define IP1_REG1_COUNT 5


#define IP1_REG1_ADDRESS(baseAddress,idx) \

  (baseAddress + (idx*IP1_REG1_SIZE) + IP1_REG1_OFFSET)


After Enhancement


#define IP1_REG1_OFFSET 0x4U


#define IP1_REG1_SIZE 0x14U

#define IP1_REG1_PER_INSTANCE_SIZE 0x4

#define IP1_REG1_COUNT 5


#define IP1_REG1_ADDRESS(baseAddress,idx) \

  (baseAddress + (idx*IP1_REG1_PER_INSTANCE_SIZE) + IP1_REG1_OFFSET)
