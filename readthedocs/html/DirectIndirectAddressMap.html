Direct Indirect Address Map

Communication between an 8-bit direct mapped registers and 32-bit wide indirect map registers

(Compatible with IDS version 5.20.30.0 and above)


In certain applications the bus communicates directly with a bank of 8 bit registers. There may be another block of wider registers say 32-bit which is accessed indirectly through the 8-bit registers. This can be represented in IDS by the following example



Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View



IDS-NG Spreadsheet View





SystemRDL


property indirect_map {type = string; component = reg;  };

property datagroup {type = string; component = reg;  };           

property hw_rd_trigger {type = string; component = reg;  };           

property hw_wr_trigger {type = string; component = reg;  };           

property datagroup_order {type = string; component = reg;  };

property index_reg { type = string; component = reg; };

 

addrmap datagroup {

  name  = "datagroup Address Map";


  reg index { 

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } Findx[7:0] = 8'h0;


  };

 

  reg Data3 { 

     indirect_map = "Group1";

     datagroup = "A";

     hw_rd_trigger = "index,wr";

     hw_wr_trigger = "Data3,wr";

     datagroup_order = "Data0,Data1,Data2,Data3";

     index_reg = "index";

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } f0[7:0] = 8'h0;


  };

 

  reg Data2 { 

     datagroup = "A";

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } f1[7:0] = 8'h0;


  };

 

  reg Data1 { 

     datagroup = "A";

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } f2[7:0] = 8'h0;


  };

 

  reg Data0 { 

     datagroup = "A";

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } f3[7:0] = 8'h0;


  };

 

  index index @0x00;


  Data3 Data3 @0x04;


  Data2 Data2 @0x08;


  Data1 Data1 @0x0C;


  Data0 Data0 @0x10;


};


  1. Four 8-bit registers (Data0, Data1, Data2, Data3) are grouped together by specifying the property “datagroup = A” on each of them. Where the value “A” is the group name.
  2. The property "indirect_map=Group1” is used to specify the indirect register arrays(32-bit in the indirect map), which is accessed by the index and (Data3 to Data0) data registers in the direct map. The value of indirect_map property can be a reggroup name which contains the indirect register array definitions in register templates. This is not a normal regroup, it has a property “virtual = true”, which implies that the registers inside this regroup are excluded from the address map.


  1. Implicitly a write/read on Data3 triggers write/read to the indirect registers in the Indirect_map. You could also explicitly specify properties to trigger read/write to indirect registers by following properties:-

hw_rd_trigger = Index, wr

Where a write to the “Index” register (8bit) triggers read from the indirect register (32bit)

hw_wr_trigger=Data3, wr 

Where a write to the “Data3” register (8bit) triggers write to the indirect register (32bit)

  1. The property “datagroup.order= Data0, Data1, Data2, Data3” specifies the order, where Data0 is MSB.


Note:

The properties in 2 to 4 can be used only on data register with "index_reg" property


IDS will also generate the indirect register model and rtl Verilog file for that indirect register section or referred document which is having “virtual=true” property
