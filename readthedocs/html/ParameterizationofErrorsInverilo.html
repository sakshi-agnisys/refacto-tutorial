Parameterization of Errors In verilog output


The issue persists when a user applies “repeat” property over a register and generates verilog output from it and override/change the value of repeat in the verilog parameters using some different value i.e. greater than the default value, then there is  a warning of:

Warning-[SIOB] Select index out of bounds


assign error  = Reg1_write_error[0] | Reg1_write_error[1] | Reg1_write_error[2] | Reg1_write_error[3]...................;


Errors in Different Cases


Case 1:   Errors due to register access

Fig1: Word Example

IDSExcel Example:

Fig2: Excel Example

SystemRDL Example:

addrmap Block1 {

  name = "Block1 Address Map";   

  reg Reg1 { 

     regwidth = 32;

    field {

      hw = rw;

      sw = r;

      

    } F1[31:0] = 32'h0;


  };

  Reg1 Reg1[4] @0x00;

};

Here, write error will be generated for the register for which the error logic is shown below:

wire [Reg1_count-1 : 0]Reg1_write_error;        // ERROR SIGNAL 

generate

        genvar Reg1_i;

        for( Reg1_i = 0; Reg1_i < Reg1_count; Reg1_i = Reg1_i + 1)

        begin : Reg1_gen

assign Reg1_write_error[Reg1_i] = Reg1_decode[Reg1_i] && wr_stb;

           end //Reg1

    endgenerate


assign error = (|Reg1_write_error);           

Case 2:   Errors due presence of protection

IDSWord Example:

Fig3: Word Example

IDSExcel Example:

Fig4: Excel Example

SystemRDL Example:

property reg_prot {type =string; component = reg; };             

addrmap Block1 {

  name  = "Block1 Address Map";

  reg Reg1 { 

     reg_prot = "priv" ;

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

      

    } F1[31:0] = 32'h0;

  };

  Reg1 Reg1[4] @0x00;

};

Here, error will be generated due to the presence of protection from SW side. These protection errors will be generated only in the case of APB, AHB and AXI Bus. 

wire [Reg1_count-1 : 0]Reg1_read_error;    // ERROR SIGNAL;           

wire [Reg1_count-1 : 0]Reg1_write_error;   // ERROR SIGNAL;           


generate

        genvar Reg1_i;

        for( Reg1_i = 0; Reg1_i < Reg1_count; Reg1_i = Reg1_i + 1)

        begin : Reg1_gen

assign Reg1_write_error[Reg1_i] = Reg1_decode[Reg1_i] && wr_stb && ~Reg1_wprot;

assign Reg1_read_error[Reg1_i] = Reg1_decode[Reg1_i] && rd_stb && ~Reg1_rprot;


           end //Reg1

    endgenerate

assign error = (| Reg1_read_error) | (|Reg1_write_error);


Case 3:   Errors due presence of empty address

SystemRDL Example:

property size{type=longint;component=addrmap;};

addrmap Block1 {

  name  = "Block1 Address Map";

  addrmap section {

    name  = "section Address Map";

   size = 100;

    reg Reg1 { 

       regwidth = 32;

      field {

        hw = rw;

        sw = rw;

      } F1[31:0] = 32'h0;

    };

    Reg1 Reg1[4] @0x000;

  };

  section section[2] @0x000;

};

Here, error will be generated due to the empty space inside the section.

wire [63:0] emptyaddress0_error_l[section_count-1 : 0];

wire [63:0] emptyaddress0_error_h[section_count-1 : 0];

wire [section_count-1 : 0]emptyaddress0_error;   

 wire invalid_address_error;

generate

        genvar section_i;

        for( section_i = 0; section_i < section_count; section_i = section_i + 1)

        begin : section_gen

             genvar section_Reg1_i;

      for( section_Reg1_i = 0; section_Reg1_i < section_Reg1_count; section_Reg1_i = section_Reg1_i + 1)

           *

       *

end //section_Reg1

assign emptyaddress0_error_l[section_i] = block_offset + 'h10 + section_i * 'h100;

assign emptyaddress0_error_h[section_i] = block_offset + 'hFF + section_i * 'h100;

assign emptyaddress0_error[section_i] = ((address[addr_width - 1 : 0]  >= emptyaddress0_error_l[section_i][addr_width - 1 : 0]) && (address[addr_width - 1 : 0]  <= emptyaddress0_error_h[section_i][addr_width - 1 : 0])) ? 1'b1 : 1'b0 ;

 end //Block1_section

endgenerate 

assign invalid_address_error = (|emptyaddress0_error) ;  

assign error = invalid_address_error;
