Next property


(compatible with IDS version 5.10.0.0 and above)


"next" property describes next input value of field. It can have any signal name, any other field within the same register or any field from other register.

Example 1


  • next= “a”;

'a' can be any field within same register or any external signal name.


  • next= “reg1.b”

Now field with this property has next value modified by field 'b' from within register 'reg1'.


  • next= “reg1@intr”

‘@’ specifies dynamic properties of the register or field, for example- interrupt “irq” from reg1 is stored in this field.


  • next= "reg1.b && a"

       It also supports complex expressions as shown above.


  • next = "<fieldname>@incrsaturate/@decrsaturate/@incrthreshold/@decrthreshold"

          Check Example 3 for the more details about it

Conditional assignments in "next"


‘Next’ property now supports conditional assignment of next input value on a field. These conditions can be read/write events of another internal register of the generated block IP.


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View



IDS-NG Spreadsheet View



SystemRDL


property ids_next{type=string; component=field; };

addrmap Block1 {

     addrmap secA {

           reg Reg1 { 

              regwidth = 32;

                     field {

                        hw = rw;

                        sw = rw;

                     ids_next = "Reg2.Fld1 if Reg2@sw_write";

                    } F1[31:16] = 16'h0;

                    field {

                       hw = rw;

                       sw = rw;

                    ids_next = "Reg2.Fld2 if Reg2@sw_read";

                    } F2[15:0] = 16'h0;

           };

         Reg1 Reg1;

      };

          reg Reg2 { 

             regwidth = 32;

                  field {

                      hw = rw;

                      sw = rw;

                   } Fld1[31:16] = 16'h0;

                  field {

                      hw = rw;

                      sw = rw;

                  } Fld2[15:0] = 16'h0;

          };

       secA  secA;

       Reg2 Reg2;

 };

Note:

  • Conditional next assignments on a field are supported only for assigning value of some other field in the same IP.
  • Conditional assignments in RDL input specification is supported using user defined property (UDP) “IDS_next” (as given in example above).
  • Software read/write conditions of non-repeat internal register can only be used to assign next value in a field.




Dynamic assignment of next input value in a field w.r.t iterations on the repeated parent section (or register)  is also supported for SystemRDL input specification using the user defined property (“IDS_next”) which accepts string as its value. Using this UDP, a field can be assigned a next input value dynamically on particular repeat iterations of its parent section/ register depending on software events (write/read) of another internal non-repeated register of the same IDS block.

Note:

In cases where repeat on both parent register and ancestor section is used to assign dynamic next value, the repeat on immediate parent is considered for assigning next input value. 


"two-way" conditional assignment

(v7.8.0.0 onwards)


‘Next’ property has now been extended to support two-way conditional assignment of next input value in a field. These conditions can be read/write events of another internal register of the block IP.

SystemRDL


property IDS_next {type = string; component = field ; };

addrmap Block1 {

  addrmap SecA {

reg Reg1 { 

  regwidth = 32;

  field {

hw = rw;

sw = rw;      

  } F1[7:0] = 8'h0;

  

  field {

hw = rw;

sw = rw;      

  } F2[31:8] = 24'h0;

};  

Reg1 Reg1;

  };

  

  reg Reg2 { 

regwidth = 32;

field {

  hw = rw;

  sw = rw;

} Fld1[7:0] = 8'h0; 

field {

  hw = rw;

  sw = rw;

} Fld2[15:8] = 8'h0;  

field {

  hw = rw;

  sw = rw;

} Fld3[23:16] = 8'h0;  

field {

  hw = rw;

  sw = rw;

} Fld4[31:24] = 8'h0;

}; 

  SecA SecA[4];

  Reg2 Reg2;

  SecA[0].Reg1.F1 -> IDS_next = "Reg2.Fld1 if Reg2@sw_write";

  SecA[1].Reg1.F1 -> IDS_next = "Reg2.Fld2 if Reg2@sw_write";

  SecA[2].Reg1.F1 -> IDS_next = "Reg2.Fld3 if Reg2@sw_write";

  SecA[3].Reg1.F1 -> IDS_next = "Reg2.Fld4 if Reg2@sw_write";

  Reg2.Fld1 -> IDS_next = "SecA[0].Reg1.F1 if SecA[0].Reg1@sw_write"; 

  Reg2.Fld2 -> IDS_next = "SecA[1].Reg1.F1 if SecA[1].Reg1@sw_write";

  Reg2.Fld3 -> IDS_next = "SecA[2].Reg1.F1 if SecA[2].Reg1@sw_write";

  Reg2.Fld4 -> IDS_next = "SecA[3].Reg1.F1 if SecA[3].Reg1@sw_write"; 


};

Note

  • Two-way conditional next assignments on a field are supported only for assigning values of some other field in the same IP.
  • In case of repeat, where either hierarchy of field where ‘next/IDS_next’ property is applied has repeat or the value of field used in the property has repeat on itself or ancestor hierarchy, a proper instance of repeat needs to be specified.
  • Such two-way conditional assignments in RDL input specification are supported using user defined property (UDP) “IDS_next”.
  • Software read/write conditions of non-repeat internal registers can only be used to assign the next value in a field.
