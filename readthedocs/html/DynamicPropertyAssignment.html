Dynamic Property Assignment

Dynamic Property assignment 


Often there is a need to modify or apply properties from the top component on to its child components such as section, register or field. This provision is available in SystemRDL, a similar feature is implemented in other IDS input formats as well. Properties of a referenced instance shall be accessed via the arrow operator (->).


Syntax


Relative_path -> property_name = property value 

Where,

Relative_path : Hierarchical path of the component on which property has been applied


 Example 1


In the example below , the property has been applied on a reffered register group (Ref1) under which a register with the name Reg1 and field F1 exists, the software access has been modified to ro(read-ony) by applying the following property “Ref1.Reg1.F1->sw=ro”.
