Wider External Access with Smaller Width Bus

Property name: “wide_external_access=true”

In IDS, we have supported direct access to external memories having width greater than the bus width using the above mentioned property.

Logic Diagram:

Fig: Wider External Access with Smaller Bus


Suppose we have a 128-bit memory which is being accessed by a 32-bit bus. For a combined write transaction, we will create a 96-bit internal buffer to store three 32-bit external write transactions on the bus interface, and generate a 128-bit write transaction on the external interface when a fourth 32-bit write transaction occurs on the bus interface.

In case of combined read transaction, we will read the entire 128-bit on the external interface when the first (LSB) read transaction comes on the bus interface. The lower 32-bits will be sent to the bus interface while the rest of the 96-bits will be stored in the buffer, which can then be read through the required 32-bit read transaction on the bus interface.

Note: The wider memory width is a multiple of bus width and they should be aligned.

Async reset in memory with wide_external_access property


Enhancement has been done to generate the RTL output with an async reset, using the wide external access parameter and the reset type=async property. Currently, a synchronous reset is there for memory even when using the reset_type=async property. Now, IDS  generates RTL output with the “wide_external_access” property on memory with an asynchronous reset clock.


SystemRDL Input

property map_type { type=string; component=addrmap|reg;};

property reset_type {type =string; component = addrmap | reg | field ; };

property wide_external_access {type =boolean; component = mem ; };

       hw = r;
