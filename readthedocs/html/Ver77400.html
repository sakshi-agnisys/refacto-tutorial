Known Issues(if any):


  1. Parameterization inheritance is not applicable for multi-bit tie connection because tie value is fixed, e.g., soc_connect -source block1.p1[W8-1:0] -tie 1’b01000110
  2. When dealing with multiple receivers/drivers, the wire names for their connections will remain consistent. It will utilize the wire name from the first connection for all connections and disregard any user-defined wire names from other connections.
  3. Connecting multiple driver and receiver blocks with other blocks using the same wire name is not supported.
  1. In the case of connecting multiple sources to multiple destinations, The number of sources should be equal to the number of destinations.
  2. Parameterized wire width is not supported in the case of connecting multiple sources to multiple destinations.
  3. For clock_name property, the string argument cannot be repeated across the multiple blocks in a particular input file, for VHDL output. But, it can be repeated when the property is used on multiple registers within same block.
