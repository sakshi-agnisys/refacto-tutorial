Customizing reset value of registers

rtl.default


This property is used to customize the register reset value to be driven by an external signal. It is applied on a register/field. It is for the vhdl/verilog output. Here, in this example the property is applied on the register and field.


Example:        IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View:



IDS-NG Spreadsheet View:



SystemRDL:


      property reg_default {type =number; component = reg ; };

     

      property rtl_default {type =string; component = reg|field ; };

     

addrmap Block1 {

    // Signals    

    signal { activelow; sync; signalwidth = 32; 

       desc= " This signal is sync and activelow with width 32"; } sig;

     

  reg regname #(def1 = 0x1234) { 

     reg_default = "4660" ;

     regwidth = 32;

    field {

      rtl_default = "sig" ;

      hw = rw;

      sw = rw;

    } F[31:0] = 32'h0;

  };

 

  reg Reg2 #(def2 = 9) { 

     rtl_default = "sig" ;

     regwidth = 32;

    field {

      hw = rw;

      sw = rw;

    } F1[21:0] = 22'h0;

    field {

      hw = rw;

      sw = rw;

    } F2[27:22] = 6'h0;

    field {

      hw = w;

      sw = r;

    } F3[31:28] = 4'h9;

  };

      } F[31:0] = 32'h0;

        hw = rw;

      } F1[30:0] = 31'h0;

      field {

        hw = rw;

        sw = rw;

      } F2[31:31] = 1'h1;

    };  

    regname regname @0x00;

  reg Reg5 #(def2 = 9) {

  };

default_reset_name


The rtl property default_reset_name is used to change the default name of the hard reset signal. In the programs, the default name of reset signal is reset_l and the user can write any string in this place. The example below shows that the property is used on block so the whole code has changed with the property string. 


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL
