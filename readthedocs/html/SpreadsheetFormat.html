Spreadsheet Format

In spreadsheet format, register specification is described in a tabular format. For specifying the elements like, chip, block, registers, etc, we provide certain IDS templates that are fully customizable and users can use them as per their requirements. To create specification in spreadsheet format, Agnisys® provides following two flavors:


  • IDS-Excel


To create register data in spreadsheet format, it is essential to have an ‘IDS_template’ sheet. This can be included/imported in the same workbook or in a separate file. In case it is in a separate file, the user needs to point to that file by setting the path in the IDS environment variable, ‘IDSEXCEL_TEMPLATE_FILE’. The later option enables the entire team to use the same format.


This sheet contains the headers that are used while creating the specification. To import this sheet in workbook, user needs to click on the ‘IDS Template’ button provided on the IDesignSpec™ ribbon, as shown below:

To create a custom template, users can start with a template provided in ‘IDSDocs\..\templates\excel_template’ bundle.


IDS_template has following columns:

  • User header: It defines the name of the column in the user specification sheet for the particular IDS identifier.
  • IDS identifier: It is required to use these identifiers in the ‘IDS_template’ sheet along with the user header for the appropriate element and its properties. For example User header ‘block’ has ‘$block_name’ as IDS Identifier in the ‘IDS_template’ sheet. (For details refer to ‘IDS_template.xlsx’ in IDS Supporting Files).


Note: The IDS Identifier for built-in "User headers" are fixed and cannot be changed.


  • Default value: Some of the IDS identifiers $reg_default, $field_default, $field_sw_access, $field_hw_access etc. may have default values. The values defined for these identifiers in the default column will be used if their values are not mentioned in the specification.
  • Property name: It applies to the user defined properties. These are not applicable for the IDS default properties.

 

Moreover, users can also define their own properties in addition to the IDS default properties in the format shown below. Note that these properties are available in the Tcl API that can be used to generate custom code.



Similar to the IDS default properties user defined properties also have the following columns:

  • User header
  • IDS Identifier
  • Default Value
  • Property Name

 

All columns have the same functionality as for the default properties except that for Property name and the IDS identifier.

 

Property Name: Property name is significant only in case of the user defined properties. The names mentioned under this column will be used in the various output generation. User cannot specify same property name for same user header


IDS Identifier: The IDS identifier for the user defined properties depends on the element for which the property is defined.

 


Following are the identifiers that can be used for the user-defined properties: 

  • $block_property : to be used for the properties defined for the block
  • $reg_property: to be used for the properties defined for the register
  • $field_property: to be used for the properties defined for the field
  • $section_property: to be used for the properties defined for the reg group
