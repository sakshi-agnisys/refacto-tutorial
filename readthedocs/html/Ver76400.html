Ver 7.64.0.0

v7.64.0.0

(May 12th, 2023)


IDesignSpec™ (IDS)


RTL Enhancements


  1. F#22099: Support for a special accumulator register for Verilog. More Details
  1. B#2155: Support for the “rtl_hw_wt=true” property for a toggle bit generation whenever the register is updated from the software side. More Details
  2. B#2221: Support for directly reading the original register by adding a new option in “buffer_trig_reg” UDP for Verilog. More Details
  3. B#2224: Support of the top property power_opt = <number> and exclude_clock_gating=true for power optimization. More Details
  4. B#2223: Support for MUX select and Flopped output from IDS. More Details
  5. B#2171: Support RDL UDPs for eval property. More Details


General Enhancements


  1. F#22484: Support for header_hermetic=”true:notime” property for generating the HTML output without the “generated on” title in the header. More Details
  2. B#2173: Support for a feature that allows users to split long HTML pages with many registers into multiple pages using the "-if_html_section" property and the "+top_property doc_pagination=numeric_value". More Details
  3. B#2174: Support of the -if switch for the python output for separate files generation for each chip and block elements. More Details
  4. B#2178: Support for Python output format generation in chip in chip scenario. More Details
  5. B#2181: Support for adding the path of a block in the header for HTML output. More Details
  6. B#2185: Support for the improvisation of some properties in our documents with downloadable improved examples.


UVM Enhancements


  1. B#2175: Support for the UDP "auto_hdl_path=true" for generating the automatic hdlpath at component level as well as top level. More Details
  2. B#2178: Support for dynamic reg_block build using the property  "[uvm loc=build_top]<user_code>[/uvm] property" which enables users to add custom code just beneath the build function on each block individually. More Details
  3. B#2177: Support for per block factory usage command line control which allows users to apply the "uvm_opt=0" property on a specific block in order to retain its factory that may have been removed using the "-top_property" with the value "uvm_opt=1". More Details


Bug Fixes


RTL


  1. F#22410: Fix for the .v file generation with the property “out_enb_stages” having a numeric value as its argument.
  2. F#22182: Fix for next property when used for sections for interrupts.
  1. F#22330: Fixed the issue with  rtl_pack_array in the one-bit field and (hw=rw) issue with the multidimensional repeated register.
  2. F#22275: Fixed the syntax error issue in Verilog in the case of an increment threshold in a counter.
  3. F#22446: Fixed the issue regarding the “parity_error=noread” property.
  1. B#2201: Fixed the issue with clock_enable UDP logic which was not handling  CSRs with fields sw=r and hw=w plus we property properly.
  2. B#2196: Fixed the issue of not generating a proper enable signal when we use the “rtl_port_suffix” property.
  3. B#2202: Fixed the compile and lint issue when we use “rtl_port_suffix” at child addrmap instead of the top addrmap.
  4. B#2203: Fixed the issue with clock_enable UDP in respect of external CSRs.
  5. B#2204: Fixed the issue of flop synchronization with the clock when we used the reset property with SW=r HW=w access.
  6. B#2205: Fixed the issue of generating invalid RTL when we change the reset value of a field outside of a regfield.

    

UVM


  1. F#22322: Fix the issue of using a section as an external memory for memory=all property.
  2. F#22275: Fixed the UVM issue that sets the incrthreshold value to another field in the same register.
  1. B#2206: Fixed the issue of non-supported combinations of write-ones and external CSRs.
  2. B#2195: Fixed the naming issue in UVM output in case of the “inst_name_cppstyle=true”  property.
  3. B#2207: Fixed the UVM issue that is generating the same CB class multiple times resulting in a compilation error.
  4. B#2208: Fixed the UVM issue for the register instance which is not taking correct parameter values as passed in RDL.
  5. B#2209: Fixed the long runtime issue of 12 hours by 3.5 hours.
  6. B#2210: Fixed the issue of unrecognized declaration in UVM.


General


  1. F#22406: Fix for software access r/wc and w1s for CMSIS-SVD output.
  2. F#22444: Fix the output generation issue in YAML input at the chip and block level.
  1. B#2211: Add the date to the timestamp printed in the IDS-Batch.log file.
  2. B#2212: Fixed the issue of not generating a c header when we were trying to use cheader_add_regmap with nested address maps(top level).
  3. B#2222: Fixed the issue of repeated sections that are incorrectly generated in C headers.
  4. B#2213: Fixed the issue of generating intermediate Annotation when the value of the string index is out of range. 
  5. B#2214: Fixed the error of the check process for SV when we use the “rtl_name_format” property at the chip level.
  6. B#2215: Implement a new feature by adding the run time summary in the existing log file.
  7. B#2208: Fixed the issue of not taking correct parameter values by Register instance as passed in the RDL file.
  8. B#2216: Fixed the issue of external CSRs in case of clock_enable property.
  9. B#2218: Fixed the issue of empty default values for CSRs in the case of  JSON and IP-XACT.
  10. B#2219: Fixed the issue of ids batch crash issue.
  11. B#2199: Fixed the issue of big html file.
  12. B#2220: Fixed the issue OXRSL file which does not mask instance_name, hierarchy ID and display_name.
