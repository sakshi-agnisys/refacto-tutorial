IDS-Calc

Introduction


IDSCalc is an add-on for OpenOffice Calc on Windows/Linux machines.


System and Software Requirements


  • Operating System: Windows 2000 (Service Pack 2 or higher), Windows XP, Windows 2003, Windows Vista, Windows 7, Windows 8 or 8.1 or 10 Windows Server, Linux: 

     Red hat, Cent OS, Ubuntu 32bit, Fedora etc.

  • 256 MB RAM (512 MB RAM recommended)
  • 1024 x 768 or higher resolution with at least 256 colors
  • Apache OpenOffice 4.0


Installation Steps


There are two ways to install IDS-Calc.oxt:


OPTION 1

  • Login to AgnisysÂ® website (www.agnisys.com) with the username and password provided to you and download the IDS-Calc.zip. If you do not have this info, contact support@agnisys.com
  • After login visit the Download Software page to get the appropriate software. 
  • Click on IDS-Calc addon button. This will download a setup file(Example IDS-Calcx.x.oxt) to your system.


OPTION 2

  • Download IDS-Calc.oxt from this URL:

http://extensions.openoffice.org/en/project/idesignspec-calc

  • To install IDS-Calc add-on, double-Click on IDS-Calcx.x.oxt file. This will install the IDS-Calc add-On extension for OpenOffice.


                                       


  • Choose OK to install the IDS-Calc extension.


                                       


  • Allow IDS-Calc.oxt to install completely, then close the extension manager.
  • IDSCalc.oxt is installed successfully.

Getting Started


Register data can also be captured by design teams using OpenOffice. To create executable specifications in OpenOffice, AgnisysÂ® provides IDS-Calc.

 

IDS-Calc is offered as an add-on for OpenOffice. Besides OpenOffice, similar applications are available for MS-Word and MS-Excel and a standalone batch utility called IDS-Batch is also available. 


First step would be to download and install the IDS-Calc plug-in from AgnisysÂ® website. For installation details refer to the topic "Installation Steps > IDS-Calc" in "Agni Tools Installation". In addition, download the Supporting IDS-Docs which contains sample examples.


Similar to IDS-Excel, IDS-Calc enables the users to specify the register data in a tabular form. The formatting and column headings for the table are completely user customizable.

To create a register specification in IDS-Calc, the design hierarchy can be specified in the manner shown in the screenshots below, i.e. all the elements should appear in rows as we move down the hierarchy. This way the specification is simplified as some of the common properties of different elements are specified under the same column.

IDS registers can be accessed from the external bus like AMBA etc or any other proprietary bus. This field indicates the policy of software access.

Configuration Settings in IDS-Calc


The input specification in languages like behavioral VHDL, algorithmic SystemC, C++ etc and the RTL description in VHDL/Verilog is produced as the result of synthesis. 


The Configure button can be used to make settings according to requirement, and choose the kind of files that need to be generated. Make the appropriate settings in the Configure window.

Options

Description

RTL

Designs are written in an HDL at a register Transfer Level RTL. The term RTL is used for an HDL description style. Verilog, VHDL, and SystemC are the most popular HDLs.

Verification

Verification basically Test Benches consists of drivers and monitors that drive the stimuli into DUT and analyze the output of DUT against expected output. Test Bench enables verification of the design intent. This tool helps in development of test bench in UVM, OVM and VMM environment.

Header/Classes

This tool can generate header files for C++ and SystemVerilog, which helps the firmware, software and diagnostic teams to create their deliverables.

Documentation

IDesignSpecâ¢ generates HTML 4.0 and PDF documentation that can be shared on the web. It can also generate XML output which can be used to interface with other tools.It also generates the graphical view of the specification in the form of Scalable Vector Graphics (SVG).

Standards

Standards like IP-XACT, define and describe electronic components and their design that enables automated configuration and integration through tools. It ensures delivery of compatible intellectual property descriptions from multiple intellectual property vendors.

Sequence

Sequences enables users to document the sequences and create outputs suitable for various 

teams such as: Design, Verification, Firmware and Software. Having the same Spec drive outputs for the various teams,

 has the same impact as creating common register spec and sharing that with the teams 

does. In other words, all teams are automatically synchronized, errors are minimized which leads to 

higher quality and improved time to market.

Output Directory

Specify the user defined output directory.

  • Advanced Verification


The IDS generated UVM reg model needs additional info like constraint on values, coverage blocks, HDL paths for back-door access etc. for effective verification. IDS in addition to getting this information using properties/syntaxes, provides a way to automatically generate the said features using Advanced verification settings.
