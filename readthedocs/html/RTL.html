RTL

RTL Generation 


IDS generate an RTL (Register Transfer Level) for the register map defined. Verilog, VHDL-synthesizable, System C and SV-synthesizable RTLs can be generated. 

Training Video: Creating RTL Models

IDesignSpec™ Generated Logic Design 


The logic generated by IDS consists of: 


1. A Widget which is essentially a bridge between the register bus selected by the user and an internal bus. 

2. A decode logic block for decoding the register address 

3. Logic for all registers and register groups (except those marked "external"– see below for more details) 

4. Read-back mux for reading data back from the registers and "external" registers or register groups. 

5. Ports for hardware access and external registers and register groups. 


The generated Hardware code is clocked with the Register Bus clock. The clock domain on the Hardware side can be different from this Bus.

Block Description in IDesignSpec™


Target: For each block in the IDesignSpec™ document, a target RTL block is created that implements the specified registers. The interface to the selected bus protocol is implemented by a widget. The generated decode logic, read_back_mux, internal registers and some glue logic are all instantiated in the target rtl block.

 

Decode Logic: This block accepts the address from the bus through widget and decodes the address based upon automatically generated address locations of the registers by IDesignSpec™. The register address decoder decodes the address bits and selects the register enable signals. 


Priority Decode Logic: This is the decision making block when both hardware and software is trying to write to the registers. In this case, hardware is given high priority for write operation.


Internal Registers: These are the internally implemented registers generated by IDesignSpec™. 


External Registers: Registers implemented using memory banks or registers in an external device are called external registers. These registers are not implemented by IDesignSpec™ and need to be implemented manually. IDesignSpec™ provides appropriate handshake signals for the external register to be hooked to the hardware logic generated by IDesignSpec™. 


Read back mux: This block receives the read-back data from the various registers and passes it back to the bus. 

The generated design consists of Block and Register address decode, read-back logic paths, and the registers. As shown in the figure above, the software initiated register access happens through the Register Bus. 


Widget: This block is the bridge between register bus and the internally implemented registers. It changes the external bus or any other proprietary bus signals to local signals which is then used by the target. The widget is changed based on the bus available. The IDesignSpec™ generated hardware is the same but the widget block is instantiated depending upon the bus protocol used.
