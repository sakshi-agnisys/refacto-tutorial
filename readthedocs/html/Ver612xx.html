v6.12.32.0

16th May 2018


IDesignSpec™

RTL Enhancements

  • next, assign, and we property: Complex expressions are now supported with these properties (except ternary operator) More Details
  • Trigger Buffer Register: Different registers can be triggered by a register using buffer_trig_reg property in line with trigger buffer register More Details
  • rtl.sw_latch=both: This is property is used to store the transition of high to low/ low to high pulse transaction from SW side More Details
  • rtl.hw_latch=high: This property is used to store the high transition from hardware side More Details
  • rtl.hw_latch=low: This property is used to store the low transition from hardware side More Details
  • rtl.hw_latch=both: This property is used to store the high to low and low to high transition from HW side More Details
  • rtl.sw_write=clear: By using this property, register can be cleared after the write transaction More Details
  • rtl.sw_read=clear: By using this property, after the read transaction, the register gets cleared and data from the HW side gets stored in it. More Details
  • rtl.bmax=true,rtl.bmin=true: By using this property, the register stores the maximum or minimum value from the hardware side More Details
  • shadow_reg: Shadow register can now also be declared by specifying a property “shadow_reg” which is the name of the original register which is to be shadowed. More Details
  • crc: This property is used to generate and check the crc of the data registers More Details
  • parity: This property is used to generate and check the parity of the data registers More Details
  • ecc_sniffer: This property is used to iterate through all the parity registers More Details
  • buffer_trig_reg property with repeat: buffer_trig_reg property can be used with repeat property where, the first instance of the repeated register will be the trigger for the buffer registers. More Details
  • Trigger on output port: When buffer_trig_reg property is used along with registered=false on register, <reg_name>_trig_out signal is an output signal in the port list is created More Details


UVM Enhancements

  • Trigger Buffer register: Trigger Buffer Register is implemented in UVM More Details
  • ignore_prop=hdl_path: This property is used to ignore the prepended hdl_path for reggroup, register and field More Details

IP-XACT Enhancements

Multi-out files can be generated for block having the property subblock on them. More Details


General Enhancements: 

-fast: When this option is used code can be generated at a very high speed. This option is currently available for UVM output only. More Details

SV Header:

SV Header can be generated at a high speed when -out_third_party d option is used More Details


IDS-NG™

  • IDS-NG is a multi-platform product which helps user to create SoC specification at an enterprise level. More Details

Automatic Register Verification ( ARV™ ) Enhancements: 

  • uvm_regmodel_container: This property is used to specify the container of the register model 
  • uvm_sequencer: This property is used to change the uvm_sequencer on which the sequence will run 
  • arv_setup_seq: This property is used in case the  user want to run a sequence before any ARV™ sequence is run 
  • unlock_seq: By using this property, customer can unlock the lock register by using their own custom sequences
  • no_reg_test and no_field_test: no calculation for coverage is done when no_reg_test property is applied on a register and same is reflected in html output, similarly when no_field_test is applied on a field coverage is not calculated for that field.

IDesignSpec™ ( ISS ) Enhancements: 

  • Signals: Signals can be written or read, and their values can be assigned to registers/fields. More Details
  • Structs: Structs are generated and used in UVM and firmware Sequences. More Details
  • MATLAB Output: The new output is now generated with ISS Sequences. More Details

Bug Fixes:

  • Fix for subblock while importing to excel: While importing to excel, subblock is now generated under the section instead of block.
  • HTML description when variant table: Description for variant table is now visible when HTML output is selected
  • Parameter: Parameter declaration when UVM output is selected in case of ref has been fixed
  • Parameters in Cheader: While doing ref and generating multiout files, parameters of the top file are now declared in their child referred files.
  • Fixed issues with interrupts with software precedence
  • Fixed issues on default_reset_name when used with registered=false and pulse generation
  • Undeclared parameter fix for lowest level block in the hierarchy
  • Defines declaration fixed in case of multi-out output.
  • Corner cases bug fix for alias in different block
  • Fixes in use of Sequence Index in UVM and C: Sequence index was not working earlier.
  • Bug fix for no_reg_access_test property for ARV™ with 100% coverage
  • avalon_nonburst property: avalon_noburst property applied other than on top will be ignored
  • Defines are handled correctly in VHDL now
  • Fixed issue in lock when applied on both field as well as reg. Also, Fixed lock functionality on multiple field level locks
  • Modified rt.hw_enb to disable the _in_enb signal
  • Modified assign, we and next properties to have a field declared in RO-WO pair used in these properties
  • Now multiple sequence with same ip can be specified in an Index sheet in IDesignSpec™. 
  • Now after wait next write is coming in next row instead of same row in CSV output in IDesignSpec™.

v6.12.0.0

24th Feb 2018 (DVCon edition)


RTL Enhancements

  • Format Property: This property is used to represent unsigned, signed, unsigned fixed point and signed fixed point fields which is now supported in VHDL as well More Details
  • clock_name: This property specifies the clock name of the register on which it is applied. It is now supported in VHDL as well More Details
  • ignore_prop: This property can be used to ignore any property in the hierarchy. E.g. when value is hdl_path_internal it ignores the property 'hdl_path_internal' at reg/field/section below the current node More Details
  • lock_bitmask: This property is used to mask the bits of a locked register i.e if the register map is being accessed, the associated bitmask will have 0’s for the locked bits and 1’s for the unlocked bits More Details
  • Indexed Indirect Register: The software access of “read” and “read/write” besides the simple “write” on Indexed indirect register are supported now More Details
  • Repeat on Memory: Repeat property can be applied on memory now to vectorize a memory interface More Details

SystemC Enhancements

  • Repeat on Register Group: Repeat property is now supported on register group in systemC
  • “-if_sysc”: using this command line option separate systemC file for all the blocks in a chip are created More Details

UVM Enhancements

  • Interrupts: All interrupt properties such as intr.status, intr.enable, intr.pending are now supported in UVM More Details
  • Counters: All counter properties such as counter.sw.wr, counter.sw.rd are now supported in UVM More Details
  • Coverage: Using the property coverage=off at the top level, the build_coverage and add_coverage functions can be removed from the generated UVM regmodel More Details
  • uvm_add_regmap: This property is used to create additional register map for the same bus domain More Details

IP-XACT Enhancements

  • “-if_ipxact”: using this command line option separate IP-XACT file for all the blocks in a chip are created More Details

General Enhancements

  • "-ignore_enums": using this command line option enums can be ignored from the input XML file More Details
  • Cache Directory: An environment variable can be specified as IDS_CACHE_DIR to set the cache directory for IDS other than the default “.IDS” 
  • Velocity API: Performance and speed of the Velocity API has been improved More Details
  • override_refname: This property is used to override the module name from the referred document instantiated in TOP More Details


Industry Standards 

Verification of AXI4Full(Non-pipelined)Bus: Verification of Axi4Full_np.v widget has been done with Functional Coverage = 99.5 and Code Coverage = 95.2


SystemRDL 

  • Memory: Supported third party property “memory” in SystemRDL More Details

Bug Fixes:

  • Counter in VHDL: Fully supported all counter properties in VHDL
  • clock_enable in Verilog: This property is used to make write on a register depend upon either the signal or the access types
  • Lock in VHDL: Supported all lock properties in VHDL
  • Interrupt in VHDL: Supported 'intr.nonsticky' for single bit as well as multiple bits.
  • Python: Annotation check applied when keywords like offset address are used
  • SystemC Header offset: Fixed Offset value in SystemC.
  • SystemC Header function: Fixed naming issue of read function.
  • changed #defines for mem in Cheader: fixed #defines in Cheaders in case of memory
  • Fix for same expression used in lock and lock_r in UVM
  • Description in HTML for block and chip with Line Break.
  • PDF formatting issue and conversion from DOCX fix
  • PDFs, SystemC and SVheader generation from TOP: Fixed multiout options using command Line.
  • Now buswidth from the specification can be used while generating output from the batch
