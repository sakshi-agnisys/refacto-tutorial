Registers are basic building block of the register space. Each bit or set of bits within the register controls some behavior of the circuit.

The IDS's register template consists of different fields explained below. Additional properties can be added to a register or field using the curly brackets "{}"


Every register has at least two accesses: Software Access and Hardware Access. The Software Access is the access from the host register bus e.g. AMBA, or OCP-IP etc. The Hardware Access is the access to the register from the user defined hardware logic. For more details see the section on "Hardware Generated by IDesignSpecâ„¢".



Property

Definition

reg_name

  A unique string that identifies the name of the register.


  Example : Read_write_reg

offset

The address of the register in the address map is with respect to the address of the container (for register the container is block/section). If the user defines the offset then value in the offset is the displacement added to the memory location of the container. The value can be decimal value or a hexadecimal value.


Example : 10 or 0x10

external

The possible values of this property field can "true" or "false". The default value of this field is "false". If this property is set to "true" then an RTL implementation is not created by the IDS, instead appropriate ports are created so that the user can provide its own implementation to that register. For more details refer to next section.


Example : true or false 

size

The width of the register and the value specified must be a multiple of 8.

bits

The width of the particular field of the register.


Example : 0:16

sw access(s/w)

Indicates the policy of the software access. IDS register can be accessed by external bus like AMBA etc or any other proprietary bus.


Example : rw or wo or rc or rs or wrc or wrs or wc or ws  or wsrc or wcrs or w1c or  w1s or  w1t or  w0c or  w0s or w1src or  w1crs or w0src or  w0crs or  r/w1c r/w1t or r/w0t or  r/w0t or r/wc or  r/w0c or r/ws  or r/w1s or  w0t 

hw access(h/w)

Indicates the policy of the hardware access. IDS register can be accessed by the hardware within the block.


Example : ro or rw or wo or na 

default

This is the default value of the field. A value for default field is mandatory. The value can be a decimal or hexadecimal or octal or binary value along with the base specified. The decimal value can be given without specifying the base.


Example : 12 or 'b0101 or 'hf or 'o12

description

A description of the functionality associated to the register. In addition to this, the various properties can also be defined in this field. For a list of properties, see here.


Any text can be added here. The text can be formatted as well (e.g. Bold, italicized, colored etc.)
