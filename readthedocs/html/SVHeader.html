sv_param


The property sv_param is applied on the top-level module for SV haeader outputs. It converts the `define to the parameter inside the package.


Below is the example for the same:


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View



IDS-NG Spreadsheet View



SystemRDL


property sv_param {type = boolean; component = addrmap; };

 

addrmap block_name {


  sv_param = true;


  mem memory_name{

  mementries = 32;

  memwidth = 32;

};

  addrmap reggroup_name {

 

    reg reg_name { 

       regwidth = 32;

      field {

        hw = rw;

        sw = rw;


      } F1[31:0] = 32'h0;


    };

 

    reg_name reg_name @0x00;


  };

  memory_name memory_name @0x00;


  reggroup_name reggroup_name[2] @0x80;


};

svheader_pkg_name


To remove the generated package name in svheader output as per the requirement we introduced a  -top_property “svheader_pkg_name = “any_string””.


IDS-NG Input: 


SystemRDL Input:


 addrmap block1 {

   reg {

     field {

   hw=rw;

   sw=rw;

} f0 [0:0];

   } reg0 ;

};


svheader Output:


`define BLOCK_ADDR 'h0

`define BLOCK_SIZE 'h4

`define BLOCK_R2_ADDR `BLOCK_ADDR + 'h0

`define BLOCK_R2_DEFAULT 'h00000000

`define BLOCK_R2_F1_MSB 16

`define BLOCK_R2_F1_LSB 0

`define BLOCK_R2_F1_DEFAULT 'b00000000000000000


package block_header_pkg;


/*-------------------------------------------

Reg: BLOCK/R2


-------------------------------------------*/

typedef struct packed {

    logic [31:17] padding_31;

    logic [16:0] f1;

} block_r2;

typedef struct packed {

   block_r2 r2;

} block;

endpackage


Command line:


<idsbatch test.rdl -out “svheader” -if -preserve -top_property “svheader_pkg_name= “Any String””  -dir “out”>

Changing the generated output file name in SVHeader


In many scenarios, when generating a single file for svheader, it defaults to naming the output file after the input file. To address this, we've introduced support for the module_name property in the specification, allowing users to customize their output file names. For multi-output scenarios, we've already implemented support through a command line switch called -if_svheader, which generates output files named after the top modules used in the IP. 

Now, the module name property has been supported to customize the output file name for the svheader output.


Command Line:


<idsbatch test.rdl -out “svheader”  -dir “out”>


IDS-NG Register View:



IDS-NG Spreadsheet View:



SystemRDL Input:  test.rdl


property module_name { type=string; component=addrmap; };

addrmap block1 {

  module_name = "foo";

  reg {

    field {

      hw=rw;

      sw=rw;

    } f0 [0:0];

  } reg0 ;

};



SVHeader Output: foo.svh has been generated in the directory.
