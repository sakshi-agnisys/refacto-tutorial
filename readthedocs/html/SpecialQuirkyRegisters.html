Special / Quirky Registers

Special / Quirky registers

Training Video: Creating Complex Registers


Designer creativity, the demands of the application, or implementation constraints often require special register behaviors to be implemented. IDesignSpec™ can be used to model these special registers. IDS automatically generates the RTL and UVM models from their specification.


Here is the alphabetical listing of all the special register behavior that can be modeled using IDesignSpec™ :


  1. Alias Registers
  2. Counters
  3. FIFO
  4. Indirect register
  5. Interrupts
  6. Lock Register        
  7. Paged Register
  8. Paged Block
  9. RO, WO pair Register sharing the same address
  10. Shadow Register
  11. TMR
