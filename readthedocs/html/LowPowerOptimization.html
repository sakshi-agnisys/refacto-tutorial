Low Power Optimization


Training Video: Creating RTL Models


IDesignSpec™ can be used to generate Low Power RTL code. The code thus generated consumes low power because it eliminates assigning the same value at every clock edge or eliminating write operations all together. However, the generated code may take up more area due to the additional logic gates required to make the determination of a frivolous condition. Hence, in IDesignSpec™, user can choose if the low power RTL is to be generated in configuration window as shown in Fig.1



Please note that this feature may be available as an add-on option.


In the following paragraphs, we have explained how "Low Power Optimization" is achieved with respect to various software access using IDS generated RTL code.


  • Software Access : w0c (SW writes '0' to clear)


Non-Optimized Code

Code Optimized for Low Power Consumption

   

  if (wr_valid) 

        begin

               q<=q & (~(~wr_data[msb:lsb] & reg_enb[msb:lsb]));

        end

  

   if (wr_valid && (~wr_data[msb:lsb]) && ( |reg_enb[msb:lsb])) 

                begin

                    q <= q & (~(~wr_data[msb:lsb] & reg_enb[msb:lsb] ));

                end

 Explanation:  In low power optimized code, if all the bits of write data bus are '1', then according to software access mode nothing has to be cleared and hence unnecessary write operation is not performed.

Low power coding style with property rtl.write


Low power coding style for the property "rtl.write=swclear" with  ‘-lowpower’ switch in the command line or low power GUI option in the configuration window is enhanced  to eliminate assigning the same value to the register fields.



Example:        IDS-NG        IDS-Word        IDS-Excel        SystemRDL



IDS-NG input



SystemRDL input


property rtl_write {type = string ; component = reg;};

addrmap Block1 {

    reg TEST_WR1 {

    rtl_write = "swclear";

      field {

        hw = r;

        sw = rw;

      }sw_w1clr_function1[31:0] = 32'h12345678 ;      

    };

   TEST_WR1 TEST_WR1;

};


Generated Output


always @(posedge clk)  begin

if (!reset_l)

    begin

        TEST_WR1_sw_w1clr_function1_q <= 32'd305419896;

    end

else

    begin

    if (TEST_WR1_wr_valid && (|reg_enb [31 : 0] )) //SW_W1CLR_FUNCTION1 : SW Write

        begin

            TEST_WR1_sw_w1clr_function1_q <= (wr_data [31 : 0]  & reg_enb  [31 : 0] ) | (TEST_WR1_sw_w1clr_function1_q & (~reg_enb  [31 : 0] ));

        end

    else if(TEST_WR1_wr_valid_delay && (|TEST_WR1_sw_w1clr_function1_q))

        begin

            TEST_WR1_sw_w1clr_function1_q <= 32'h0;

        end

    end

end
