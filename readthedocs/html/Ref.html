Ref

IDS is user friendly, not just because it provides the user with various components, but also because it helps the user to reduce the efforts to reuse the already specified specification. This compelling capability is provided through the use of  "Ref" template.

The various properties of the Ref template are explained below:



Property

Description

ref name

A unique string that identifies the name of the ref template. 


  Example : Instance_name

offset

The address of the referred element in the address map is with respect to the address of the container. If the user defines the offset then value in the offset is the displacement added to the memory location of container. The value can be decimal value or a hexadecimal value.


Example : 10 or 0x10

name

A string that identifies the name of the referred element.The referred element can be any register, reg group, block or chip .


  Example : Read_write_reg

path

The path specified in the path field will take the referred element from that path. If the referred element lie in the same document then no need to specify path(keep the field blank), but if the referred element lie in some other document then user needs to specify the path.


Example : C:\Users\Agnisys®\Documents\Example_1.docx 


Note : 

  • The IDS file pointed to by the path property can be either an XML/XRSL(generated by IDS), IPXACT, SystemRDL or another .DOCX file with IDS content.  
  • The document area of the "ref" template can also be used to specify properties.


Properties specified in Ref template


  • Any property attached to the ―Ref template will override the property in the template referred to. Note that the offset property is never replicated because user can always specify the offset in the Ref template.

  • Possible ways of referring to an element

Following are the possible ways of referring to an element:


  • Element in same Spreadsheet

When a user wants to refer to an element already specified in the same design sheet of Excel/CSV file, then only the element's name needs to be included under the 'ref' column and no need to specify the path of the element (see figure above for reg_3 and block2)


  • Element in different Spreadsheet of same Excel file

It may be possible that user wants to refer to an element present in a different spreadsheet of the same Excel file(same workbook). In this case user needs to include the element's name under 'ref' column and specify the sheet name in which the element is present under the 'ref_path' column (see figure above for block3)


  • Element in different Excel

Another possibility arises when the element to be referred is present in an absolutely different Excel file (different workbook) or different CSV file. In this case, user will have to define the path of the external file in which element exists in the 'refpath' column as shown in following figure.

General Points regarding Ref: 

  • Ref path can be absolute, e.g. C:\Users\user1\data\Design2.xlsx:spec1 or it can be relative with respect to the current file, e.g. .\data\Design2.xlsx:spec1.
  • For an element in an external excel file, path specified must include the name of the spreadsheet in which the element is present (e.g. in above figure, bk1 element is present in spreadsheet named 'spec1').
  • The reference can be made to other Register, RegGroup, Block or Chip templates using their name. The first name that matches will be referred. 
  • When a reference is made to a container template (a template that creates a hierarchy – e.g. RegGroup, Block, Chip), all the contents of the referred template are replicated as well. 
  • Any property attached to the ―Ref template will override the property in the template referred to. Note that the offset property is never replicated because user can always specify the offset in the Ref template. 
  • The element being referred should not be further referring to any other element.


With Refs, it is possible to create multiple document dependency. So one document points to another, and that document points to yet another. Care should be taken not to create a dead-lock or a circular reference, otherwise Excel may hang or an error may result. 



Ref=0 Property


"ref = 0" property is used to remove all the components having the property while referring one specification in another. This property can be applied on any component such as field, register, section and a block. Its effect will be seen when the specification or component containing the property will be referred in another document, in this case the component containing the property will disappear/removed from the document in which the file has been referred. For example: if the property exists on the field, all the fields under the register will exist except for the field on which the property has been applied. Similarly, if the property is applied on any register file, then all the register under that section will also disappear.
