16. Dynamic names with hexadecimal values


IDS supports dynamic names with decimal and  hexadecimal values


  • Dynamic name by using eval can be used to set component names dynamically
  • It can be used any property value to assign that property value dynamically
  • We have to extend Dynamic names with values in hexadecimal returns
  • To return value in hexadecimal, use hex() inside eval():-
    1. eval(‘some string’+hex(<Integer value>))

For Example : eval( ‘reg_name’ + hex(%r.address) ) then reg name would be reg_name_0x0

IDSNG

Note:- 

  • To append string in eval(), use single quptes(‘). 

For example,

eval(‘reg’+’name’ )

  • To get value using in hexadecimal, Pass only integer value inside hex(). 

For example,

eval(‘reg’+’name’+hex(%r.address))

eval(‘reg’+’name’+’%d’)

17. -include_css


While generating HTML outputs, IDS generates CSS files separately. These CSS files are included in HTML files wherever required. Users may sometimes need to avoid generating the CSS files separately and may instead want their contents to be copied within the HTML files themselves.

    • Purpose: To allow users to choose not to create external CSS files for HTML outputs.
    • Scope:
      • This feature is applicable for HTML and HTML-ALT2 outputs.
      • This feature will be available for IDS tools like:- IDS-Batch, IDS-Word, IDS-Excel, IDSNG
      • This feature is provided through a command line switch, “-include_css”.
  • Overall Description
    • By using -include_css,IDS  will not create any external CSS file and supporting images directory for HTML outputs rather it copies the stylesheets within HTML files.
    • By default it will create an external CSS file along with the supporting files and outputs files to ensure backward compatibility.
    • For using this switch in GUI, user will insert this switch in command line option to ignore creating external CSS file and supporting images for HTML output
    • This switch can also use in TCL interface API by using set_switch function
  • For example 
    • To use it IDS-Batch command line would be :-

IDSbatch <inputfile> -out htmlalt2 -include_css

18. IDS Import


In IDS-NG, user can either import IDS-files or YAML file. 


1. IDS files

IDS-NG is capable of importing register data from various formats. It will automatically create a document from the imported data.


Import can be done by using ‘IDS Import’ button. 

      

Following import formats are supported in IDS-NG:

    • IDS-XML
    • IP-XACT
    • System-RDL 
    • RALF
    • IDS Excel
    • IDS Word
    • IDS CSV
    • IDS YAML


First user can select file type they want to import, then they can browse or copy-paste the location link of particular file. There is an additional option to import the complete configuration as well if we click on the check box as shown in the following snippet:

 

                         

2.  YAML File


To import or update param, user must switch to the param view. For switching, user must first create an IP or Register Spec, which will be overwritten as soon as he/she imports param.

    • Click on “Import Param” or “Update Param” and select the .yml file.



    • Similarly, to update param, choose ‘Update Param’ option.

Support of Author Name


The "created by" row was generated to be empty in the current IDS flow till now, meaning that the author name used to be missing from the output-generated file. The created_by field will contain the name of the person who created the register specification.

In order to clarify who the author of the generated output is, a top property called "created_by" has been implemented that accepts a string as input and appends it before the "created by," which was not done previously.


Command line


idsbatch sample.rdl -out verilog -bus apb -dir ids -top_property “created_by=”string abc xyz” ”


Examples: IDS-NG        IDS-Word        IDS-Excel        SystemRDL
