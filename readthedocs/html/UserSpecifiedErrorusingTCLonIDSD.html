User Specified Error using Tcl on IDesignSpec™ Document 


TCL scripts may be used to customize outputs generated by IDesignSpec™. They can be used to create custom outputs or to specify custom errors. For example, in case illegal component names used.


For Example: Error in case an illegal register name is specified. 

"set_error.tcl script"


The following TCL script (set_error.tcl) will check for any register named “reg_name” and output an error if that condition is violated:   


proc set_error_msg { elem } { 

set elem_type [get_type $elem] 

switch $elem_type { 

system{ 

set children [get_objects $elem] 

foreach child $children { 

set_error_msg $child 

} 

} 

board { 

set children [get_objects $elem] 

foreach child $children { 

set_error_msg $child 

} 

} 

chip { 

set children [get_objects $elem] 

foreach child $children { 

set_error_msg $child 

} 

} 

block { 

set children [get_objects $elem] 

foreach child $children { 

set_error_msg $child 

} 

} 

reggroup { 

set children [get_objects $elem] 

foreach child $children { 

set_error_msg $child 

} 

} 

reg { 

set regName [get_prop $elem name] 

set aliasProp [get_prop $elem alias] 

set shadowProp [get_prop $elem shadow_reg] 

if {[string match -nocase $regName reg_name] == 1} { 

set_error $elem "The name of the register should not be regname; " 

} 

if {$aliasProp != ""} { 

set_error $elem "alias property has been applied on the register; " 

} 

if {$shadowProp != ""} { 

set_error $elem "Shadow_reg property has been applied on the register; " 

} 

} 

} 

} 

set top [get_top] 

set_error_msg $top

In IDS-Word or IDS-Excel, one or more TCL files can be loaded by specifying the local directory and by selecting the TCL file name with a checkbox. In this example the local directory is C:\Local_dir, and the available files are listed with check boxes in the Local Path Outputs box. 



The check on the TCL specified restrictions will be done when the user clicks on the “Generate” button in the IDesignSpec™ ribbon. The error will be displayed in a popup, and the user must click on the + box next to “Error” to display the specific error message(s). The error in this case is only for the complete register name “reg_name”. The register name “reg_name1” does not register as an error even though the name partially includes this string. Similarly, if a register contains alias or shadow_reg property then the error pops-up indicating the same. 



In IDS-Batch, the command line argument to utilize the TCL file (for this specific example) is as follows (for a Windows environment): 


IDSbatch -tcl C:\Local_dir\set_error.tcl Example.docx 


For this example, the following command line will run IDS and will include the set_error.tcl script to check for any register named “reg_name”. 


IDSbatch -tcl C:\Local_dir\set_error.tcl Example.docx -nocache 


The output to the screen will be as follows: 


<CMD> IDS-Batch -tcl C:\Local_dir\set_error.tcl Example.docx -nocache 

ERROR-A:The name of the register should not be regname; at -> /block(block_name)/reg(reg_name) 

ERROR-A:alias property has been applied on the register; at -> /block(block_name)/reg(reg_name1) 

ERROR-A:Shadow_reg property has been applied on the register; at -> /block(block_name)/reg(reg_name2) 

ERROR-B : Above custom checks must fix before output generation.
