IDS-Excel


Design teams often use MS-Excel to capture register information. IDS-Excel offers an automated way to generate design, verification, firmware code, as well as documentation from it. IDS-Excel is offered as a plug-in for MS-Excel. Besides MS-Excel, a similar application is available for MS-Word, in addition, a standalone batch utility called IDS-Batch CLI is also available.


First of all, download and install the IDS-Excel plug-in from Agnisys® website. For installation details refer to the topic "Installation Steps > IDS-Word or IDS-Excel" in "Agni Tools Installation". In addition, download the Supporting IDSDocs which contains sample examples.

Chip : A chip can be specified as shown in figure above.


Name

The displacement of the chip in the memory from the address of the container(for a chip the container is board). Can be specified in decimal or hex.

size of the chip

Description of the board. Description can also contain the properties, having syntax {property1_name=value; property2_name=value}

Block : A block can be specified as shown above in figure above.


Name

Description

block

Name of the block, i.e. the string that identifies the element in the specification

block offset

The displacement of the block in the memory from the from the address of the container (for a block the container template is chip). Can be specified in decimal or hex

block size

Size of the block

block description

Description of the block. Description can also contain the properties, having syntax {property1_name=value; property2_name=value}  

Ex:{coverage=on; clock_edge=posedge}

Note: properties should be separated by semicolon “;”.

block constraint

Enum or constraint for the entire block is defined here. Enum is defined in curly braces “{}” for example {0:false,1:true}.



Register : Register can be described as shown in figure below:




 Name

Description

register

Name of the register, i.e. the string that identifies the element in the specification

reg offset

The displacement of the register in the memory from the address of the container (for a register, container template is block/section)

reg width

Different width registers can be specified using reg_width. For the registers with different width from the one selected in "Configuration Settings"  are required to be specified in this column

reg instance

Number of times the register is repeated in the memory

reg external

True or 1 if the register is external

reg constraint

Enum or constraint for the entire register (i.e. for the constituent fields) is defined here. 

Enum is defined in the curly braces “{}” Ex: {0:false,1:true}

reg description

Detailed description about the register is mentioned here. This column can also contain the properties having syntax {property1_name=value;property2_name=value} 

Ex: {coverage=on; reset_type=async}.

field

Name of the field, i.e. the string that identifies the element in the specification

bits

Bits of the field

sw access

IDS registers can be accessed from the external bus like AMBA, Avalon etc. or any proprietary bus. This field indicates the policy of software access.

hw access        

IDS registers can be accessed from the hardware within the block. This field indicates the policy of hardware access.

field default

Default value for the field. Field default is mandatory

field description

Short field description can be added here. This column can also contain the properties having syntax {property1_name=value;property2_name=value} 

Ex: {coverage=on; reset_type=async}.

field constraint

Enum or constraint for the field (i.e. for the constituent fields) is defined here. 

Enum is defined in the curly braces “{}” Ex: {0:false,1:true}.



RegGroup/Section : A section or RegGroup can be specified in a way shown in the figure below:




Name

Description  

section name

Name of the section, i.e. the string that identifies the element in the specification

section size

Size of the section

section offset

The displacement of the section from the address of the container (for a section the container template is block)

section count

Number of times all the contents in section are repeated

section external

True/1 if section is external

section description

Section description can be added here. This column can also contain the properties having syntax {property1_name=value;property2_name=value} Ex: {coverage=on; reset_type=async}.

end section

Section is terminated with “end section” tag. If section is used then this is mandatory. This tag should be included in the same column as  section_name



Thus, using IDS-Excel plugin for MS-Excel, users can customize the data formats and generate suitable code and documentation from it without leaving the application. This helps speed up the design and verification of chips, FPGAs and IPs.
