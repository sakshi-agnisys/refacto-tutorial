Ver 6.32.0.0

v6.32.0.0 

(Jan 21st,2020)



Note: 6.30.0.0 has been withdrawn due to slow speed


IDesignSpec™(IDS)


RTL Enhancements:

  • AXI4Lite and APB aggregation logic has been supported. (More Details)
  • "buffer_trig_reg" property on trigger buffer is supported in VHDL, where the triggering event is a register. (More Details) 
  • "buffer_trig_reg" property is enhanced to take signals as read/write trigger, it is also supported in case of repeat on register in verilog. (More Details)
  • "rtl_port_suffix" property is supported used to change the predefined port suffix. (More Details)
  • "lock" property is supported even in case of a block with different register widths. (More Details)
  • "tmr_error=true" has been supported in TMR register for error detection functionality. (More Details) 
  • "clock_enable" property has been enhanced by adding a fourth argument ‘default’ by which the clock gating logic will be automatically generated and instantiated in IDS block. (More Details)
  • "out_enb_stages" property to add delay stages in out enb port is now supported in verilog. (More Details)
  • "ext_decode_add" property has been supported on all external components to send output port the address on which read/write operation is performed. (More Details)
  • "sv_interface=struct” property is supported at block level which creates struct instead of interfaces in SystemVerilog. (More Details)
  • "rtl.byte_enable" property has been supported for all buses to remove the byte enable signal. (More Details)


SystemRDL Enhancements:

  • SystemRDL 2.0 compiler has been made the default compiler, the command line switch "-rdl_version 2a" has been removed. To switch back to the SystemRDL 1.0 compiler, the switch  "-rdl_version 1" can be used. (More Details)
  • Embedded Perl preprocessor is natively supported in IDS-Batch, the dependency to download perl package has been removed. (More Details)
  • {swwel=true} property to make the software write enable active low is supported in Verilog. (More Details)
  • {wel=true} property to make hardware write enable active low is supported in Verilog . (More Details)
  • {rsvdset=true} property which states the read value of all the fields which are not explicitly defined are set to 1 is supported. (More Details)
  • {rsvdsetx=true} property which states the read value of all the fields which are not explicitly defined are set to unknown is supported. (More Details)
  • hdl_path_gate_slice property has been supported which assigns a list of gate-level hdl_path for a field in UVM. (More Details)
  • errextbus: This property which specifies that a transaction is terminated with an error is supported on all the external components. (More Details)


UVM Enhancement:

  • Cross Coverage is supported in UVM RAL. (More Details)


CHeader Enhancements:

  • C files are POSIX compliant.
  • Structs are now supported in C headers. (More Details)
  • Time stamping added in the generated '.c' files header. 
  • Following format changes has been done in generated output:
    • altered the names of the local variables and constants that are automatically generated by IDS in camelCase
    • all source files now ends with an empty line
    • unwanted whitespaces have been removed 


SVHeader Enhancements:

  • Structs are now supported in SV headers (More Details)


General Enhancements:

  • Additional software accesses like r/wor, r/wand, wand, wor are supported in Verilog and UVM. (More Details)
  • Following enhancements have been made in C# output: (More Details)
    • Memory is now supported
    • If register and field names are same then, names are generated in hierarchical order in the output  
  • “reqid=<user_defined_ID>" property has been supported which allows user to make unique group for identification based on ID in HTML output. (More Details)
  • "add_doc=<souce item> and add_doc_above=<source item>" properties has been supported which allows user to add any external item between the templates and the same will be reflected in HTML-alt2 and pdf outputs. (More Details)
  • "-python_api" switch has been supported to generate Python HAL. (More Details) 
  • The "ispresent" property is enhanced to generate macros-based output to change the presence of a component at compile time. (More Details)
  • "start_array_index=<integer>" property has been supported, to append the name of the repeated component with desired integer value. (More Details)
  • Struct template for IDS-Word input is now supported. (More Details)
  • PDF files can be imported in IDS-Word. (More Details)
  • If user specifies any property, then description regarding that property is reflected in the description of HTML-alt2 output.(More Details) 
  • Checkbox for "out_third_party" has been added in GUI.
  • "ref = 0" property has been supported to remove all the components having the property while referring one specification in another. (More Details)
  • Command line option "-dir_out_specific" is supported to direct the outputs to specific folders. (More Details)
  • Standard library for IPs, named AGNI Library,has been created in IDS-Word. It includes following IPs:
  • GPIO: General Purpose Input/Output is a standard interface which is used to connect microcontrollers with other devices. (More Details)
  • I2CM: It is a serial protocol for two wire interfaces to connect low speed devices like microcontrollers, A/D and D/A converters, EEPROM, I/O interfaces and other similar peripherals in embedded system. (More Details)
  • TIMER: It provides the timing information which includes pulse width, period of the clock by using external events. (More Details)
  • PIC: Programmable Interrupt Controller is an external device which takes interrupt sources from various peripherals and depending upon their priorities route them to one or more CPU lines. (More Details)


Bug fixes:

  • Fixes done for bus width less than register width with interrupts on repeated section.
  • Bug fixes done for "module name" property while referring a block inside chip.
  • Address decode logic is now fixed in case of external register.
  • Fixes done in "rtl.name_format" property for SV output.
  • Fixed font size issue of company name in footer for PDF outputs.
  • Fixed memory flat issue (in case of repeat) in Table of content of HTML output.
  • Fixed large empty spaces inside description for HTML-alt2 output.
  • Fixed the problem of ToC not being generated for HTML-alt1 output.
  • Fixed non generation of index for HTML-alt2 output.
  • Fixed line return issue for PDF and HTML-alt2 outputs.
  • Fixes done in IDS-Word for using enum table along with memories.
  • Fixes done in address calculation of stride for RDL input.
  • Fixed bugs in offset calculation with respect to address unit for HTML-ALT2 output.
  • Fixed line return and square brackets issue in UVM parameter.
  • Fixed hyperlinks inside description cell for HTML-alt2 outputs, which allow users to add hyperlinks inside description of input fille.
  • Volatile property is supported in RALF input.

Automatic Register Verification (ARV™)


ARV™-Formal Enhancement:

  • Delay properties such as rb_valid_stages, rb_data_stages and addr_decode_stages are supported. (More Details)
  • Repeat is supported on register component.


ARV™-Sim Enhancement:

  • Wave Dump has now been supported in ARV™. (More Details)
  • WISHBONE bus is supported in ARV™.
  • Enhancement done in counter sequences.
  • Lock sequences have been fixed. 
  • The following properties are now supported: rtl.default , rtl.hw_w0p, rb_valid_stages, rb_data_stages, addr_decode_stages, reg_prot, reg_report and dontcompare.



Specta-AV™ 


Specta-AV™ is a comprehensive UVM Testbench Generator for IPs/SOCs. A tool that automates verification using an industry-proven code generation technology. With the ability to parse hierarchical register specification from IP-XACT, System RDL, Word or Excel, and the ability to retarget complex sequences into various modelling languages such as System Verilog, Specta-AV™ facilitates a methodology where multiple SoC groups can align and work from a golden specification for autogenerating UVM Tests/Environments/Agents.

It provides the best framework to generate complete UVM testbench including sequence items, configurations, checkers, coverage and even the plumbing within UVM automatically - and all this not just for addressable registers but for application logic as well. (More Details)



IDesignSpec™ (ISS)

Enhancements:

  • Constraints and randomization are now supported in 'firmware' output. (More Details)
  • Hardware pins and signals now can be passed in loops and conditional statements.
  • A new output 'Platform' has been created for board prototyping. (More Details)
  • 'uvm_door' property has been made hierarchical.
  • Added new property 'iss_uvm.package' to change the name of the package in UVM. (More Details)
  • Added new property 'uvm.verbosity' in UVM to change the verbosity level in 'display' command. (More Details)
  • 'Call' command supported in Python input.
  • Following changes have been done in SV output:
    • Restructured read/write on registers with/without consolidation. (More Details)
    • Supported concept of parallelism (More Details), structures (More Details), arrays of variables (More Details) and write on repeated registers (More Details).
    • Generation of new files: 'maskfile.svh' which contains masks for fields and 'struct.sv' which will only be generated while using structs.



Bug Fixes:

  • Bug fixed for using constants, loops and conditional statements in MATLAB output.
  • Bug fixed while using multiple structures in 'Struct' sheet.
  • Bug fixed for multiple randomization (with/without constraints) on same register during the course of execution of sequences.
  • Bug fixed for 'for' loop, 'while' loop, subsequence call, usage of variables and reading a register in SV output.
