IDesignSpec™ (IDS)


RTL Enhancements


  1. F#16704 - Support of custom string in lock expressions of a field using eval() expression in the verilog output. More Details   
  1. F#19998 - Support of  “sharedextbus” property along with “external_ack=optimized” property in the verilog output. More Details
  1. F#16405 - Introduced a new property "enum_remove" to remove the unused enums from the RTL. More Details


     

General Enhancements


  1. F#15957 - Support of a new switch “-third_party_rtl“ to stop the generation of hardware interface and its instantiation in case of third party RTL. More Details
  1. B#1650 - Support of a command line switch “-capi_return_type <int/hwint>“ to generate an int/hwint type write API in the generated C APIs. More Details
  2. B#1661 - Support of a new top property “spec_ver” to add corresponding specification/register input design version in IDS generated outputs for version control of the specification. More Details
  3. B#1592 - Enhancement for issue in case of block in Address macro the SIZE macro is replaced by PER_INSTANCE_SIZE macro when cheader_opt=true is used. More Details
  4. F#16178 - Support of new property in C-Header “cheader_map_notpresent”. More Details  
  1. F#19566 - Support of multiple default values in TOC for HTMLalt2 & PDFalt4. More Details


UVM Enhancements


  1. G#JAVA#421 - F#16178 - Support for configure IP/Block Offset in the default_map or the map that the user wants to create and configure accordingly.
  2. F#16178 - Support of new property in UVM, namely "uvm_map_access" and “uvm_map_notpresent”.  More Details
