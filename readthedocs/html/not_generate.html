not_generate

This property is used to disable generation of UVM and C-header(headeralt4) chip/block components on which it is applied. In UVM and C-header the instance of chip/block remains available to its hierarchically top files so that the user can provide its own code block for that void. But for Verilog RTL this flow is useful in SoC scenarios where user does not want to generate Verilog RTL code for some of the chip/block (use third party RTL or reuse existing chip/block Verilog RTL instead) but requires generation of DV collaterals (UVM/headers/HTML) for all chip/block in the specification.So in Verilog RTL the instance of the disabled code also gets disabled.  


SystemRDL Input:


property not_generate {type=string; component=addrmap;};

property chip {type=boolean; component=addrmap;};


`include a_csr.rdl

`include b_csr.rdl


addrmap c_csr {

   reg regA{

     field {}f1;

   };regA regA;

};


addrmap chip1 {

   chip = true;

   a_csr a @0x0000_0000;

   a->not_generate="verilog,uvm,cheader";

      b_csr #(.PARAM(true)) b1 @0x0010_0000;

      b_csr #(.PARAM(false)) b0 @0x0020_0000;

      b0->not_generate="verilog,uvm,cheader";

      c_csr c @0x0030_0000;

};


addrmap top {

   chip=true;

   chip1 chip1;


};

VHDL Support for not_generate property


This property is used to disable generation of VHDL code of components on which it is applied.


SystemRDL input:


property not_generate {type = string; component = addrmap;};

property chip {type = boolean; component = addrmap;};


addrmap chip1 {

  chip=true;

  addrmap block1{

         reg reg1{

                 field f1{

                         sw=rw;

                         hw=rw;

                }f1[31:0]=0;

        };reg1 reg1;

};

 block1 block1_1[2];

  block1 block1_2[2];

 block1_2->not_generate="vhdl";

  block1 block1_3[2];

  block1_3->not_generate="vhdl";

 block1 block1_4[2];

 block1 block1_5[2];

};

Improvements in property not_generate


The property "not_generate" is used for disable class generation in the UVM,verilog and cheadar  for a specific component, but it doesn't remove the instance of the chip or block from higher-level files. This allows you to add your own code to that part as needed. With this enhancement we improvise this property so now the whole component will be removed from the output file with all instances related to that component will  be not generated.
