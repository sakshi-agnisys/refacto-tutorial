Ver 6.38.0.0

v6.38.0.0 

(May 31st,2020)


IDesignSpec™ (IDS)


RTL Enhancements:


  • AMBA3AHBLite bus is supported for VHDL output at block level
  • ‘external_ack’ property has been enhanced with an option “optimized”, to merge multiple instances of external registers or sections into a single instance, thus reducing the area (More Details)
  • ‘rtl.name_format’ property is supported on register, section, memory, and block level
  • In virtual register, selection of the alternate mode of a register based on the value of a field of some other register’s field value is supported (More Details)
  • ‘partial_access_err’ property is introduced to acknowledge the partial access of the bus transfer (More Details)
  • ‘invalid_addr_err’ property is introduced to generate error for invalid address based on read/write transaction (More Details)
  • Tile-Link 1.8 bus is supported at chip-level
  • For paged register, limitation of specifying page_count in exponents of 2 has been removed and ‘page_wr_select’ and ‘page_rd_select’ properties are introduced to have two separate registers to control the read/write mux (More Details)
  • Wishbone bus is supported on chip-level with and without repeated blocks in verilog output
  • Different ‘bus_width’ is supported in multiple blocks for the same bus interface



SV Enhancements: 


  • VHDL is supported on chip-level for ARV™
  • Structs are supported for Tile-Link bus
  • In ‘sv_interface = struct’ property, following enhancements have been done:
    • Support for repeated block (More Details) 
    • Created chip wrapper for chip-level (More Details) 
    • Struct signals in wrapper module are connected using hierarchical names at block and chip-level
  • The property ‘rtl.name_format’ is supported on register, section and block level (More Details)



UVM Enhancements:


  • VHDL is supported on chip-level for ARV™
  • Memory inside section is supported 
  • ‘uvm.reg_access’ property is supported for all software accesses
  • ‘uvm_guard_band’ property is enhanced to remove default header guards from the UVM classes
  • Different ‘bus_width’ is supported in multiple blocks for the same bus interface



SystemRDL Enhancements:


  • UDP, refPath=”<name_of _the_external_file>:ComponentName”, has been added in RDL compiler version 2.0 to instantiate the instances of block/section/register from other files (More Details) 
  • Dynamic assignment of reset for multiple instances of the register is supported using SystemRDL input to generate UVM RAL model 


General Enhancements:


  • Introduced a command line switch, -hide_pdf_prop all, to hide properties in the description of PDF output (More Details) 
  • ‘Signals’ passed as ‘default values’ can be interpreted in HTMLalt2 and PDF output (More Details) 
  • ‘doc_unused_remove’ property is introduced to remove the component in PDF and HTMLalt2 output when ‘ispresent’ and ‘is_rsv’ property is applied (More Details) 
  • TCL based configuration API is supported in IDesignSpec™ for runtime configuration instead of using switches at command line (More Details) 
  • Supported AI based sequence detection in IDS-CLOUD (More Details) 
  • Added information of AGNILM_LICENSE_FILE and LM_LICENSE_FILE environment variables in ‘About’ section for IDS-Word and IDS-Excel (More Details)
  • ‘-dir_out_specific’ command line switch is enhanced for HTMLalt2 output to direct output in specific output directory
  • C-tests can be generated automatically which includes random value tests, walking one tests, write/read 0’s and 1’s tests and tests for all special software accesses (More Details)  
  • Supported multiout output for VHDL output in IDS-Word
  • A new switch, ‘-config_template <file>’, is introduced where file is in yaml format and it can contain:
    • ISS output configuration template
    • C APIs to customize C-tests


     Bug Fixes:


  • Variables declared inside embedded perl-preprocessors can be used inside RDL input file 
  • Fixes done for linting issue with ‘external_ack’ property
  • Fixes done for signals not being declared as vectors in verilog output when ‘-no_lint_warn’ switch is used
  • Fixes done for incorrect ‘error’ logic in case of the absence of empty space in the register map for verilog output
  • Fixes done for 3-way aliasing for verilog output
  • Fixes done for ‘*_rd_data_in’ not being declared as vector in system verilog output
  • Fixes done for 'module_name' property at chip level in verilog output
  • Fixes done for incorrect request logic while using ‘external_ack’ property and only readable registers in the specification for verilog output
  • Fixes done for declaration of signals for external component (section) in deep hierarchy for system verilog output
  • Fixes done for ‘addressing’ property when it is used along with ‘external_ack’ property in a specification containing repeated registers inside repeated sections in verilog output
  • Fixes done for ‘resetsignal’ property in UVM output
  • Fixes done for paged register in case of repeated component for UVM output
  • Fixes done for UVM property ‘config_hdl_path’ for repeated registers and sections
  • Fixes done for ‘-vertical_reuse’ switch in case of ‘rdl_define’ property
  • Removed ‘-turbo’ option from the command line
  • Fixed IP-XACT validation error for IP-XACT 2014 file
  • Removed dependency on ‘-nocache’ option for ‘.docx’ input files in IDS-Batch
  • Multiple underscores ‘_ _’ can be handled in hdl path with -preserve option
