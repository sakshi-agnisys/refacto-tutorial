RegAlign


RegAlign states “it specifies the components are packed so each component’s start address is a multiple of its size (in bytes)”. Array elements are aligned according to the individual element’s size (this results in no gaps between the array elements).


It supports when register width less than and equal to (register size) bus width(i.e regwidth <= buswidth).

So from the definition of RegAlign,


          Register with width 8 bits can have starting address

                      0,1, 2, 3, ……….              Where 0, 1, ... are byte number

          Register with width 16 bits will have starting address

                      0, 2, 4, ………….               Where 0, 2, … are byte number

          Register with width 32 bits will have starting address

                      0, 4, 8, ………….               Where 0, 4, … are byte number

 

Example:


If we have 3-register width of 8, 8, 16 with bus width 32, then read and write operation will work as:


8-bit register [0:7]

8-bit register [8:15]

16-bit register  [16:31]


All registers will be accessed in one clock cycle.


Examples:


1) SystemRDL


          addrmap b1{

                    addressing = regalign;

                    reg reg1{ 

                       regwidth = 8;

                       field {

                       } fld[7:0];

                     };

            

               reg reg2{

                       regwidth=8;

                         }fld1[7:0];

                     };

             

               reg reg3{

                        regwidth = 16;

                        } fld2[15:0];

                   };


       reg1 reg1;

       reg2 reg2;

       reg3 reg3[4];


  };

NOTE:


1.     Register width should be less than or equal to busWidth.

2.     Starting address of RegGroup should be multiple of bus width and the total size of iteration of one regroup should be a multiple of bus width (in bytes).

3.     Property should be assigned on top level.

4.     Not supported for Multiple Bus Domains.
