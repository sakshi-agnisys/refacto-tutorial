hdl_path & hdl_path_internal property example

Introduction


In IDS, the property “hdl_path” is used to generate the UVM hdl_paths for registers and memories. The value of “hdl_path” property defines the hierarchical path of registers and memories inside the spec.

The structural hierarchy of registers and memories in the spec may vary from their actual design hierarchy, as memories and external registers have their implementation outside the design generated from IDS. So for defining the hdl paths of internal registers in a spec containing memories, external and internal registers, another property “hdl_path_internal” is added. This property is used to specify hdl path for internal registers only and if specified on a block level, then the specified path will be prepended to the hdl paths of all registers within that block. But this property will not prepend the path to any external register or memory within that block. User have to specify the complete hdl_path for the external register or memory by using the property “hdl_path”.

Example showing usage of properties hdl_path and hdl_path_internal 

For example, in the below Spec  the block “Block1”  has a property “hdl_path_internal”  and the specified path “A.X.Y” will be prepended to the hdl paths of registers “Reg2” and “Reg3” as these are internal registers. But the path “A.X.Y” will not be prepended to the hdl path of memory “Mem1” and register “Reg1” since both have their implementation external to IDS. The complete hdl path for “Mem1” and “Reg1” is specified by the properties “hdl_path = A.B.C.mem1” and “hdl_path = A.B.C.reg_q” respectively.

 Ignore_prop


For ignoring hdl paths of particular internal registers  in a spec, when “hdl_path_internal” property is used at block (containing memories, external and internal registes), another property “ignore_prop” is added. This property is used to prevent  the automatically prepended hdl path for internal registers.  User have to specify the complete hdl_path for the that internal register on which this “ignore_prop” is used.There is no need to used this property for internal register on which “registered=false” is used.

Removal of callback class using ignore_prop


UVM generates the post_predict with “uvm_path_e path” enum which is incompatible with the standards IEEE 1800.2-2020. Further, for interrupt registers, uvm_reg_cbs classes are generated by idsbatch, but there is no alternative available for removing it. 

The interrupt-based callback class removal has been implemented using the property “ignore_prop”. With the use of this property, the callback classes defined in case of interrupt registers will be removed for the respective interrupt registers.

SystemRDL Input

Disable parity callbacks in UVM RAL


Callbacks in UVM RAL provide flexibility and customization options for register modeling and verification, allowing you to adapt the behavior of your register models to the specific requirements of your design and verification environments.

class parityCB  extends uvm_reg_cbs;

        uvm_reg_field parity_f;

        string parity_t;

        uvm_reg rg;

        uvm_reg_data_t mask_data;

        int parity_pos;


        function new(string parity_type, uvm_reg_field parity_field, uvm_reg_data_t mask, int pos);

            parity_f = parity_field;

            parity_t = parity_type;

            mask_data = mask;

            parity_pos = pos;

        endfunction


        virtual task pre_write(uvm_reg_item rw);


            rg = parity_f.get_parent();

            if (rw.kind == UVM_WRITE ) begin

                case(parity_t)


                    "even" :

                    begin


                        `uvm_info("PARITY_EVEN", $sformatf("DATAREG %s PARITYREG %s PARITY %b DATA %h ",rw.element.get_full_name(),rg.get_full_name(),(^(rw.value[0] & mask_data)), (rw.value[0] & mask_data)), UVM_LOW);


                        if(rg.get_full_name()!= rw.element.get_full_name()) begin

                            void'(parity_f.predict((^(rw.value[0] & mask_data)), -1, UVM_PREDICT_DIRECT,rw.path, rw.map ));

                        end

                        else begin

                            rw.value[0][parity_pos] = (^(rw.value[0] & mask_data));

                        end

                    end


                    "odd" :

                    begin


                        `uvm_info("PARITY_ODD", $sformatf("DATAREG %s PARITYREG %s PARITY %b DATA %h ",rw.element.get_full_name(),rg.get_full_name(),~(^(rw.value[0] & mask_data)), (rw.value[0] & mask_data)), UVM_LOW);


                        if(rg.get_full_name()!= rw.element.get_full_name()) begin

                            void'(parity_f.predict(~(^(rw.value[0] & mask_data)), -1, UVM_PREDICT_DIRECT,rw.path, rw.map ));

                        end

                        else begin

                            rw.value[0][parity_pos] = ~(^(rw.value[0] & mask_data));

                        end

                    end

                endcase

            end

        endtask

    endclass

Summary

Thus by combining the properties “hdl_path” and “hdl_path_internal” we could specify hdl paths for internal registers, externals register and memories all of them specified in a single IDS Specification. However, to ignore hdl paths for internal register, IDS introduces ignore_prop, which can be applied at that particular register. This can also be used in case of virtual register.

 


Expressions in hdl_path


The hdl_path expression can be used to modify the hdl_path of the repeated elements. Following are notations that can used to modify hdl_path:


'%s.size' denotes size of parent element 

'%s.offset' denotes offset of parent element 

'%s1.size' denotes size of 1 hiearchy above the parent section 

'%s1.offset' denotes offset of 1 hiearchy above the parent section 

'%s2.size' denotes size of 2 hiearchy above the parent section 

'%s2.offset' denotes offset of 2 hiearchy above the parent section 

'%r.size' denotes register size

'%r.offset' denotes register offset 

'#1' denotes first repeated element

'#2' denotes the second repeated element
