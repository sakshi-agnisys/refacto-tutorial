In this above design specifications, the variant table have four variant name v1 value 0, v2 value 0, v3 value 1 and v4 value 1. When "refvariant" property is present, then current document variants overrides the variant list of the referred document, along-with the referred variant using "refvariant" property is set 'true'. When "refvariant" property is not present, then implicitly current document variants overrides the referred document variants.


Note: Top level variant template override the value of the variant used in the referred document.
