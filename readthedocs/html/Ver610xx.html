v6.10.0.0 - Major Release

28th December 2017

  • RTL Enhancements
  • Register Aliasing - Supported aliasing across different blocks. The aliasing register and the aliased register can now be in different blocks. More Details
  • Virtual Register- IDS now supports Virtual Registers. It consists of two registers at the same address location and based on the execution mode, one register is selected. More Details
  • Non-aligned memory support- Now IDS will generate address decode logic for unaligned memory. Earlier, in order to optimize, IDS gave error in case of unaligned memory, now only a warning is generated.
  • Vertical_reuse- This property is used at chip level in Verilog to remove busdomain name in prefix of bus signals(e.g. axi4full_ARESETN) and generates relative offsets in order to make the RTL “relocatable”.
  • Registered=false:nomux : With the use of this property, the mux which was earlier generated inside the IDS block, will not be generated. More Details
  • Clock_enable - This property will enable the write on register depending on the access of the field More Details
  • Wr_rd_valIDS - This property will create the rd_valid and wr_valid port of internal register group. rd_valid will have the ORing of all the registers  inside that reggroup. wr_valid will be the ORing of all the write valIDS of all the registers inside the reggroup More Details


  • UVM Enhancements
  • uvm_add_regmap- A new property has been added for creating additional register maps in UVM for the same bus domain More Details
  • vertical_reuse - A new property applied at chip level to create “relocatable” UVM.


  • IDS-NG™

IDS-NG™ enables users to capture the specification of IP and SoC. It is a platform independent tool with which user can create specifications on all version of RedHat, Ubuntu and others, Windows and Mac OS. More Details

  • IP-XACT Enhancements
  • Parser has been improved to incorporate a larger variety of IP-XACT.
  • General Enhancements
  • “-if “option- When -if command line option is given different files will be generated for different  blocks in a chip with file name same as its module name. More Details
  • "-if_pdf" option- When -if_pdf command line option is given different PDF files are generated for each block in the chip. More Details
  • “-if_svheader” option- using this command line option separate svheader file for all the blocks in a chip are created More Details
  • Reset and initialization- IDS, ARV™-Formal now verifies whether the register has been initialized to its default value or not with respect to reset signal. More Details
  • -top_property - command line option to specify property at top level. This is very useful to add new properties from the command line (e.g. –top_property “coverage=on” More Details
  • Bits column formatted as Text when Excel output is generated.
  • “-mbdc”- using this command line option multiple bus domain table can be overridden from the command line More Details
  • Velocity template (for custom output files), top level and block velocity templates can be run with a single command using "block:<velocity_file>" or "chip:<velocity_file>" . This way, multiple outputs will be generated for various levels. More Details
  • "-velocity"-this command line option is used to generate new custom outputs. User can write their own script to generate files as per their own requirements. More Details
  • "-force_out"- this command line option is used to force fully generate all modules even though modules maybe already generated. More Details


  • Industry Standards 
  • I2C Bus- Now the I2C widget supports the 10-bit addressing mode. Currently in beta. More Details
  • Third Party Interfaces 
  • Qsys Integration- Offsets from Qsys generated file can now be imported in IDS More Details
  • Bug Fixes: No major issue or backward incompatibility in this release!
  • Bits column formatted as msb:lsb in Agnisys® excel - when the lower offset value and the higher offset value are same then it will show only one offset value in PDF. Ex:- for 7:7 will be represented as 7. 
  • SystemC Headers issue- Bug fix in systemC alt2, field offset corrected 
  • RTL external logic error (corner cases)- IP_MASK extended for Duloug output
  • Discontinuous address spaces issue fix
  • Lint errors in widget (axi_widget and axi4full_widget)- Bugs in widget due to blocking and non-blocking assignments resolved.
  • external_logic widget issue- addr_width used instead of bus_width
  • Parametrized memory- Bugs fixes for depth and offset in memory (parameter and parameter override)
  • Parameter name fix when same module instantiated again- at both block and chip level files
  • BUSER and RUSER of different width (AxiBurst)
  • Empty space issue in velocity- bug fix in bus domain, removed additional emptyspace
  • Bit field reading error- Bug fix when register width is greater than the bus width
  • Double quotes issue fixed in Verilog generated from TOP level module- encoding issue for special character
  • Register bundle array offset fix- when repeat is used on register bundle array
  • Alias across block- Supported aliasing of register across different block and when register width is greater than bus width.
  • In IP-XACT when no field is specified in a register then  placeholder field is created.
  • MBD fix in PDF
  • Access type in UVM fix (for W1t)
  • Annotation checks for preceding and succeeding underscores from Verilog removed
  • reserved_bit from velocity- Annotation check removed
  • Parameterized offset fix in Verilog- Bug fix when parameterized offsets are used on register bundle array
  • EmptySpace issue fix in case of MBD- Each set of blocks inside chip will have empty address index starting from zero instead of previously used continuous indices. This was done to make the wire names same both from block as well as chip level
  • Filename issue in case of register width greater than busWdith
