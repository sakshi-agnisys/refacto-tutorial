Python Interpreter

Python API for IDS-Interface

API version : Beta-0.2

Introduction

IDesignSpec™ (IDS) enables users to customize and create new outputs using Python. This document describes the python script that works on the IDesignSpec™ documents. For example, “Templates” in IDS documents translate into “Objects” in python. The Python API introduces 7 object types, each corresponding to a document template, e.g., Property/Value pairs on the templates translate into ‘properties’ within Objects. So for a “block”, the value specified for offset can be accessed by getting the ‘offset’ property of the block object in python.

IDS Object Types and their Properties

The following types of Objects have been introduced in the Python API of IDS.

System

System has following properties:-


name : Name of the register after performing following actions: 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

Board

Board has following properties:-

 

name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.


Chip

Chip has following properties:-

 

name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

 

offset :  Its value is in the same unit as defined by the settings in the IDS Configuration.

 

address :  byte address of the chip.

 

size :  Total size of the chip. Its value is in the same unit as defined by the settings in the IDS Configuration.

 

external : true if the chip is external, else its value is false.

 

Block

Block has following properties:-

 

            name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

 

 

offset :  Its value is in the same unit as defined by the settings in the IDS Configuration.

 

            address :  byte address of the block.

 

size :  Total size of the block. Its value is in the same unit as defined by the settings in the IDS Configuration.

 

            external : true if the block is external, else its value is false.

RegGroup


RegGroup/ Section has following properties:-

 

            name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

 

 

offset :  Its value is in the same unit as defined by the settings in the IDS Configuration.

 

            address :  byte address of the RegGroup.

 

size :  Total size of the RegGroup per iteration. Its value is in the same unit as defined by the settings in the IDS Configuration.

 

            external : true if the RegGroup  is external, else its value is false.

 

            repeat/count :  Number of times the RegGroup is repeated.

 

Reg

 

Register has following properties:-

 

            name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

 

 

offset :  Its value is in the same unit as defined by the settings in the IDS Configuration.

 

address :  byte address of the RegGroup.

 

external : true if the RegGroup  is external, else its value is false.

 

width :  width of the register i.e 8/16/ 32 / 64/128.

 

doc : documentation specified for the register

 

Fields

 

Field has following properties:-

           

             name : Name of the register after performing following actions:

 

  • If ‘Preserve name’ is set in the IDS Configuration, then all spaces are trimmed and then converted to “_” and everything other than underscore, numbers and alphabets is removed.
  • If ‘Preserve name’ is NOT set in the IDS Configuration, then everything except alphabets and numbers is removed.

 

 

             bits :  As specified by the user.

 

             sw_access/sw : Type of software access.

 

             hw_access/hw : Type of hardware access.

 

             default/default : Default value of the field.

 

             high_offset : Value for the high offset.

 

             low_offset : Value for the low offset.

 

             size :  Size of the field.

 

             doc :  Documentation specified for the register.

 

Enum

 

Enum is a simple list. It is not an IDS object. This API provides access to the ‘enum’ definitions and ‘enum’ values used in the register fields.

To access ‘enum’ definitions you can use the functions ‘get_enums’.

IDS enum is represented as a list in python, with the first index being the enum name and the second index has the list of enum fields. Each ‘enum’ field is further a list of mnemonic, value and optional description at indexes 0, 1 and 2 respectively.

Example of ‘enum’:

       {Enum1 {{ON 1 {1 is on}} {OFF 0 {0 is off}}}}

 

Define

 

Define is a simple list. It is not an IDS object.

This API provides access to the ‘define’ definitions. To access ‘define’ definitions you can use the functions ‘get_defines’.

IDS define is represented as a python list having define var, value and optional description at indexes 0,1 and 2 respectively.

Example of ‘define’:

{ON 1 {1 is ON}}

 

Supported Commands

get_IDS_version 

Usage: get_IDS_version()

 

Returns: String.

 

Description:  Return the version of IDesignSpec™.

 

get_py_api_version

Usage: get_py_api_version()

 

Returns: String.

 

Description:  Return the version of python API.

 

get_system

        Usage :  get_system (obj)

 

Arguments: (obj) : Object of type: board, chip, block, reg, and field.

 

Returns : Object of type: system 

 

Description :  Get the enclosing system for a given board, chip, block, register or field. Returns a NULL* Object, in case a system doesn’t exist.

 

get_board

Usage :  get_board (obj)

 

Arguments: (obj) : Object of type: chip, block, reg, and field.

 

Returns : Object of type: board

 

Description :  Get the enclosing board for a given chip, block, register or field. Returns a NULL* Object, in case a board doesn’t exist.

 

get_chip

Usage :  get_chip (obj)

 

Arguments: (obj) : Object of type:  block, reg, and field.

                    

Returns : Object of type: chip

 

Description :  Get the enclosing chip for a given block, register or field. Returns a NULL* Object, in case a chip doesn't exist.

 

get_block

Usage :  get_block (obj)

 

Arguments: (obj) : Object of type:  reg, and field.

 

Returns : Object of type: block

 

Description :  Get the enclosing block for a given register or field. Returns a NULL* Object, in case a block doesn't exist.

 

get_all_sections

Usage :  get_section (obj)

 

Argument (obj): object of system, board, chip, and block

 

Returns : list of all sections

 

Description : Get the enclosing all sections for a given obj. Returns a NULL* Object, in case a section doesn't exist.

 

 

get_fields

 

Usage :  get_fields (obj)

 

Arguments: (obj) : Object of type:  reg

 

Returns : Object of type: field

 

Description :  Returns a list of field Objects for a given register.

 

 

get_objects

 

Usage :  get_objects (obj)

 

Arguments: (obj) : Object of type:  system,board,chip,block.

 

Returns : Object of type: system or board or chip or block or register

 

Description :   Get all objects inside a given object.

 

get_object_by_name

 

Usage :  get_object_by_name (hierarchial_name)

 

Arguments: (hierarchial_name) : Complete name of the desired object separated with a “/” character specifying the complete hierarchy.

 

Returns : Object of type: system or board or chip or block or reg or field.

 

Description :   Get all objects inside a given object.

 

get_top

Usage :  get_top()

Arguments: None

 

Returns : Object of type: system or board or chip or block.

 

Description :  Get the top most Object in the source document

 

 

get_property

Usage: get_property(obj, property_name)

 

Arguments: (obj): Object of any valid type

(property_name) : Name of any valid property

 

Returns:  String: Value of the property.

 

Example:

 

blockName = get_property(top_blk, 'name')

blockSize = get_property(top_blk, 'size')

 

 

Description:  “property_name” is an optional argument for get_prop, when specified, returns the value of the specified property for the given object. Otherwise returns the list of all the available properties on the specified object.

 

get_parent

 

Usage :  get_parent(obj)

 

Arguments: (obj):object of any valid type

 

Returns : Object of type: System, board, chip, block , reggroup or reg

 

Example :   

parent = get_parent (regObj)

 

get_type

Usage :  get_type (obj)

 

Arguments: (obj) : (obj): Object of any valid type

 

Returns : String: Specifying any of the valid types.

 

Description :  Returns the type of IDS object. One of “system”, “board”, “chip”, “block”, “reg”, “reggroup”, “field”. Note that enum and define are not objects but simple python lists.

 

get_enums

 

Usage :  get_enums (obj)

 

Arguments: (obj) : (obj): Object of any valid type

 

Returns : List of all the ‘Enum’ definitions.

 

Description : Returns the list of all ‘Enum’ definitions for the given object.

 

get_defines

 

Usage :  get_defines (obj)

 

Arguments: (obj) : (obj): Object of any valid type

                    

Returns : List of all the ‘Define’ definitions.

                   

Description :  Returns the list of all ‘Define’ definitions for the given object.

 

get_params      

Usage :  get_params (obj)

 

Arguments: (obj) : (obj): Object of any valid type

 

Returns : List of all the ‘Parameters’ definitions.

 

Description :  Returns the list of all ‘Parameters’ definitions for the given object.

get_param_objects        

Usage :  get_params_objects (obj)

 

Arguments: (obj) : (obj): Object of any valid type

Returns : List of all the nodes of the ‘Parameters’ definitions.

 

Description :  Returns the list of all nodes of ‘Parameters’ definitions for the given object.

 

 

get_variants

Usage :  get_variants (obj)

 

Arguments: (obj) : (obj): Object of any valid type

 

Returns : List of all the ‘variant’ definitions.

 

Description :  Returns the list of all ‘Variants’ definitions for the given object.

 

 

get_default_value_enum

 

Usage :  get_default_value_enum (obj)

 

Arguments: (obj) : (obj): Object of type: field

 

Returns : ‘enum’ or an empty string (if no enum is used in the reg field’s default).

                   

Description : Returns the enum for the field’s default value.

 

get_value_constraint

 

Usage :  get_value_constraint (obj)

 

Arguments: (obj) :  (obj): Object of type: field

 

Returns : List containing the min and max values defined in the user. It’s syntax is as follows: {{min <min_value>} {max <max_value>}}

 

Description : Returns a list of set of min and max values for a given field. Returns an empty string if no constraint is defined.

 

 

get_unique_name

 

Usage :  get_unique_name (obj, ref_obj, separator)

 

Arguments: 

(obj): Object of any valid type.

<ref_obj>: This must be a valid parent of the (obj). Accepts, object of type: system, board, chip, block, section

                    

Returns : String

 

Description : Returns a name for a given object which is unique in the complete hierarchy by default. The hierarchy can be limited by specifying a reference Object, which must be a valid parent of the ‘obj’.

 

 

get_reg_width

Usage :  get_reg_width (obj)

 

Arguments: (obj) :  (obj): Object of type: reg.

 

Returns : String

 

Description : Returns the register width for a given register object


get_unit

Usage :  get_unit (obj)

 

Arguments: (obj) :  (obj): Object of any valid type

 

Returns : String

 

Description : Returns the address unit for a given object.

 

get_domains

Usage :  get_domains (obj)

 

Arguments: (obj) :  (obj): Object of any valid type

 

Returns : List of bus domains

Description : Returns the list of all bus domains on which the given object exists.

 

 

get_id

Usage: get_id (obj)

 

Arguments: (obj): Object of type: system, chip, block, memory, reggroup, memory, reg or field.

 

Returns:  String

 

Description: Returns complete hierarchical path to the given object separated by slash “/”. Each node in the path will be a type of that node.

 

is_equal

Usage :  is_equal (obj1, obj2)

 

Arguments: (obj) :  

(obj1): Object of any valid type.

  (obj2): Object of any valid type.

 

Returns : true | false.

 

Description : Checks whether two Objects are equal. Returns true if both objects are equal. Objects are equal only if they refer to the same template in the IDS document.

 

is_null

Usage :  is_null (obj)

 

Arguments: (obj) :  (obj): Object of any valid type

 

Returns : true | false.

 

Description : This check must be used to check if a returned IDS Object is NULL.

 

Example:

 

IDSbatch sample.rdl -py test.py -out “htmlalt2 verilog uvm” -dir IDS1 

 

Python Script: test.py

 

# ******** © 2021 Agnisys®, Inc.  ALL RIGHTS RESERVED ********

 

ver = get_IDS_version()

print (ver)

 

ver = get_py_api_version()

print (ver)

 

 

top_blk = get_top()

print 'Top Object ::', get_property(top_blk, 'name')

 

block = get_property(top_blk, 'name')

 

#getting all the objects inside the "top_block"

blk_objects = get_objects(top_blk)

 

#Iterating over each object

for obj in blk_objects:

    type = get_type(obj)

    if type == "reg":

        

        reg = get_property(obj, 'name')  

        print 'Reg Name:: ', reg

        

        fields = get_fields(obj)

        for fld in fields:

            field = get_property(fld, 'name')  

            props = get_property(fld, '')

            

            for prop in props:

            

                print 'Field ', prop, ':: ', get_property(fld, prop)

            

            print '\n'

        print '\n'
