Ver 5.0.0.0

v5.0.0.0

30 May 2014 


IDS Enhancements:


1. Counters are now generated in RTL - You can now describe a counter and generate Verilog output. For more details, please click here.


2. Special register

Alias register

Lock register


   Describe an alias register and a lock register and generate UVM and Verilog.


3. Automatically show unused spaces in the ToC - IDS now generates the free space in the Table of content in IDS-Word.


4. AMBA3 AHBLite Bus - IDS now supports the AMBA3 AHBLite Bus for the Verilog and VHDL output. For more details, please click here.


5. Error signal - Error signal is now generated for APB and AMBA-AHBLite and AMBA-AHB bus


6. Parameterization in VHDL - IDS gives the override feature on the ref template. User can change the parameter of the referred IDS template while generating the output. (Only supported in IDS-Word)


7. Pipelined Address decode stages - Add address pipeline stages in decode logic with new property “addr_decode_stages”. For more details, please click here


8. Memory Technology Mapping - Instantiation of memory is now supported for Synplify PRO. For more details, please click here


9. C API with header alt1 - Read and write function same as supported in MISRA-C output now supported in C header alt1


10. Backward Incompatibility in IDS : Single register in section become UVM_REG_FILE in UVM - Now single register in section become uvm_reg_file instead of register array. This is an incompatibility with previous releases of IDS.


11. Multidimensional repeat supported in IDS - User can specify the multidimensional repeat on the IDS Template.


Example: repeat=3,4,5,6


This property is supported on register, section and block template. By using this property user can specify the multi dimension on single template instead of using the multiple section in IDS.


a. This property is supported on register, section and block template.


b. By using this property user can specify the multi dimension on single template instead of using the multiple section in IDS.


c. This feature is only supported in UVM and Verilog


12. AMBA AHB widgets – pipelined transfer, error signals (hresp) - AMBA-AHBLITE and AMBA-AHB widgets were updated to supported


a. Pipelined read and write transfers with both wait/no-wait states.


b. Proper error response is now sent through “hresp” signal.


13. IDS-Excel now supports References


14. Generate Assertions from Constraints.  For more details click here.
