Ver 7.86.0.0

v7.86.0.0

(Apr 30th, 2024)


IDesignSpec™ (IDS)


RTL Enhancement


  1. Support for various properties in VHDL output:
    1. B#3044: TMR property More Details
    2. B#3045: ANDED property More Details
    3. B#3046: ORED property More Details
    4. B#3088: parity inject property More Details
    5. B#3061: CRC property More Details
  2. F#23657: Support for “reg_clk = <clock_name>” property at top level for chip in chip case, for Verilog output. More Details


UVM Enhancements


  1. F#23707: Support for interrupt-based callback class removal using the property “ignore_prop=uvm:intr”. More Details
  2. B#2997: Support for “uvm_mem_class=<string>” property for customizing the extended class name for uvm_mem. More Details


Bug Fixes


RTL


  1. F#23680: Fix for the syntax error due to extra “end”s in the generated RTL.
  2. F#23753: Fix for generating package instead of defining them as ports in case of reg_clk property for SV output.
  3. B#3131: Fix for incorrect RTL generated while using SV Struct.
  4. B#3136: Fix for offset mismatch in SVHeader and doc vs IP-XACT seen in IP97. 
  5. B#3149: Fix for rtl.name_format property not working with counter properties.
  6. B#3150: Fix for an extra “end” when the alias property is used with a buffer trigger register.


UVM


  1. F#23674: Fix for fifo with coverage issue for UVM RAL.
  2. B#3145: Fix for wrong behavior when define_as_param property is used with the preprocessor directives.


General


  1. B#3132: Fix for unions created with constants.
  2. B#3133: Fix for idsbatch generating documentation differently from IP-XACT.
  3. B#3134: Fix to find the number of instances from Python API.
  4. B#3135: Fix for duplicate macros getting generated in cheader output.
  5. B#3152: Fix for wrong definition being generated when ispresent property is applied on one of the two instances of a particular register.


IDS-NG


Bug Fixes


  1. B#3035 : Fix for the issue with HW input signal " *_in " not defined for property "registered=false" when using "hw=na", extra parentheses were created for sw= wcrs.
  2. B#3084 : Fix for the issue with the property "clock_name" applied on the chip generating incorrect RTL code when used on reggroup and field generating warning messages.
  3. B#3086 : Fix for verilog output when using the property "default_clk_name" on blocks generating incorrect  RTL output.
  4. B#3076 : Fix for verilog output when signal assigned as output port in signal table for less than the field width value.
  5. B#3107 : Fix for verilog output compile fail in case of counter.sat UDP due to signal (undefined/wrong variable)
  6. B#3078 : Fix for verilog output property "rtl.bmax=true" compile failed due to syntax error.
  7. B#3117 : Fix for verilog output property "stages" out_enb_stages=0 does not generate RTL.
  8. B#3119 : Fix for few changes in the RTL code for FIFO. 
  9. B#3040: Fix for IDS-NG prop_hint for the default value  being generated to be blank.
  10. B#3010: Fix for description entered in the signal table for RTL output only for verilog95 output.
  11. F#23233: Fix for handling multiple ipxact extensions.
  12. F#23521: Fix for UVM output issue containing extra \n.
