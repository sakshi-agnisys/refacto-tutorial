Alias Register

Registers accessible from multiple addresses in the same address map. The fields in aliased registers have different behavior based on the addresses used to access them. This implies the fields in a register may be readable and writable when accessed using one address, but read-only when accessed from another. 

 

Logical view


Template and Property


An aliased register is implemented as two templates in IDS. These two templates are linked using the ‚Äúalias‚Äù property. This property can be applied to a field or a register.


An alias of a register is defined by the property ‚Äúalias‚Äù in a register field. The value of this property is the hierarchical path to original register field which is aliased. If a register has multiple fields then some of its fields can be aliases of other register fields and some can be just normal register fields.


Following rules apply to alias register:

  • The hardware access of the alias register is NA.
  • A field cannot be an alias of an already aliased field.
  • The parent register of an aliased field cannot be external.


Below IDS-Word example shows a register ‚Äúreg1‚Äù at address 0x0000000 having SW access read/write (rw). 

Alias register of "reg1" is  ‚ÄúReg2‚Äù at address 0x0000001 which has fields ‚ÄúF1‚Äù and ‚ÄúF2‚Äù having SW accesses Ro and W1s respectivley.

The aliasing is shown by the property "alias = reg1" applied on "Reg2"


Note: alias registers are supported with repeat in both veriilog and UVM output.

reg event1 { 

      field { 

       hw=rw; 

       sw=rw;

       } Field1[31:16];

          field { 

       hw=rw; 

       sw=rw;

       } Field2[15:0];

addrmap SpecialRegs { 

   event1  reg1 @0x100; 

   alias reg1 event1 Reg2;

   Reg2.Field1->sw = r;

   Reg2.Field2->sw = w;

   Reg2.Field2->woset = true;

   

};

 3-Way Aliasing


3-way aliasing means to access a particular register from 2 different locations i.e. one from within the block and other from another block.


Figure 3 : View of 3-way alias register


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL


IDS-NG Register View



IDS-NG Spreadsheet View



SystemRDL


property IDS_alias { type = string; component = reg | field ; };

property aliased { type = boolean; component = reg | field ; };


addrmap chip {

  addrmap Block1 {

    reg Reg1 {

      field {

         aliased=true;

         hw=r;

         sw=rw;

         }Fld[31:0] = 0;

    };

    reg Reg2 {

      field {

         hw=r;

         sw=rw;

         }Fld[31:16] = 0;

         field {

         aliased=true;

         hw=r;

         sw=rw;

         }Fld2[15:0] = 0;

    };

      Reg1 Reg1;

       Reg2 Reg2;

       

  };

  Block1 Block1;

  addrmap Block2 {

    reg aliasReg1 {

       IDS_alias = "Block1.Reg1";

      field {

         hw = na;

         sw = rw;

         }f[31:0] = 0;

    };

    reg aliasReg2 {

      field {

         hw=na;

         sw=rw;

       }Fld[31:16] = 0;

       field {

         IDS_alias = "Block1.Reg2.Fld2";

         hw=na;

         sw=rw;

         onread = r;

         onwrite = woclr;

       }Fld2[15:0] = 0;

    };

   aliasReg1 aliasReg1;

   aliasReg2 aliasReg2;

  };

 

  Block2 aliasBlock;

 

};

  1. Alias property can be specified on registers as well as fields in IDS Spec. While specifying on the register level, the value of this property should be the hierarchical path to the original register which is aliased. 
  2. Alias registers can be described inside block and regroup. 
  3. A register field can be an alias of some other field of the same register also.
  4. In IDS aliasing is also possible between registers which are in different regroups.
  5. Note that a new version of IDS will support more complex interrupts.
