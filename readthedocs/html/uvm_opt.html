uvm_opt


“uvm_opt=”1”” Property


Introduction


This property removes UVM factory registration macros, e.g., 'uvm_object_utils' which is used to register uvm objects and uvm components respectively with the factory. Instead of the UVM factory "create" in the build function, it will use the "new" keyword.


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL

    uvm_opt = “1”;

                field {

                   hw=rw;

                   sw=rw;

                } f0 [31:0]=1;

             } reg0[3] ;

          };

“uvm_opt=”2”” Property


Introduction


From v7.18.0.0 onwards, we have supported the "uvm_opt = 2" property which replaces the 'h prefix in the arrayed register names with a decimal index.

Previously, this UDP was supported as a boolean type:


property uvm_opt {type = boolean; component = addrmap;};

addrmap chip1 {

uvm_opt = 1;

......

......


From now on, it will be supported as string type:


property uvm_opt {type = string; component = addrmap;};

addrmap chip1 {

uvm_opt = "2";

.....

......


We can use both options at the same time by comma separated i.e.,  uvm_opt = "1,2". In this case both the functionalities associated with the number will work.

“uvm_opt=4” Property


By using -top_property "uvm_opt=4" in the command line, only one package will be created with all the callback classes.


This property is also supported by a top property named uvm_common_pkg="ABC:1:0" which generates an output file ABC_ext_pkg.regmem.sv where ABC is the first argument, second and third argument used to include and import that ABC_ext_pkg.regmem.sv file into top_element_pkg.regmem.sv file and is expalined in second example.

Where :
First argument is ABC which defines the name of the ext_pkg.regmem.sv file.
e.g: ABC_ext_pkg.regmem.sv 


Second argument is a binary digit either 0 or 1 which  includes the `include definition if equal to “1” and remove it if equal to “0”  into the top_component_pkg.regmem.sv

Third argument is a binary digit either 0 or 1 which include the `import definition if equal to “1” and remove it if equal to “0”  into the top_component_pkg.regmem.sv

Support for factory retention by using "-top_property" with "uvm_opt=1"

“uvm_opt=16” Property


The "uvm_opt=16" top property shows a specific behavior where a m_context_id is set for sections. Following is the pattern shown: 


     sect1[sect1_i].m_context_id = sect1_i;


It's important to note that this m_context_id is added only when the "-top_property uvm_opt=16" flag is specified. Otherwise, the mentioned line is not included.


IDSBatch Command Line: idsbatch <input_file> -out uvm -top_property uvm_opt=16


Example: IDS-NG        IDS-Word        IDS-Excel        SystemRDL
