Ver 5.20.0.0

v5.20.0.0

12/28/2015


IDesignSpec™ (IDS-Word, IDS-Excel, IDS-Batch CLI) Enhancements

  • General Enhancements
    • Stop Button (IDS-Word, IDS-Excel) -IDS  now comes with new interface button, that allows you to stop the Check, Generate or Turbo process anytime.
    • Floating IDSEditor license - IDS licensing has been enhanced to support floating license for Editor functionality.
    • Lock Register (UVM, Verilog) - The software write access of a register can now be locked based on an expression consisting of different register fields or some logical expression.
    • Repeat on Block (C++ Header, PDF) - Repeat on block in specification has been enhanced to support PDF and C++ header output.
    • Upgrade Latest IDS version Notification - If a newer version of IDesignSpec™ is available on our website, the Upgrade option will be enabled in IDesignSpec™. 
    • New Properties
      • addr_sort -  This property allows you to specify registers in any order, irrespective of their address/offset.
  • RTL Enhancements
    • AHB Bus Enhancement (Verilog) -  AMBA-AHB standard is now completely supported, along with burst functionality. 
    • AHB-LITE Bus Enhancement (Verilog) -  AMBA-AHB3LITE standard is now completely supported, along with burst functionality. 
    • AMBA-APB Bus Enhancement (Verilog) -  AMBA-APB standard is now completely supported. 
    • Remove Hard Reset (Verilog) - Property to remove hard reset in generated Verilog RTL.
    • Field-Level RTL (Verilog) - The Verilog output is generated on a per field basis instead of per register basis. This makes more field level functionality possible in a later release.
    • <reg>_enb output signal (Verilog, SV) (Minor Backward Incompatibility) - The <register>_enb output signal (used to indicate any write operation from software) is now not created by default if the hardware access for the register is "not readable". Prior to this release, this port was always created when the software interface was writable, irrespective of hardware access. To generate the port, set the "rtl.reg_enb" property to "true". For more details click here.
    • <reg>_<field>_in_enb signal (Verilog) - The hardware interface enable signal <reg>_<field>_in_enb can now be omitted by setting "rtl.hw_enb" property to "false". This port is generated by default when hardware interface can write to the field. This property will be applicable only when the field is not writable from software interface. For more details, please see the RTL Properties.
    • "sharedext" SystemRDL standard (Verilog) - A new property to configure all external registers to share a common bus has been supported. For more details, please see the RTL Properties.
    • Interrupt Registers (Verilog) - Interrupt Registers have been enhanced. There is no restriction on same offset for status or enable and pending interrupt registers, but their field size should be same.
  • TCL Enhancements
    • get_variant - Command to get list of all variants used in the specification. Enhancements have been made in TCL API for custom output generation.
  • Documentation Output Enhancements
    • Remove Define/Variant/Enum Table (HTML, PDF) - Properties to remove define, variant or enum table from the  HTML and PDF outputs. For more details, please see the General Properties.
  • Header Enhancements

The following header enhancements have been made. For more details, please see the C/C++ Properties.

    • "header.struct_guard" (C header alt1, alt2) - Property used to separate macros from union and structures. 
    • "compiler.struct.maxsize" (C header, SV header) - Used to restrict struct size. 
    • "compiler.arrayindex.maxvalue" (SV header) - Used to limit Maximum value of the array index. 
    • "header_package" (SV header) - Property to exclude "package" and "end package" for the SV header output. 


  • New Outputs
    • Excel - An excel format of the register specification can be generated using IDS-Batch. 
    • C API - The C API file will now generate an API file while generating C header output using IDS-Batch. 
  • IDS-Word Enhancements 
    • Remove Properties (Datasheet) - You can now configure the generated Datasheet output. Register templates can be specified without extra properties defined in curly braces.
    • Remove variant’s name from TOC - The name of variants is not shown in IDS Table of content(TOC).
  • Turbo Enhancements 
    • Signals - You can now add signals template,  and create named external ports, in the block or chip level, to connect certain internal design signals to the external world with Turbo. 
    • Block Inside Block - You can now change the block hierarchy to be inside another block at document level or section can be configured to behave as block with Turbo. 
  • Industry Standard Output Enhancements
    • Multiple Bus Domains (SystemRDL) -You can now import a SystemRDL code with multiple bus domains, and generate required outputs. 
  • General Changes
    • "0x" format changed to 'h format, for default value in SV header output.
    • Added Meta-data to SV header, V Header and VHD header outputs.


Bug Fixes


    • Specification validation was not working with Turbo in IDS-Word.


    • Address Sorting in C header output was not working.


    • Large "Field Names" were overlapping with other columns in PDF.


    • Address rows were repeating multiple times, for registers with repeat in HTML.


    • Default value 'h format was not being parsed by IDS-Batch.


    • Naming convention in C Header output were generated wrong on referring a memory multiple times.


    • Naming convention was wrong for Block inside Block.
