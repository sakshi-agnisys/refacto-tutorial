RALF



The Register Abstraction Layer File (RALF) is used to specify all the registers and memories in the design.


IDesignSpec™ Supports RALF as input as well as output. This documentation gives information on how features of RALF can be used in IDesignSpec™.


 

The basic building block in RALF contains field, register, virtual register, memory, regfile, block, system.


Ralf can be used using IDS-Batch with the command given below:

% IDS-Batch <file_name.ralf>  -out “<IDesignSpec™ Outputs>” 


NOTE: When more than one top level is present in “.ralf” file, use “–top” to specify top level (“system_name” or “block name”) in RALF.

User can set the “-block_size”, “-chip_size” and “-address_unit” from the command line argument in IDS-Batch.

For more details of RALF Syntax, please refer to the RALF User Guide. A basic example is shown below:-


Example:


system top {

   block myblk {

        bytes 4;

        endian big;


        register R1 @0x0100 {

             bytes 4;

             field a1 {

                       bits 4;

                   access rw;

                 }

                 field X {

                   bits 7;

                   access rw;

                }

                field a2 {

                  bits 21;

                  access rw;

               }

       }


       register R2 @0x405 {

          bytes 16;

          left_to_right;

            field a1 @'b111 {

             bits 4;

               access rw;

            }                

            field Y @'d35 {

                bits 15;

                access rw;

            }

            field a2 @64 {

                bits 15;

                access rw;

            }

       }           


       memory m {

          bits 32

          size 1024

      }

   }

}
