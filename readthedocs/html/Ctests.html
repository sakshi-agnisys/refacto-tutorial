C-tests

C-tests


C-tests can be used to test IP which can include CPU simulation environment or pure UVM/SV environment.

  1. C-tests can be run on CPU which is connected to IPs with different bus interfaces.
  2. C-tests can be run on UCM/SV environment using the DPI calls. 

Tests Includes:

  • All type access including special accesses.
  • Different types of tests like Random read/write, walking ones, 0’s and 1’s writes.
  • Memory read/Write test with random values, walking ones, 0’s and 1’s writes.
  • Customize the generation of tests using properties.
  • Test for special registers like alias register, shadow register, etc.


Customization can be done using the “-custom_template  <template file> ” option in command line:


<Template file> is a python format yaml file 


Sample : 

configure = {

       'output' :{

            'ctest' : {

                    'mout': 'false', 'default data-type':{                  

                                'Variable':'int' 'template' :{

                                     ‘Default’ :{

                                        'reg_read' : 'read_mirror(%a,%rd)', 'reg_write': 'write_mirror(%a, %d)'

                                                       }

                                   }

                      }

           }

  }


mout – to generate multi-out file

default data-type – for setting the datatype of variable in the c-tests

%a – for address of register

%d – for write data

%rd – for read data


C-tests Properties:

  • ctest_type=sw<+/-><field sw access>:hw<+/-><field hw access>:<test_name>

This property can be applied at top of the test-case or in the “-top_property” command line

option.

Example of possible Values for the property:

  • ctest_type=sw+rw:hw+ro:ReadResetValue 

(Generate ReadResetValue test for register fields having sw access RW and hw access RO)

  • ctest_type=sw+rw:hw-rw-wo:ReadResetValue

(Generate ReadResetValue test for register fields having sw access RW and not hw access RW/WO)

  • ctest_type=sw+rw::WalkingOnes 

(Generate WalkingOnes test for register fields having sw access RW and any hw access)

(Note: It is mandatory to generate the c-headers with the c-tests)
