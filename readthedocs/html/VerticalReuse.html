Vertical Reuse

Vertical reuse

(Compatible with IDS version 6.8.0.0 and above)


When a block is vertically reused in a chip/system environment (top level flow). The block related UVM classes (register UVM classes and block UVM class) are getting regenerated again by IDS at the chip/system level. It shouldn't be regenerated but should reuse the block related UVM class definitions generated at the block/IP level. For this property vertical_reuse can be used. The value of this property can be "true" or "false".

Top Level Flow without generation of register level outputs

When we apply vertical_reuse=true at the top level with out_third_party="d" feature, only the intermediate and top-level files will be generated and the files containing register (that is, at the leaf level) will not be generated.


Vertical_reuse command line option


"-vertical_reuse" command line option is used to create package for all level of hierarchy, i.e., Chip/Block/Subblock. Each package imports its corresponding child container package. This option works with "-if" or "-if_uvm" command line option. It generates separate file for each block and chip.


Enhanced support of vertical reuse  

   

In IDS, when a block is vertically reused in a chip/system environment (top level flow), the block related UVM classes (register UVM classes and block UVM class) are getting regenerated again by IDS at the chip/system level. It shouldn't be regenerated but should reuse the block-related UVM class definitions generated at the block/IP level. For this property, vertical_reuse can be used.


“vertical_reuse” property can have the following values, “vertical_reuse=verilog” for verilog output and “vertical_reuse=uvm” fo uvm outputs, in case of uvm output it is similar to “vertical_reuse=true” property. Property can also have both the values comma separated, i.e., “vertical_reuse=verilog,uvm”.

1) Top file

       

2) Referred file (ref.idsng)




SystemRDL input:


1)Top file

 property vertical_reuse {type = string; component=addrmap; };

 property chip {type = boolean; component=addrmap; };

 property refpath {type = string; component=addrmap|regfile|reg;};


           addrmap ChipTop {


            chip=true;

 

            addrmap B1 {


             vertical_reuse="verilog";

             refpath="ref.rdl:Block1";

            };

             reg {

              }F1[31:0]=0;

            };


           B1 B1;

           Block2 Block2;

           };
