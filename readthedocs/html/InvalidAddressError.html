Invalid Address Error



In IDesignSpecTM (IDS), error gets asserted whenever a bus tries to do read/write operation on undefined addresses. Currently,while asserting error, IDS does not consider the type of bus transaction, i.e., read/write, being taking place. So, to make it based on bus transaction, a property ‘invalid_addr_err’ has been introduced. The possible value for this property can be ‘read’ or ‘write’ and it can be applied at the block level only. 

When the value of this property is set to ‘write’,  i.e., {invalid_addr_err=write}, then invalid error will only get asserted for write transactions of bus and will return zero for read transaction. Similarly, invalid error for read transactions will be asserted and zero will be returned for write transactions on unspecified  addresses, if ‘read’ is passed as a value to the property.

invalid_addr_err=razwi


RAZWI(read-as-zero-write-ignore):- The write transactions of the bus are ignored in case the empty address decode gets asserted and read as zero by default.


The possible value for this property can be ‘read’ or ‘write’ or ‘razwi’. ‘invalid_addr_err=razwi’ it can be applied at block and chip level.


When the value of this property is set to ‘razwi’ i.e., {invalid_addr_err=razwi}, then no invalid address error will be asserted neither for write transaction nor read transaction of bus.


When the value of this property is set to ‘write’,  i.e., {invalid_addr_err=write}, then invalid error will only get asserted for write transactions of bus and will return zero for read transaction. Similarly, invalid error for read transactions will be asserted and zero will be returned for write transactions on unspecified  addresses, if ‘read’ is passed as a value to the property.


Note- This property will have impact only on Verilog output.
